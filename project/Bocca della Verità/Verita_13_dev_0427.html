00. 문서 정보 및 프로젝트 개요

작업 범위 네이버 지도 API 연동, 위치 기반 검색, FastAPI 백엔드 구축, ELECTRA 모델 통합
핵심 목적 네이버 지도 리뷰의 광고 여부를 AI로 분석하여 "진짜 리뷰" 요약 정보 제공

본 문서는 광고성 블로그 리뷰를 필터링하기 위한 기술적 구현 사항을 담고 있습니다. 프론트엔드는 Flutter Web을, 백엔드는 FastAPI를 사용하며, 광고 판별에는 파인튜닝된 ELECTRA 모델을 적용했습니다.

Architecture

01. 시스템 아키텍처

1.1. 전체 데이터 흐름

// Search Pipeline
Flutter → API 요청 → FastAPI
  ├─ Supabase 캐시 확인 (HIT 시 요약 반환)
  └─ 캐시 MISS 시:
       Naver 블로그 API 수집 → Supabase 저장 → ELECTRA 광고 판별 → LLM 요약 → 반환

1.2. 지도 기능 아키텍처 (Proxy 구조)

보안 및 일관성을 위해 네이버 지도 API를 백엔드에서 프록시 처리합니다.

  • Renderer: flutter_map (OpenStreetMap 타일 활용)
  • API Proxy: FastAPI를 통해 네이버 역지오코딩 및 로컬 검색 수행
  • 결정 사항: API 키 노출 방지 및 검색 키워드 일관성 유지를 위해 FastAPI 프록시 채택

2.2. places.py (위치 기반 식당 검색)

  • 병렬 다중 검색: 네이버 API의 한계를 극복하기 위해 10개 키워드(맛집, 카페 등)를 asyncio.gather로 병렬 호출.

AI & Data

03. AI 모델 및 데이터 정제

3.1. 리뷰 데이터 HTML 정제 (_clean)

검색 API에서 반환되는 불필요한 강조 태그 및 엔티티를 정규식으로 제거합니다. 

def _clean(text: str) -> str:
    text = re.sub(r"<[^>]+>", "", text) # 태그 제거
    text = unescape(text)                # HTML 엔티티 디코딩
    return text.strip()

3.2. ELECTRA 모델 통합 정보

항목 내용
아키텍처 ElectraForSequenceClassification
베이스 모델 koelectra-base-v3 (vocab 30000)
레이블 0 (비광고) / 1 (광고)

⚠️ 주의: numpy 2.x와 torch 충돌 방지를 위해 numpy<2.0 다운그레이드가 필요합니다.
