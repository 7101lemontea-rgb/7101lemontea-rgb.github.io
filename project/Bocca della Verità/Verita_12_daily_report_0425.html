Day 1

프로젝트 기획 및 설계

주요 완료 사항

  • 방향 확정: 블로그 리뷰 내 광고 필터링 및 진짜 리뷰 요약 제공 서비스.
  • 기술 스택: FastAPI, Flutter, Supabase, OpenAI API 선정.
  • 아키텍처 설계: 캐시 우선 탐색 → 크롤링 → LLM 분석 → 응답 구조 확정.
  • DB 스키마: reviews 테이블 내 is_ad_llm_pred, is_ad_bert_pred 컬럼 분리.

논의 사항: 협찬/제공/무료체험 등 광고 판별 기준 정의 및 OpenAI API 대체 결정.

Day 2

백엔드 기초 구현

주요 완료 사항

  • FastAPI 초기 세팅: 라우터 및 서비스 레이어(Naver, Supabase) 분리.
  • Naver API 연동: 최대 20개 포스트의 제목, URL, 작성일 파싱 기능 구현.
  • Supabase 서비스: 캐시 조회 및 데이터 Bulk Insert 함수 구현.
// 이슈 및 해결
1. supabase_url is required: load_dotenv() 누락 및 URL 슬래시 문제 해결
2. API 인증 실패: 하드코딩된 키를 os.getenv() 환경변수 방식으로 수정
Day 3

LLM 연동 및 API 완성

주요 완료 사항

  • LLM 서비스: GPT-4o를 이용한 classify_ad(광고 판별) 구현.
  • 요약 기능: GPT-4o-mini를 활용한 비광고 리뷰 3줄 요약 로직 구축.
  • 통신 설정: CORSMiddleware 추가로 Flutter와 FastAPI 간 통신 활성화.

Trouble: AsyncOpenAI 클라이언트 교체 및 SQL Editor를 통한 직접 테이블 생성으로 이슈 해결.

Day 4

Flutter 연동 및 통합 테스트

주요 완료 사항

  • Flutter 모델: ReviewItem 및 SearchResult 모델 구현.
  • ApiService: Dio 기반의 API 호출 및 자동 모델 파싱 로직 완성.
  • 통합 테스트: 검색부터 LLM 분석, 요약, Flutter 화면 출력까지의 전체 플로우 검증.
  • 캐시 검증: 동일 검색어 재요청 시 DB 캐시 즉시 반환 동작 확인.
