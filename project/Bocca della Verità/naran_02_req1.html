01 — As-Is Analysis

현황 분석

핵심 페인 포인트: 기존 도구는 단순 단어 번역에 그쳐 교과 개념 설명이 불가하며, 부모 또한 한국어 교과 내용을 지도하기 어려운 환경에 처해 있습니다.

1.1 대상 도메인 및 서비스 범위

다문화 가정 초등학생의 수학 교과 학습을 보조하며, 특히 베트남어권 학생과 부모가 방과 후 학습 상황에서 겪는 언어적 장벽을 해소하는 데 집중합니다.

1.2 현재 사용자 행동 흐름

Step
사용자 여정
문제 접함 → 지문 이해 불가 → 부모님 질문 → 설명 실패 → 학습 포기/이해 없이 통과

1.3 현재 문제점 진단 (정량/정성)

항목 현황 데이터 주요 문제점
다문화 학생 수 약 20만 명 (2025) 맞춤형 학습 지원 체계 부족
국적별 비중 베트남 31.3% 가장 큰 비중에도 전용 도구 부재
해결 소요 시간 개념당 20~30분 학습 효율성 저하 및 집중력 하락
  • 번역기의 한계: 교과 개념이 아닌 단어 단순 매칭만 제공
  • 콘텐츠 부재: 학생 수준에 맞는 이중언어 교과 해설 콘텐츠 전무
  • 학습 격차: 이해 없이 넘어가는 악순환으로 인한 기초 학력 결손 누적
02 — To-Be Requirements

요구사항 정의

2.1 프로젝트 목표 (10일 MVP 기준)

수학 1개 단원 범위 내에서 한국어와 베트남어로 즉시 질문하고 AI로부터 쉬운 설명과 예시를 받을 수 있는 최소 기능 학습 도우미를 구현합니다.

2.2 기능적 요구사항 (Functional)

FR-001 Text Question
문제 해설 화면에서 모르는 개념을 텍스트로 질문 가능
FR-002 AI Response
한국어 또는 베트남어로 개념 설명 및 관련 예시 제공
FR-003 Concept Retry
오답 발생 시 해당 문제와 연계된 개념 재설명 기능

2.3 비기능적 요구사항 (Non-Functional)

항목 요구사항 상세 내용
성능 응답 속도 AI 응답 생성 시간 5초 이내 목표
사용성 직관적 UI 초등학생 대상 단순 UI/UX, 모바일 최적화
안정성 에러 처리 API 오류 시 재시도 로직 및 안내 메시지
03 — MoSCoW Priority

요구사항 우선순위

Must Have 핵심 구현 대상
  • AI 개념 질의응답 엔진 구축
  • 이중언어(한-베) 프롬프트 엔지니어링
  • 문제 풀이 및 인터랙션 UI
Should/Could 확장 기능
  • 오답 기반 개념 재설명 (Should)
  • 핵심 용어 대역어 표시 기능 (Should)
  • 학습 이력 저장 및 난이도 조절 (Could)
Summary

종합 요약

해결 방향: 학생이 학습 중 막히는 순간 친구처럼 물어볼 수 있는 이중언어 AI 학습 도우미를 통해, 보호자의 도움 없이도 스스로 개념을 이해하고 문제를 해결할 수 있는 환경을 조성합니다.
