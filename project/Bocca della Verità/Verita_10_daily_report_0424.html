Milestones

02. 오늘의 목표 및 실제 수행 작업

2.1. 데이터 확보 및 Supabase 업로드

광교중앙역 주변 식당 데이터를 확보하고, 과적합 방지를 위해 수집 개수를 최적화했습니다.

  • Naver API: 식당 1,194개 중 리뷰 495개 추가 확보 및 업로드 완료.
  • Playwright: 식당 1,075개 데이터 확보 및 CrawlingReviewList 업로드 완료.

2.2. Flutter 개발 환경 안정화

경로 오류로 인한 빌드 불능 상태를 복구하고 웹 환경 대응을 위한 엔진 교체를 결정했습니다.

  • Git 충돌 해결: lib/ 디렉토리 경로 수정 및 pubspec.yaml 의존성 복구.
  • 지도 엔진 교체: flutter_naver_map(Native 전용)을 flutter_map(Web 지원)으로 교체.

03. 이슈 상황 및 해결

// 이슈 1: Git Merge Conflict (프로젝트 경로)

// 이슈 2: flutter_naver_map 웹 실행 불가

해결: 데이터 요청 로직은 유지하되 렌더링만 flutter_map으로 교체

04. 내일의 계획 및 회고

Next Step

Flutter(프론트) - FastAPI(백엔드) - AI 모델 - Supabase(DB)를 연동하는 전체 시스템 아키텍처 구현에 집중할 예정입니다.
