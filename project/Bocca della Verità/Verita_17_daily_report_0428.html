Deep Dive

02. 상세 작업 내역

2.1. AI 분석 모드 전환 기능 구현

앱 설정 화면에서 분석 모델을 런타임에 동적으로 전환할 수 있도록 analysisModeProvider(Riverpod)를 통한 전역 상태 관리를 구현했습니다[cite: 3]. 프론트엔드에서 모드를 선택하면 백엔드 및 Supabase 컬럼(is_ad_electra_pred / is_ad_llm_pred)이 모드에 따라 동적으로 선택되어 파싱됩니다[cite: 3].

2.2. ELECTRA → OpenVINO 변환 적용

PyTorch 기반 .safetensors 모델을 OpenVINO(.xml/.bin) 형식으로 변환하여 추론 성능을 개선했습니다[cite: 3]. 로드 방식을 OVModelForSequenceClassification으로 변경하고, 토크나이저는 PreTrainedTokenizerFast를 직접 사용하여 경량화를 달성했습니다[cite: 3].

2.3. 워드클라우드 기능 구현

네이티브 의존성 없이 Flutter CustomPainter로 직접 워드클라우드를 구현했습니다[cite: 3]. review_title 기반 키워드를 추출하고, 고정 시드(Random(42))를 통해 빈도에 비례한 12~28px 크기의 레이아웃을 일정하게 유지합니다[cite: 3].

2.4. 화면 구조 개선 및 통합

별도 페이지였던 BlogListScreen을 제거하고 RestaurantDetailScreen 하나로 통합했습니다[cite: 3]. 화면 상태를 5단계(initial → checking → noData/analyzing → loaded)로 세분화하여 UX를 개선했습니다[cite: 3].

2.5. 음식점 카테고리 이미지 시스템 구축

카카오 API의 1차 분류를 파싱하여 한식, 일식 등 14개 카테고리에 대한 이미지 매핑 시스템을 구축했습니다[cite: 3]. 헤더용(800x500) 및 썸네일용(400x400) 에셋을 분류하고 3단계 fallback 구조를 적용했습니다[cite: 3].

2.6. 리뷰 신뢰도 등급화 및 UI 적용

광고 확률(0.0~1.0)을 3단계 등급(상/중/하)으로 변환하여 카드 배경 및 테두리 색상을 차별화했습니다[cite: 3]. _AdGradeBadge 위젯과 150ms 전환의 Hover 효과를 적용하여 가시성을 높였습니다[cite: 3].

2.7. LLM 리뷰 분석 기능 변경 (float 기반)

기존 이진 분류를 확률값(is_ad_prob) 출력 방식으로 변경하여 모델 간 등급 체계를 통합했습니다[cite: 3]. summarize_reviews() 함수를 추가하여 광고 확률 0.6 미만의 진성 리뷰만 GPT로 3줄 요약합니다[cite: 3].
