01. Git — Flutter 프로젝트 경로 충돌

1.1. 증상 및 원인

Git merge 과정에서 프로젝트 루트 경로가 어긋나 lib/ 디렉토리가 프로젝트 외부에 위치하게 되었습니다. 이로 인해 pubspec.yaml과의 상대 경로가 파손되어 Flutter가 소스 코드를 인식하지 못하는 오류가 발생했습니다.

1.2. 해결 순서

  • pubspec.yaml 위치를 기준으로 lib/ 디렉토리를 동일 레벨로 재배치
  • Merge 충돌로 소실된 dependencies 항목 복구
  • 프로젝트 전반의 import 절대/상대 경로 수정
# 환경 재설정 및 빌드 테스트
flutter doctor
flutter pub get
flutter run
Flutter Cross-platform

02. Flutter — 지도 라이브러리 웹 실행 불가

2.1. 증상 및 원인

flutter_naver_map 라이브러리는 네이티브 SDK를 래핑한 형태이므로 Android/iOS만 지원합니다. 이로 인해 크롬(Web) 환경에서 지도 렌더링을 시도할 때 런타임 에러가 발생했습니다.

2.2. 해결: 라이브러리 교체

웹 환경을 공식 지원하는 flutter_map으로 렌더링 엔진을 교체했습니다. 데이터를 요청하는 네이버 지도 API 로직은 렌더링 라이브러리와 독립적이므로 기존 코드를 그대로 유지했습니다. 

// pubspec.yaml 의존성 변경
dependencies:
  flutter_map: ^6.1.0
  latlong2: ^0.9.0
구분 라이브러리 (변경 후) 플랫폼 지원
지도 렌더링 flutter_map Android / iOS / Web
데이터 요청 네이버 지도 API 플랫폼 무관 (HTTP)
