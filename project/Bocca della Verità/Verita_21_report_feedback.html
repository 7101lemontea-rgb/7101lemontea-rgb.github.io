Architecture & Structure

01. 구성 및 구조 개편

  • 협업 프로세스 통합: Jira와 Git 브랜치 전략을 별개 페이지가 아닌 하나의 협업 워크플로우로 통합 구성합니다.
  • 명칭 변경: '서비스 소개'를 프로젝트 본연의 의미를 담은 '프로젝트 소개'로 변경합니다.
  • 네비게이션 보강: 각 보고서 상단에 대분류(간지)를 표기하고, 전체 프로젝트 타임라인 페이지를 신규 추가합니다.
  • 개인정보 보호: 팀 소개 섹션에서 전화번호와 이메일을 삭제하여 정보 공개 리스크를 차단합니다.

Logic & Concept

03. 용어 정의 및 모델 설계 보강

"네이버 검색 API는 블로그 리뷰의 '요약 텍스트'를 제공받는 방식이며, 이는 URL에 직접 접속해 전문을 긁어오는 '크롤링'과는 명확히 구분되어야 합니다."

모델링 전략 보강

  • 사전 정의: 모델 설계 전 "우리가 구축하고자 하는 모델의 목적"을 명시하는 페이지를 선행 배치합니다.
  • 계열 명시: BERT와 ELECTRA가 동일 계열 모델임을 명시하고, 모델링 과정의 작업 효율성을 강조합니다.
  • 분류 메커니즘: 이진 분류 모델의 Confidence 스코어를 기반으로 사용자에게 '상·중·하' 등급을 출력하는 로직을 구체화합니다.

구분 적용 및 개선 사항
