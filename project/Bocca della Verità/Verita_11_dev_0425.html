Overview

01. 프로젝트 개요 및 기술 스택

사용자에게 광고성 리뷰를 배제한 순수 방문 후기만을 제공하기 위해 최신 AI 기술과 크롤링 파이프라인을 결합했습니다.

영역 적용 기술
데이터 수집 Python, Playwright, 네이버 검색 API
서버/AI FastAPI, OpenAI API (GPT-4o)
모바일 앱 Flutter, flutter_map, Riverpod
인프라/DB Supabase (PostgreSQL)
Architecture

02. 시스템 아키텍처 및 구조

2.1. 데이터 처리 워크플로우

// Flutter → FastAPI → Supabase → LLM
1. Supabase 캐시 확인 (기존 검색어 존재 시 즉시 반환)
2. 캐시 부재 시 Naver 검색 API 호출 및 블로그 수집
3. 수집된 Raw 데이터를 Supabase에 저장
4. LLM API(GPT-4o)를 통한 광고 분석 및 필드 업데이트
5. 최종 필터링된 결과를 Flutter로 반환

2.2. 디렉토리 구조 명세

  • Backend: routers/(엔드포인트), services/(핵심 로직), models/(스키마)로 구성된 레이어드 아키텍처.
  • Frontend: features/ 단위로 지도(map)와 상세페이지(detail) 기능을 모듈화.
Database

03. 데이터베이스 설계 (DB Schema)

수집된 리뷰 데이터와 AI 분석 결과를 효율적으로 관리하기 위한 reviews 테이블 구조입니다.

컬럼명 타입 설명
name TEXT 사용자 검색어 (인덱싱)
review_url TEXT 블로그 포스트 원문 주소
is_ad_llm_pred INTEGER LLM 분석 결과 (0: 일반, 1: 광고)
created_at TIMESTAMP 데이터 저장 시각 (인덱싱)
Modules

04. 주요 모듈 및 로직 설명

4.1. LLM Service (광고 판별 및 요약)

GPT-4o 모델을 활용하여 광고 여부를 판별합니다. 판별 기준은 다음과 같습니다.

  • 협찬·제공·무료체험 등 대가성 문구 포함 여부
  • 과도한 긍정 표현 및 핵심 정보(가격 등) 결여 여부
  • 과도한 해시태그 나열 패턴 분석

4.2. Supabase Service (캐싱 전략)

동일한 검색어에 대해 불필요한 API 호출을 방지하기 위해 get_cached_reviews()를 통해 응답 속도를 최적화했습니다.
