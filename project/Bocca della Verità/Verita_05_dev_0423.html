Architecture

01. 전체 시스템 흐름 및 방식 비교

사용자가 입력한 식당 데이터를 기반으로 신뢰할 수 있는 리뷰를 추출하기 위해 두 가지 수집 경로를 병행 운영합니다.

항목 Playwright 크롤링 네이버 검색 API
입력 방식 네이버 지도 링크 식당 이름
리뷰 정확도 해당 식당 전용 수집 검색 키워드 기반
본문 수집 전체 본문 가능 미리보기(300자) 제한
광고 탐지 하단 대가성 문구 탐지 가능 불가
속도/비용 느림 / 무료 빠름 / 무료(제한적)
Crawling Engine

02. 방식 1: Playwright 크롤링

브라우저를 직접 제어하여 네이버 지도의 iframe 구조를 뚫고 실제 블로그 본문 텍스트를 추출합니다.

2.1. 환경 구성 및 파일 구조

# 환경 설치
pip install playwright requests
playwright install chromium
  • main.py: 수집 프로세스 총괄 진입점
  • map_crawler.py: 지도 페이지에서 블로그 URL 목록 추출
  • capture.py: 개별 블로그 본문 텍스트 스크래핑
API Integration

03. 방식 2: 네이버 검색 API

공식 API를 사용하여 대량의 검색 결과를 신속하게 확보합니다. 다만, 미리보기 텍스트의 한계로 인해 광고 판별에는 제한이 있습니다.

판별 신호 예시 신뢰도
제목 "내돈내산" "내돈내산 후기" 중
해시태그 위주 "#광교맛집 #추천" 중
대가성 문구 "소정의 원고료..." 미리보기 포함 안됨 ❌

※ 실측 결과: 20개 샘플 중 광고 의심 15개(75%), 일반 5개(25%)로 API 미리보기만으로는 신뢰도가 낮음.

Flutter Application

04. Flutter 앱: Truth Map

분석된 데이터를 지도로 시각화하여 사용자에게 '진실 점수(Truth Score)'와 함께 제공합니다.

4.1. 기술 스택 및 데이터 모델

  • 지도 SDK: flutter_naver_map ^1.2.0
  • 상태 관리: Riverpod (Generator 포함)
  • 디자인 토큰: primary(#1A1A1A), mapTeal(#4BBFBF), markerMid(#F5C518) 
// MarkerType 분류 로직
High: 80점 이상 (Black Marker)
Mid: 70~79점 (Gold Marker)
Low: 70점 미만 (Gold Marker)

05. 트러블슈팅 및 다음 단계
