Issue 01

01. 카카오맵 SDK 로드 실패

Flutter Web 환경에서 카카오맵 라이브러리를 호출할 때 네트워크 에러가 발생하며 화면이 렌더링되지 않는 이슈입니다.

에러 코드 [object ProgressEvent] (JavaScript Network Error)
발생 위치 프론트엔드 (Flutter Web)
원인 분석 카카오 개발자 센터 등록 도메인과 실제 실행 포트 불일치

해결 방법

Flutter Web 실행 시 무작위로 할당되는 포트를 고정하고, 해당 주소를 카카오 플랫폼 설정에 등록했습니다. 

# Flutter 실행 시 포트 8080으로 고정
flutter run -d chrome --web-port 8080
Issue 02

02. API 파라미터 유효성 검사 실패 (FastAPI)

백엔드 Query 모델에 정의된 제약 조건보다 큰 값을 프론트엔드에서 요청할 때 발생하는 422 Unprocessable Entity 에러입니다. 

# FastAPI 백엔드 수정 사항
// 변경 전: display: int = Query(default=10, le=10)
display: int = Query(default=20, le=30)

요청 가능한 최대 범위(le)를 30으로 상향 조정하여 유연성을 확보했습니다.

Issue 03

03. 카카오 로컬 API 정책 위반

백엔드 규격은 통과했으나, 카카오 API 자체의 제한 사항(size 최대 15)을 초과하여 400 Bad Request가 반환된 사례입니다.

방어적 프로그래밍 적용

클라이언트의 요청 값이 카카오 API의 허용 범위를 넘지 않도록 백엔드에서 min() 함수를 사용해 상한선을 강제했습니다. 

# 카카오 API 호출 전 파라미터 전처리
kakao_size = min(display, 15)
params = {"query": query, "size": kakao_size}
Issue 04

04. 네트워크 소켓 바인딩 충돌

서버 재시작 과정에서 이전 프로세스가 포트를 점유하고 있어 발생하는 시스템 레벨의 에러입니다.

에러 코드 SocketException (errno = 10048)
원인 지정한 포트(8080, 8000 등)를 타 프로세스가 이미 사용 중

프로세스 강제 종료 절차

# 1. 점유 포트의 PID 확인
netstat -ano | findstr :8080

# 2. 해당 PID 종료
taskkill /F /PID [확인된_PID]

05. 문제 해결 요약

문제 유형 에러 코드 핵심 해결책
SDK 로드 실패 ProgressEvent 포트 고정 및 도메인 화이트리스트 등록
FastAPI 검사 실패 422 Error Query 모델 le 파라미터 수정
카카오 정책 위반 400 Error size 상한선 15 적용 (Min logic)
소켓 바인딩 충돌 errno 10048 PID 추적 후 프로세스 Kill 또는 포트 변경
