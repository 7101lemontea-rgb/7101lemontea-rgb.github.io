02. 데이터 수집 및 DB 업로드

2.1. 수집 데이터 최적화

모델의 과적합(Overfitting)을 방지하기 위해 가게당 리뷰 수집 개수를 50개에서 20개로 조정하여 총 1,575개(API 500개 + 크롤링 1,075개)의 데이터셋을 확보했습니다.

2.2. Supabase 테이블 매핑

  • APIReviewList: 검색 API 결과물 (파일명 → name, description → review_description 등)
  • CrawlingReviewList: Playwright를 통한 상세 본문 데이터

03. Flutter 및 Git 이슈 해결

3.1. 경로 오류 및 충돌 해결

Git 병합 과정에서 발생한 프로젝트 경로 이탈 및 pubspec.yaml 의존성 누락 문제를 해결하였습니다.

3.2. 지도 라이브러리 교체 (Cross-platform 대응)

기존 flutter_naver_map은 네이티브 의존성으로 인해 웹(Web) 실행이 불가능한 한계가 있어 라이브러리를 변경하였습니다.

  • 변경 전: flutter_naver_map (Native 전용)
  • 변경 후: flutter_map (Web/Mobile 공용)
  • 유지 사항: 데이터 요청용 네이버 지도 API 로직은 동일하게 유지
