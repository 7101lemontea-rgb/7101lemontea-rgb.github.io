01. 데이터 수집 파이프라인 구현

1.1. Playwright 크롤링 고도화

네이버 지도의 복잡한 iframe 구조와 단축 URL 리다이렉트 문제를 해결하여 안정적인 수집 환경을 구축했습니다.

  • 단축 URL: 리다이렉트 추적 로직 추가로 실제 지도 URL 도달 성공.
  • iframe 대응: pcmap.place.naver.com 프레임 직접 탐색으로 전환.
  • 결과: 블로그 URL 10개 수집 성공 (약 47초 소요).

1.2. 네이버 검색 API 병행 도입

법적 리스크 최소화 및 속도 향상을 위해 공식 API 방식을 추가 구현했습니다. 

// API 수집 결과 예시
식당 이름: 까몬 광교중앙역점
수집된 리뷰: 20개
한계: 미리보기(150자) 제한으로 대가성 문구 탐지 어려움
Flutter Dev

02. Flutter 앱 UI 구현

'Truth Map'의 핵심 컴포넌트를 설계하고 네이버 지도 위에 최종 조립을 완료했습니다.

2.1. 주요 구현 컴포넌트

  • TruthScoreMarker: 점수별 색상이 자동 분기되는 커스텀 마커.
  • RestaurantBottomSheet: AI 리뷰 요약 및 식당 정보를 담은 하단 카드.
  • State Management: Riverpod을 이용한 지도 검색 및 선택 상태 관리.

2.2. 트러블슈팅 (Troubleshooting)

현상 원인 및 해결
Async Error 줌 버튼 콜백 내 await 사용 불가 → async 블록으로 수정
Device unauthorized USB 디버깅 미승인 → 기기 내 팝업 승인 처리
NumPy 충돌 v2.x 버전 이슈 → 가상환경 재생성 및 numpy<2 고정
