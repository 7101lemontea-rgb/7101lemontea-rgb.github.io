Part A — UI/UX Optimization

레이아웃 및 시각화 개선

A1. 워드클라우드 고도화 (word_cloud_card.dart)

  • 키워드 정제: 빈도 2회 이상 단어 중 상위 8개만 추출하여 정보 밀도 최적화
  • Painter 교체: 사각형 레이아웃을 BubbleWordCloudPainter 말풍선 UI로 변경
  • 버그 수정: 텍스트 하단 치우침 해결 및 특수문자 전처리 로직 강화

A2. Sliver 기반 스크롤 시스템 (restaurant_detail_screen.dart)

스크롤 충돌을 방지하기 위해 CustomScrollView를 도입하고 복합 레이아웃을 구성했습니다.

  • 병렬 배치: AI 분석 카드(40%)와 워드클라우드(60%)를 높이 200px로 고정 배치
  • 리스트 통합: shrinkWrap: true와 NeverScrollableScrollPhysics를 사용하여 외부 스크롤과 동기화
  • 자동 분석: 캐시 미존재 시 대기 화면 없이 즉시 자동 분석을 실행하도록 UX 단순화
Part B — AI Engine

광고 판별 프롬프트 및 파이프라인 (classify_ad.py)

단순 이진 분류를 넘어 가중치 시스템을 도입하여 판별의 신뢰도를 높였습니다.

B1. 가중치 기반 점수 시스템

판별 신호 가중치 판별 신호 가중치
협찬/제공 문구 명시 +0.7 단점 및 불만 언급 -0.4
일방적 찬사 위주 +0.2 부정적 감정 표현 -0.2
해시태그 10개 이상 +0.1 구체적 개인 경험 -0.1
안정성 확보: temperature=0 설정을 통해 결과의 일관성을 확보하고, JSON 파싱 실패 시 중립값(0.5)을 반환하는 예외 처리를 구현했습니다.
Part C — Technical Analysis

전체 스택 기술 트레이드오프 요약

항목 현재 스택 (Flutter/FastAPI) React 생태계
개발 언어 Dart + Python TypeScript 단일 풀스택 가능
SEO 어려움 (Canvas 렌더링 한계) Next.js SSR/SSG로 우수
AI/ML 통합 FastAPI로 직접 연동 우수 별도 Python 서버 필요
타입 안전성 컴파일 타임 보장 (Dart+Pydantic) TypeScript + Zod 조합 필요
DB 자유도 PostgreSQL 풀 활용 (Supabase) Firebase 등 NoSQL 제약 존재
번들 로딩 Flutter Web 다소 불리 Vite/Next.js 활용 시 유리
