실험 A 학습 데이터 70/30 분할 (같은 가게) 385건 0.9793 0.9727 -0.0065
실험 B 별도 테스트셋 (새로운 가게) 249건 0.5844 0.6312 +0.0469

2.2. 실험 C: SHAP 토큰 기여도 분석

광고/비광고 각 5개 샘플(총 10개)을 대상으로 PartitionExplainer 분석 실시. 가게명 토큰은 기여도 Top 20에 포함되지 않음 → 가게명 직접 의존 아님을 확인.

03. 개선 조치: 학습 데이터 재구성

3-1. 가게 편향 제거

특정 가게에 집중된 샘플 구성을 해소하고, 다양한 가게의 리뷰가 고르게 포함되도록 재구성. 모델이 특정 가게 문체가 아닌 광고 리뷰의 본질적인 표현 패턴을 학습할 수 있는 환경 마련.

3-2. 가게명 마스킹 적용

학습 및 추론 과정에서 텍스트 내 가게명을 마스킹 처리. 가게명이 모델 판단에 노이즈로 작용하지 않도록 구조적으로 차단.

마스킹 설계 기준

항목 내용
브랜드명 추출 업종어(베이커리, 스테이크 등)와 지점명(광교점, 강남본점 등) 제거 후 브랜드명만 사용
2글자 미만 제외 봄, 강 같은 일반 명사와의 오탐 방지
공백 유연 매칭 키유스테이크 / 키유 스테이크 모두 처리
마스킹 토큰 [STORE] 대신 식당 사용 (KcBERT 학습 어휘 활용)
테스트 마스킹 테스트 데이터는 name 컬럼 기준으로 별도 브랜드명 세트 구성

3-3. 전처리 파이프라인 (순서 중요)

순서 단계 설명
① HTML 엔티티 디코딩 &amp; → &, &lt; → <
② HTML 태그 제거 <b>텍스트</b> → 텍스트
③ 가게명 → 식당 치환 특수문자 정리 전에 반드시 먼저 실행
④ 해시태그 # 제거 #맛집추천 → 맛집추천 (단어 보존)
⑤ 말줄임 제거 ... / … → 공백
⑥ 과도한 공백 정리 연속 공백 → 단일 공백, 앞뒤 제거

※ 특수문자 정리 전에 가게명 마스킹을 먼저 실행해야 함. 정규화 과정에서 가게명이 변형되면 매칭 실패.

3-4. 레이블 설계 전략: 의도적 보수적 분류

광고 리뷰를 놓치지 않는 것(Recall 극대화)을 최우선 목표로 설정.

리뷰 유형 실제 레이블 분류 결과 설계 의도
매장정보 나열형 (영업시간, 전화번호 등) 비광고 광고로 분류 광고성 패턴과 유사하므로 누락 방지 우선
해시태그 나열형 비광고 광고로 분류 광고성 패턴과 유사하므로 누락 방지 우선

Precision 일부를 희생하되, 실제 광고 리뷰를 놓치지 않는 Recall을 확보하는 트레이드오프를 의도적으로 선택.

09. OpenVINO 변환 가이드

9.1. 지원 포맷

9.2. 변환 스크립트

from optimum.intel import OVModelForSequenceClassification
import os

model_dir  = os.path.abspath(".")
output_dir = os.path.abspath("../exp_9_openvino")

model = OVModelForSequenceClassification.from_pretrained(
    model_dir,
    export=True,
    local_files_only=True    # HuggingFace Hub 접근 차단
)

os.makedirs(output_dir, exist_ok=True)
model.save_pretrained(output_dir)
print(f"변환 완료! → {output_dir}")

오류 원인 해결
