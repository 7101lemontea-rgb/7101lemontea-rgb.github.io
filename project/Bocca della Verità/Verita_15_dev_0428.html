Overview

01. 작업 개요 및 점검

네이버 지도 블로그 리뷰의 광고 여부를 판별하는 AI 엔진을 고도화하고, 분산되어 있던 분석 화면을 통합하여 단일 상세 페이지 내에서 데이터 흐름이 완결되도록 설계했습니다[cite: 3].

작업 내용 상태 비고
AI 분석 모드 전환 (ELECTRA/GPT) ✅ 완료 런타임 동적 전환 가능[cite: 3]
ELECTRA → OpenVINO 엔진 변환 ✅ 완료 CPU 추론 성능 최적화[cite: 3]
화면 구조 통합 (BlogListScreen 제거) ✅ 완료 RestaurantDetailScreen 통합[cite: 3]
리뷰 신뢰도 등급화 UI ✅ 완료 3단계 (상/중/하) 등급 적용[cite: 3]
AI & Optimization

02. AI 분석 및 최적화

2.1. AI 분석 모드 전환 기능 구현

기존 하드코딩된 모델 구조에서 탈피하여, analysisModeProvider를 통해 전역 상태에서 ELECTRA 모델 또는 GPT(LLM)를 선택할 수 있도록 구현했습니다[cite: 3]. 선택된 모드는 API 요청 시 쿼리 파라미터로 전달되어 Supabase의 각 예측 컬럼에 동적으로 저장됩니다[cite: 3].

2.2. ELECTRA → OpenVINO 변환 적용

PyTorch 기반 모델을 OpenVINO(.xml/.bin) 구조로 전환하여 추론 시 Torch 의존성을 제거하고 서버 경량화를 달성했습니다[cite: 3]. OVModelForSequenceClassification을 적용하여 저사양 환경에서도 빠른 응답 속도를 보장합니다[cite: 3].

2.3. LLM 리뷰 분석 기능 변경 (float 기반)

기존 이진 분류(0/1)에서 0.0~1.0 사이의 확률값(float) 출력 방식으로 프롬프트를 변경했습니다[cite: 3]. 이를 통해 ELECTRA 모델과 동일한 척도로 신뢰 등급을 산출할 수 있게 되었습니다[cite: 3].

Interface & UX

03. 화면 구조 통합 및 시각화

3.1. 화면 구조 통합 및 통합 UI 구축

기존의 BlogListScreen을 제거하고 모든 정보를 RestaurantDetailScreen 내 섹션으로 통합했습니다[cite: 3]. 데이터 존재 여부에 따라 로딩 스피너, NoDataCard, 또는 AI 분석 결과 카드가 동적으로 표시됩니다[cite: 3].

3.2. 워드클라우드 기능 구현

리뷰 제목에서 키워드를 추출하여 시각화하는 워드클라우드를 CustomPainter 기반으로 직접 구현했습니다[cite: 3]. 외부 패키지 없이 Flutter Web과 완벽히 호환되며, 고정 시드(Random 42)를 적용해 UI 일관성을 유지합니다[cite: 3].

Trust System

04. 신뢰 등급 및 이미지 시스템

4.1. 리뷰 신뢰도 등급화 및 UI 적용

광고 확률 점수에 따라 '상/중/하' 3단계 등급을 부여하고, 각 카드에 배경색 및 뱃지 UI를 적용했습니다[cite: 3].

  • 상 (진성): 확률 0.3 미만. 초록 테두리 및 뱃지 적용[cite: 3].
  • 중 (의심): 확률 0.3 ~ 0.6 미만. 노란 테두리 및 뱃지 적용[cite: 3].
  • 하 (광고): 확률 0.6 이상. 빨간 테두리 및 뱃지 적용[cite: 3].

4.2. 음식점 카테고리 이미지 시스템 구축

카카오 API의 1차 카테고리 분류(한식, 일식 등)를 기반으로 로컬 에셋 이미지를 매핑하는 시스템을 구축했습니다[cite: 3]. 실제 사진이 없는 경우에도 시각적 심미성을 제공합니다[cite: 3].

Pending

05. 미해결 및 추후 검토 사항

  • 이미지 연동: 현재 카테고리 기본 이미지에서 실제 네이버/카카오 업체 사진으로 교체 검토[cite: 3].
  • 캐시 고도화: 이미 분석된 데이터에 대해 분석 모드 변경 시 재판별을 수행하는 로직 구현 필요[cite: 3].
