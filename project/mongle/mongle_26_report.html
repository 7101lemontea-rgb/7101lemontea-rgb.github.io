01 — Overview

프로젝트 개요

Mongle은 "막연함, 번거로움, 불투명함, 복잡함을 해결하는 스마트한 웨딩 어시스턴트"입니다. 기존 웨딩 시장의 불투명한 가격 구조와 파편화된 정보로 인해 겪는 예비 부부들의 어려움을 AI 기술로 해소하고자 합니다.

02 — Problem

문제 인식 및 추진 배경

우리는 현재의 웨딩 시장을 "가격표 없는 메뉴판만 있는 레스토랑"에 비유했습니다. 소비자는 가격을 모른 채 계산대로 가야만 비용을 알 수 있고, 100개가 넘는 메뉴판 속에서 원하는 정보를 찾기 어렵습니다.

실제 시장의 6대 문제점

한국소비자원(2024) 및 기획재정부(2024) 조사 자료 기반

문제 01
웨딩홀 가격 비공개
85%
소비자가 가격 정보 부족을 경험. 방문 상담 시에만 가격 공개.
문제 02
스드메 불투명성
83%
예상치 못한 추가 비용 발생 경험.
문제 03
직접 상담의 번거로움
90%+
가격 확인을 위해 일일이 대면 상담 진행 필요.
문제 04
사람마다 다른 가격
2–3×
동일 서비스임에도 최대 2~3배 가격 차이 발생.
문제 05
검색의 번거로움
75%
정보 탐색을 위해 여러 플랫폼을 전전하며 피로감.
문제 06
플래너 중심의 제한적 추천
60%+
제휴 업체 중심의 추천만 받음.

이러한 문제들은 '정보 비대칭'에 의한 불안한 결정과 과도한 선택지로 인한 결정 피로(Decision Fatigue)를 유발합니다.

03 — Solution

해결책: Mongle 서비스 소개

Mongle은 AI 기술을 활용하여 예산 최적화, 맞춤형 추천, 투명한 견적 비교를 제공하는 통합 웨딩 플랫폼입니다.

경쟁 서비스 대비 차별점

기능 Mongle 메링 (경쟁사 A) 웨딩플로우 (경쟁사 B)
AI 추천 완벽 지원 (개인화 심화) 부분 지원 부분 지원
견적 분석 심층 분석 기본 없음
예산 최적화 AI 기반 최적화 기본 없음
가격 투명성 높음 중간 낮음
04 — Core Features

핵심 기능 및 사용자 흐름

웨딩 준비에 필수적인 5가지 기능을 유기적으로 연결하여 제공합니다.

01
AI 추천 시스템
사용자의 취향 프로필을 분석하여 개인화된 웨딩 업체를 스마트하게 매칭합니다.
02
견적 비교
실시간 견적 분석을 통해 투명한 가격 비교가 가능하며, 숨겨진 비용을 최소화합니다.
03
예산 관리
전체 예산 내에서 스마트하게 비용을 배분하고 자동으로 지출을 추적합니다.
04
일정 관리
웨딩 준비 단계별 필수 과업에 대한 자동 리마인더와 일정 조율 기능을 제공합니다.
05
채팅 / 상담
AI 챗봇을 통한 즉각적인 1차 상담과 업체와의 실시간 상담 기능을 지원합니다.

07 — Architecture & Stack

시스템 아키텍처 및 기술 스택

안정적이고 확장 가능한 4-Layer Architecture를 채택하였습니다.

시스템 구조

App / Web Layer Client
React Native(Expo Router) 및 Vite 기반의 클라이언트.
API Layer FastAPI
FastAPI(Python)를 사용하여 REST API 통신.
Database Layer Supabase
Supabase(PostgreSQL)를 활용한 관계형 데이터 관리 및 인증(Auth).
AI Layer OpenAI
OpenAI LLM을 활용한 추천 및 챗봇 엔진.

데이터베이스 설계 (ERD 개요)

Users(사용자) ←→ Vendors(업체) ←→ Quotes(견적) ↓ ↓ Bookings(예약) Chats(채팅) Tasks(할일) ↓ ↓ ↓ Budgets(예산) Planners(플래너)

기술 스택 상세

분야 사용 기술
Frontend React Native, Expo Router, Vite, JavaScript, CSS
Backend FastAPI, Python, Supabase (PostgreSQL)
AI / API OpenAI (GPT-4, GPT-4-mini), KakaoMap API
Tools VS Code, Figma, Git / GitHub, Notion, Discord

AI 엔진 및 챗봇

사용자 프로필, 예산, 일정을 입력받아 AI 엔진이 분석(추천·최적화) 과정을 거쳐 최적 업체 매칭 및 15% 비용 절감 시나리오를 도출합니다.

  • 챗봇은 RAG(검색 증강 생성) 기술을 활용하여 벡터화된 데이터베이스에서 정확한 정보를 찾아 답변합니다.
  • AWS Route53, AWS SES, Supabase SMTP 설정을 연동하여 안정적인 인증 메일 발송 기능을 구현했습니다.

결론

Mongle 프로젝트는 웨딩 시장의 불투명성을 AI 기술과 투명한 데이터로 해결하고자 했습니다. 단기간의 개발 일정 속에서도 핵심적인 매칭, 견적 비교, 예산 관리 기능을 성공적으로 구현하였으며, 법적 규제 변화에 대응하는 비즈니스 모델까지 확보하였습니다. "복잡한 웨딩 준비를, 더 투명하고 더 스마트하게" 만들겠다는 비전을 가지고, 지속 가능한 웨딩 플랫폼으로 발전해 나갈 것입니다.
