Issue 01 — npm

npm install 의존성 충돌

이슈 ID
TS-01
영향 범위
팀원 전체 초기 환경설정 시 발생
해결 여부
✅ 해결 완료

에러 메시지

npm error ERESOLVE could not resolve
npm error Conflicting peer dependency: react@19.2.4

원인

react@19.1.0과 react-dom@19.2.4의 peer dependency 버전이 맞지 않아 발생합니다. Expo 프로젝트에서 흔히 발생하는 문제입니다.

해결 방법

# 의존성 충돌 무시하고 설치
npm install --legacy-peer-deps

⚠️ 팀 프로젝트 주의사항  --legacy-peer-deps 옵션 사용 시 package-lock.json이 변경됩니다. 팀원 전체가 동일한 옵션을 사용하도록 프로젝트 루트에 .npmrc 파일을 생성하세요. 

# .npmrc 파일 생성 — 프로젝트 루트에서 실행
echo "legacy-peer-deps=true" > .npmrc
Issue 02 — Expo Router

Route missing default export

이슈 ID
TS-02
영향 범위
app/ 폴더 내 미구현 .jsx 파일 전체
해결 여부
✅ 해결 완료

에러 메시지

WARN Route "./(couple)/chat.jsx" is missing the required default export.
WARN Ensure a React component is exported as default.

원인

Expo Router는 app/ 폴더 안의 모든 .jsx 파일을 라우트로 인식합니다. 파일이 존재하지만 export default가 없거나 파일이 비어 있으면 이 경고가 발생합니다.

해결 방법

아직 구현하지 않은 파일에는 임시 컴포넌트를 넣어주세요.

// 임시 컴포넌트 — 나중에 교체
import { View, Text } from "react-native";

export default function ChatScreen() {
  return (
      <View style={{ flex: 1, justifyContent: "center", alignItems: "center" }}>
        <Text>채팅</Text>
      </View>
  );
}

임시 컴포넌트 추가 대상 파일 목록

  • app/(auth)/_layout.jsx
  • app/(auth)/login.jsx
  • app/(auth)/register.jsx
  • app/(couple)/budget.jsx
  • app/(couple)/chat.jsx
  • app/(couple)/docs.jsx
  • app/(couple)/timeline.jsx
  • app/(couple)/vendors.jsx
  • app/(planner)/_layout.jsx
  • app/(planner)/dashboard.jsx
  • app/(planner)/customer/[id].jsx
  • app/planners/index.jsx
  • app/planners/[id].jsx
  • app/vendors/index.jsx
  • app/vendors/[id].jsx
Issue 03 — Expo Router

Root Layout 마운트 전 navigate 에러

이슈 ID
TS-03
영향 파일
app/_layout.jsx
해결 여부
✅ 해결 완료

에러 메시지

ERROR [Error: Attempted to navigate before mounting the Root Layout component.]
Ensure the Root Layout component is rendering a Slot, or other navigator on the first render.

원인

app/_layout.jsx의 useEffect 안에서 router.replace()를 호출하면, Root Layout이 완전히 마운트되기 전에 navigate를 시도해서 에러가 발생합니다.

❌ 잘못된 방법 
useEffect(() => {
  router.replace("/(couple)");
}, []); // 마운트 전 호출로 에러 발생
✅ 올바른 방법

useEffect와 router.replace 없이 <Slot />만 렌더링합니다. Expo Router가 자동으로 첫 화면을 잡아줍니다.

해결 코드

// app/_layout.jsx — 올바른 방법
import { Slot } from "expo-router";
import { StatusBar } from "expo-status-bar";
import { SafeAreaProvider } from "react-native-safe-area-context";

export default function RootLayout() {
  return (
      <SafeAreaProvider>
        <StatusBar style="dark" />
        <Slot initialRouteName="(couple)" />
      </SafeAreaProvider>
  );
}
Issue 04 — Expo Router

Property 'Redirect' doesn't exist

이슈 ID
TS-04
영향 버전
expo-router v3 이상
해결 여부
✅ 해결 완료

에러 메시지

ERROR [ReferenceError: Property 'Redirect' doesn't exist]

원인

expo-router v3 이상에서는 <Redirect /> 컴포넌트가 제거되었거나 다르게 동작합니다. import해도 존재하지 않아 에러가 발생합니다.

해결 방법

Redirect 대신 initialRouteName prop을 사용하세요. 

// Redirect 대신 initialRouteName 사용
<Slot initialRouteName="(couple)" />
Issue 05 — Expo Router

Too many screens defined

이슈 ID
TS-05
영향 파일
app/(couple)/_layout.jsx
해결 여부
✅ 해결 완료

에러 메시지

WARN [Layout children]: Too many screens defined. Route "timeline" is extraneous.

원인

(couple)/_layout.jsx의 Tabs.Screen에 실제 파일이 없는 name을 정의하거나, 존재하지 않는 화면(mypage 등)을 탭에 등록할 때 발생합니다.

해결 방법

탭에 표시하지 않을 화면은 href: null로 숨기고, 실제 파일이 없는 name은 빈 파일이라도 생성해야 합니다. 

// 탭에 표시하지 않을 화면 숨기기
<Tabs.Screen name="budget" options={{ href: null }} />
Issue 06 — Expo Router

화면에 "layout" 글씨만 보임

이슈 ID
TS-06
영향 파일
app/(auth)/_layout.jsx 포함 모든 _layout.jsx
해결 여부
✅ 해결 완료

증상

앱을 실행하면 메인 화면 대신 "layout" 또는 "_layout" 텍스트만 화면에 표시됩니다.

원인

_layout.jsx 파일 중 하나가 올바른 네비게이터 대신 텍스트를 렌더링하고 있는 경우입니다. 특히 (auth)/_layout.jsx가 임시 컴포넌트 상태로 남아있을 때 발생합니다.

❌ 문제가 된 코드 (auth)/_layout.jsx 
const _layout = () => {
  return (<View><Text>_layout</Text></View>);
};
✅ 올바른 코드 (auth)/_layout.jsx

Stack 네비게이터를 export default로 반환해야 합니다. 

import { Stack } from "expo-router";

export default function AuthLayout() {
  return <Stack screenOptions={{ headerShown: false }} />;
}

빠른 탐색 팁: Ctrl+Shift+F로 전체 검색에서 "_layout" 텍스트를 검색하면 문제 파일을 빠르게 찾을 수 있습니다. _layout.jsx는 반드시 <Stack>, <Tabs>, <Drawer> 중 하나의 네비게이터를 반환해야 합니다.

Issue 07 — React Native

SafeAreaView deprecated 경고

이슈 ID
TS-07
영향 범위
SafeAreaView를 사용하는 모든 화면 파일
해결 여부
✅ 해결 완료

에러 메시지

WARN SafeAreaView has been deprecated and will be removed in a future release.
Please use "react-native-safe-area-context" instead.

원인

react-native 기본 SafeAreaView는 deprecated 되었습니다. react-native-safe-area-context의 SafeAreaView를 사용해야 합니다.

해결 방법

1단계: app/_layout.jsx에 SafeAreaProvider 추가 

import { SafeAreaProvider } from "react-native-safe-area-context";

// <SafeAreaProvider>로 <Slot /> 감싸기
export default function RootLayout() {
  return (
      <SafeAreaProvider>
        <Slot />
      </SafeAreaProvider>
  );
}

2단계: 각 화면 파일에서 import 변경

변경 전 
import { SafeAreaView } from "react-native"
변경 후 
import { SafeAreaView } from "react-native-safe-area-context"

별도 설치 없이 바로 사용 가능합니다. Expo 기본 포함 패키지입니다.

Issue 08 — Android

하단 탭바가 기기 네비게이션 바와 겹침

이슈 ID
TS-08
영향 범위
Android 기기 전체 (제스처 네비게이션 사용 기기)
영향 파일
app/(couple)/_layout.jsx
해결 여부
✅ 해결 완료

증상

Android 기기에서 앱의 하단 탭바가 기기의 제스처 네비게이션 바(홈, 뒤로가기 등)와 겹쳐 보입니다.

원인

Android 제스처 네비게이션 바 높이(24~48px)를 고려하지 않은 tabBarStyle 설정 때문입니다.

해결 방법

Platform을 사용해 Android/iOS별로 높이를 다르게 설정합니다. 

import { Platform } from "react-native";

tabBarStyle: {
  backgroundColor: "#fff",
  borderTopColor: "#f0e8e4",
  borderTopWidth: 1,
  borderBottomColor: "#f0e8e4",
  borderBottomWidth: 1,
  height: Platform.OS === "android" ? 70 : 60,
  paddingBottom: Platform.OS === "android" ? 12 : 8,
  paddingTop: 8,
},

Platform 분기 이유: Android 기기의 제스처 네비게이션 바(24~48px)와 탭바가 겹치는 문제를 해결하기 위해 Platform.OS로 Android/iOS 높이를 다르게 설정합니다. Android는 height: 70, paddingBottom: 12 / iOS는 height: 60, paddingBottom: 8.
