Chapter 04 — UI/UX

UI/UX 이슈

#13   대시보드 '오늘 할 일'에 오늘 일정이 아닌 항목 표시

대상 파일: dashboard.jsx  ·  증상: 오늘 할 일 카드에 날짜 무관하게 전체 일정 앞 5개가 표시됨

원인 날짜 필터 없이 상위 5개 슬라이싱
원인 1
todos.slice(0, 5)로 날짜 상관없이 상위 5개를 표시.
원인 2
urgentTodos는 상단 뱃지 카운트에만 사용되고 카드 목록에 반영 안 됨.

해결 — 표시 순서 재정의

우선순위 조건 정렬
1순위 오늘 미완료 (urgent) scheduled_date === today
2순위 이후 미완료 (upcoming) 날짜 오름차순
최대 5개 표시 — 
const urgentTodos = todos.filter(
  t => !t.done && t.scheduled_date === today
);
const upcomingTodos = todos
  .filter(t => !t.done && t.scheduled_date > today)
  .sort((a, b) => a.scheduled_date.localeCompare(b.scheduled_date));
const dashboardTodos = [...urgentTodos, ...upcomingTodos].slice(0, 5);

#14   완료 지우기 후 빈 자리가 채워지지 않음

대상 파일: dashboard.jsx  ·  증상: 완료 지우기 버튼 클릭 후 빈 자리가 새 항목으로 채워지지 않음 (5개 → 3개)

원인 DB에서 새 항목을 다시 불러오는 로직 없음
원인
setTodos(prev => prev.filter(t => !t.done))으로 state에서만 제거. DB에서 새 항목을 다시 불러오는 로직 없음.
해결
완료 지우기 시 lockedTodoIds를 초기화하고 fetchTodos()를 재호출하여 DB에서 새 5개 로드. 
onPress={() => {
  setTodos(prev => prev.filter(t => !t.done));
  lockedTodoIds.current = []; // 고정 목록 초기화
  fetchTodos(); // DB에서 새 5개 로드
}}

알림#1   알림 버튼 클릭 시 아무 동작 없음

대상 파일: app/(couple)/(tabs)/index.jsx  ·  증상: 알림 아이콘 탭해도 반응 없음, 알림 dot이 항상 표시됨

원인 onPress 핸들러 없음 + dot 항상 렌더링
원인 1
알림 버튼에 onPress 핸들러 없음.
원인 2
notifDot이 조건 없이 항상 렌더링 (하드코딩).
원인 3
useNotifications 훅 미연결 → unreadCount 값 사용 불가.
수정 전 ❌ 
// onPress 없음, dot 항상 표시
<TouchableOpacity style={styles.notifBtn} activeOpacity={0.7}>
  <Ionicons name="notifications-outline" size={20} color="#3a2e2a" />
  <View style={styles.notifDot} />
</TouchableOpacity>
수정 후 ✅ 
// onPress 연결, dot 조건부 표시
<TouchableOpacity
  style={styles.notifBtn}
  activeOpacity={0.7}
  onPress={() => router.push('/(couple)/notifications')}
>
  <Ionicons name="notifications-outline" size={20} color="#3a2e2a" />
  {unreadCount > 0 && <View style={styles.notifDot} />}
</TouchableOpacity>

알림#2   알림 화면에서 데이터가 로드되지 않음

대상 파일: app/(couple)/notifications.jsx  ·  증상: 알림 화면 진입 시 알림 목록이 비어 있음 (Supabase에 데이터 존재함에도)

원인 setUserId() 미호출 — fetchNotifications 실행 안 됨
원인
NotificationContext의 fetchNotifications()는 setUserId() 호출 시 실행. (couple) 화면 어디에서도 setUserId()를 호출하지 않음. Context 초기 userId가 null → fetch 자체가 실행되지 않음. 
// index.jsx에서 setUserId 호출
const auth = useAuth();
const coupleUserId = auth?.couple_id ?? auth?.user_id ?? null;
useEffect(() => {
  if (coupleUserId) {
    setUserId(coupleUserId);
  }
}, [coupleUserId]);

주의: auth 객체의 키 이름은 AuthContext 구현에 따라 다를 수 있음 (couple_id vs user_id)

알림#3   notifications.jsx 이중 관리 구조 문제

대상 파일: app/(couple)/notifications.jsx  ·  증상: NotificationContext와 별개로 자체 fetch 수행 → 읽음 상태 동기화 문제

원인 Context 도입 이전 코드 잔존
원인
기존 notifications.jsx가 useAuthStore + 직접 fetch 방식으로 구현. NotificationContext 도입 이전 코드가 그대로 남아 있었음.
해결
NotificationContext 방식으로 전면 교체.
항목 기존 변경 후
데이터 관리 useAuthStore + 직접 fetch useNotifications Context
UI 구조 FlatList 단순 카드 읽음/안읽음 섹션 분리
알림 탭 동작 즉시 딥링크 이동 모달 시트 → 바로 이동 버튼
모두 읽음 없음 헤더 우측 버튼
알림 삭제 없음 모달 내 삭제 버튼
딥링크 있음 (유지) 있음 (DEEPLINK_MAP으로 관리)

알림#4   notifications.jsx를 (tabs) 안에 배치 시 UI 깨짐

대상 파일: app/(couple)/notifications.jsx  ·  증상: 알림 화면 하단에 탭 바가 표시되고 탭 네비게이터가 알림을 탭 항목으로 인식

원인 (tabs) 폴더 안의 파일은 모두 탭 네비게이터 적용
원인
(tabs) 폴더 안의 파일은 모두 탭 네비게이터의 적용을 받음.
위치 결과
app/(couple)/notifications.jsx ✅ 탭 바 없음 — 올바른 위치
app/(couple)/(tabs)/notifications.jsx ❌ 탭 바 표시됨 — 잘못된 위치

알림#5   딥링크 기능 누락

대상 파일: app/(couple)/notifications.jsx  ·  증상: 파일 교체 과정에서 ref_type 기반 딥링크 로직 누락될 뻔함

해결 DEEPLINK_MAP으로 명시적 관리
파일 전면 교체 과정에서 누락될 뻔한 딥링크 로직을 DEEPLINK_MAP 객체로 명시적으로 관리. 새로운 ref_type 추가 시 DEEPLINK_MAP에 항목만 추가하면 됨. 
const DEEPLINK_MAP = {
  budget:   '/(couple)/(tabs)/budget',
  payment:  '/(couple)/(tabs)/budget',
  timeline: '/(couple)/(tabs)/timeline',
  schedule: '/(couple)/(tabs)/timeline',
  // 확장 예시
  chat:     '/(couple)/(tabs)/chat',
  docs:     '/(couple)/(tabs)/docs',
};
// 이동 흐름: "바로 이동" 버튼 탭 → closeDetail() → setTimeout 230ms → router.push(dest)
Chapter 06 — Pre-Integration Checklist

연동 전 체크리스트

DB 설정 확인

항목 확인 방법
couples.planner_id에 실제 플래너 ID가 입력되어 있는가? Supabase 대시보드 → couples 테이블
user_profiles에 role, planner_id, couple_id가 모두 세팅되어 있는가? Supabase 대시보드 → user_profiles 테이블
쿼리에 사용하는 컬럼명이 실제 DB 컬럼명과 일치하는가? CSV 또는 Supabase 대시보드에서 직접 확인
Realtime 필요 테이블에 활성화되어 있는가? Database → Replication → 해당 테이블 토글 ON

코드 패턴 확인

패턴 01 planner_id 하드코딩 여부
❌ 하드코딩 금지 — useAuth()에서 동적 조회. useEffect 내 if (!planner_id) return; early return 필수.
패턴 02 Realtime 채널명 고유성
{화면}-{테이블}-${planner_id} 패턴 — 중복 시 이벤트 누락 발생.
패턴 03 외부 컴포넌트 내부 함수 참조
❌ 직접 참조 금지 — prop으로 전달. (#10 coupleName 사례 참고)
패턴 04 Falsy fallback 처리
?? 아닌 || 사용 — 빈 문자열('')도 처리해야 하는 경우. (#12 STAGE_STYLE 사례 참고)
패턴 05 DB에 없는 컬럼 참조
❌ avatarIdx 등 하드코딩 시절 컬럼 사용 금지 — sorted.indexOf(c) 방식으로 대체.

테이블 혼동 주의사항

테이블 올바른 용도 사용 화면
planner_vendors 협력업체 파트너 목록 wedding_vendor_partners.jsx
couple_vendor_costs 커플별 업체 지출 planner_budget.jsx
couple_payments due_date 컬럼 사용 (payment_date ❌) planner_budget.jsx

⚠️ couples.stage DB 저장값: '준비중' / '최종점검' / '초기상담' / '계약완료' — 띄어쓰기 없음. 코드의 STAGE_FILTERS·STAGE_OPTIONS·STAGE_STYLE 키와 반드시 일치시켜야 함.
