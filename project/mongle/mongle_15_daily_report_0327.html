오늘의 회고

팀 협업 및 기술적 시행착오

GitHub Merge 과정에서 이전 구현 내용이 덮어쓰여 복구 작업이 발생했습니다. 형상 관리의 중요성과 브랜치 전략의 필요성을 다시 체감한 하루였습니다.

또한 실시간 작업 공유 부족으로 동일 기능을 중복 구현하는 상황이 발생했고, 이후 작업 단위 분리 및 공유 체계를 개선하기로 했습니다.

개인적 소감

다양한 기술적 오류와 협업 이슈가 있었지만 Supabase 인프라를 안정적으로 구축했습니다. 클라이언트 검증과 DB 제약 조건 정합성, 그리고 팀원 간 Sync의 중요성을 깊게 이해한 하루였습니다.
