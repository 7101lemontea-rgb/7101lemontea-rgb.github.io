1장 — Introduction

1. 문서 개요

1.1 목적

본 문서는 웨딩 그룹웨어 서비스 "Mongle" 의 MVP 개발을 위한 소프트웨어 요구사항을 정의합니다. v1.2에서는 MVP 범위를 Must / Should / Could 세 등급으로 재분류합니다.

1.2 우선순위 등급 정의

등급 의미
Must MVP에 반드시 포함되어야 하는 핵심 기능. 미구현 시 서비스 불가
Should 완성도를 높이는 중요 기능. 일정 여건에 따라 MVP에 포함
Could 있으면 좋지만 우선순위가 낮은 기능. 여유 시 구현

1.3 AI 기능 컨셉

Budget AI 예산 견적 AI

플래너가 입력한 비용 명세서를 지역 시세와 비교·분석하여 이상 항목을 감지하고 커플에게 투명하게 안내

Vendor AI 업체 추천 AI

커플의 예산·스타일·지역 조건을 분석해 최적의 웨딩 업체를 자동 추천하고 추천 이유를 자연어로 제공

2장 — Requirement Definitions

2. 요구사항 정의

2.1 일정 관리 — Must

  • 결혼 준비 일정 카테고리 정의 (드레스 투어, 스튜디오 촬영 등)
  • 월별 캘린더 화면
  • 타임라인 화면 (D-day 기준 진행 단계)
  • 일정 추가·수정·삭제
  • 플래너가 일정을 수정하면 커플 화면에도 실시간 반영
  • 일정 데이터를 예산·알림 기능과 연결

2.2 예산 관리 — Must

  • 예산 카테고리 정의 (스·드·메, 웨딩홀, 예물 등)
  • 항목별 총액·계약금·잔금·결제일 입력 화면
  • 총예산 대비 사용 금액 진행 바 (Progress Bar)
  • 예산 초과 시 경고 표시
  • 금액 입력 시 자동 콤마(,) 포맷
  • 잔금 결제일이 다가오면 알림 기능과 연결

2.3 AI 챗봇 — Must

  • 웨딩 특화 AI 답변 범위 정의 (예약 시기, 업체 추천 등)
  • AI 챗봇 채팅 화면 (하단 플로팅 버튼)
  • LLM API 연결 및 실시간 스트리밍 답변
  • 웨딩 데이터 기반 RAG (검색 증강 생성) 구축
  • 챗봇이 추천한 업체 카드 클릭 시 업체 상세 페이지로 이동

2.4 업체 정보 — Must

  • 업체 카테고리 확정 (스튜디오, 드레스, 메이크업, 웨딩홀)
  • 업체 초기 데이터 수집 및 DB 적재 (크롤링 또는 제휴)
  • 카테고리별 업체 목록 화면
  • 예산·지역·스타일 필터
  • 업체 상세 페이지 (사진, 정보, 후기)
  • 후기 정렬 기능 (최신순 / 별점순)

2.5 사용자 계정 및 연결 — Should

  • 이메일 & 비밀번호 회원가입·로그인
  • 가입 시 역할 선택 (커플 / 플래너)
  • 초대 코드 생성
  • 초대 코드 입력으로 플래너 ↔ 커플 계정 연결
  • 계정 연결 해제

2.6 플래너 기능 — Should

  • 플래너 대시보드
  • 플래너 프로필 페이지 (경력, 스타일 태그, 포트폴리오)
  • 플래너 목록 화면 (2단 그리드)
  • 스타일 태그로 플래너 필터링
  • 상담 신청 기능 (날짜·시간·예산 입력)
  • 상담 신청 시 플래너에게 알림 발송
  • 실제 이용 커플의 평점만 반영되도록 처리

2.7 AI 업체 추천 — Should

  • 예산·스타일·지역 기반 추천 로직
  • 업체 목록에 AI 추천 배지 표시
  • 예산 변경 시 추천 목록 자동 업데이트
  • 챗봇 대화 내용 기반 추천 반영

2.8 알림 — Should

  • 일정·잔금 결제일 D-7 / D-3 / D-1 자동 알림
  • 알림 센터 화면 (읽음 / 안 읽음)
  • 알림 탭 시 해당 일정·예산 화면으로 바로 이동
  • 잔금 완료 체크 시 예정된 알림 자동 취소
  • 푸시 알림 꺼져 있을 때 앱 내 배너로 대체 표시

2.9 기타 — Could

  • 업체 2~3개 견적 항목별 비교 화면
  • 내 예산 명세서를 PDF로 저장
  • PDF 공유 기능
3장 — Terminology

3. 용어 정의

용어 정의
시세 DB 지역별·카테고리별 웨딩 업체 평균 가격 데이터베이스. 크롤링 및 수동 큐레이션으로 구축
규칙 기반 엔진 머신러닝 없이 if-else 조건과 통계로 이상을 감지하는 분석 로직
LLM (GPT API) GPT API. 자연어 해설·추천 이유 생성에 사용
RAG Retrieval-Augmented Generation. 웨딩 데이터를 검색해 LLM 응답 정확도를 높이는 기법
콘텐츠 기반 필터링 업체 속성(태그·가격·지역)과 커플 조건을 매칭하는 추천 알고리즘
스·드·메 스튜디오(촬영)·드레스·메이크업의 약칭. 웨딩 준비의 핵심 3종
D-day 결혼 예정일. 일정 및 알림 기준 날짜로 사용
초대 코드 플래너가 생성하고 커플이 입력하여 양 계정을 연결하는 고유 코드
