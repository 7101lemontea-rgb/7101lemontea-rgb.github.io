01 — Document Overview

문서 개요

1.1 목적

본 문서는 웨딩 그룹웨어 서비스 "Fleur"의 MVP 개발을 위한 소프트웨어 요구사항을 정의합니다. v1.1에서는 핵심 AI 기능인 예산 견적 AI와 업체 추천 AI의 요구사항이 추가되었습니다.

1.2 v1.1 추가 범위

신규 추가: 예산 견적 AI (FR-048~054) + 업체 추천 AI (FR-055~062) — 총 15개 AI 요구사항 추가

기존 v1.0의 모든 요구사항(FR-001~047, NFR-001~022)은 변경 없이 유지됩니다. 본 섹션에서는 추가된 AI 기능 요구사항만 기술합니다.

1.3 AI 기능 컨셉

Budget AI
예산 견적 AI

플래너가 입력한 비용 명세서를 지역 시세와 비교·분석하여 이상 항목을 감지하고 커플에게 투명하게 안내

Vendor AI
업체 추천 AI

커플의 예산·스타일·지역 조건을 분석해 최적의 웨딩 업체를 자동 추천하고 추천 이유를 자연어로 제공

03 — AI Functional Requirements

AI 기능 요구사항

기존 3.1~3.7 섹션(FR-001~047)에 이어 AI 전용 기능 요구사항을 추가합니다.

3.8 예산 견적 AI (Budget AI)

핵심 차별화: 기존 4개 솔루션 모두 "비용 불투명"이 공통 한계였습니다. 예산 견적 AI는 이를 정면으로 해결하는 Fleur의 가장 강력한 무기입니다.

3.8.1 기술 구성 개요

시세 참조 DB 자체 DB + 크롤러

서울/경기 기준 스드메·홀 카테고리별 25th~75th 가격 분위 저장. 월 1회 업데이트

이상 감지 엔진 규칙 기반 통계 분석

항목별 금액을 시세 분위와 비교. 75th 초과 시 경고 플래그. 계약외 옵션 자동 감지

자연어 해설 생성 Claude API (LLM)

감지 결과를 사람이 읽기 쉬운 한국어 문장으로 변환. RAG 방식으로 시세 DB 참조

v2 고도화 RAG + Fine-tuning

실거래 데이터 학습으로 정밀도 향상. 6~12개월 데이터 축적 후 적용

3.9 업체 추천 AI (Vendor AI)

MVP는 규칙 기반 필터링 + Claude API 조합으로 즉시 구현합니다. v2에서 협업 필터링과 임베딩을 추가해 넷플릭스·유튜브 수준의 개인화 추천으로 고도화합니다.

3.9.1 기술 구성 개요

콘텐츠 기반 필터링 규칙 기반 매칭 (MVP)

업체 태그(스타일·지역·가격)와 커플 조건을 점수화해 정렬. DB 없이 즉시 구현 가능

추천 이유 생성 Claude API (LLM / MVP)

매칭된 업체에 대해 "왜 이 업체를 추천하는지" 자연어 문장 생성

협업 필터링 머신러닝 (v2)

유사 조건 커플의 선택 패턴 학습. 실거래 100건+ 필요

임베딩 유사도 텍스트 임베딩 + 벡터 DB (v2)

업체 후기·특징 벡터화 → 커플 취향과 코사인 유사도 계산. 후기 200건+ 필요

3.9.2 입력 조건 및 출력 정의

입력 — 커플 조건
총 예산 범위, 결혼 스타일(클래식/모던/내추럴/럭셔리), 선호 지역(시·구), 결혼 날짜, 게스트 수
입력 — 업체 데이터
업체명, 카테고리, 가격대, 위치, 스타일 태그, 평점, 리뷰 수, 플래너 연계 여부
출력 — 추천 결과
카테고리별 상위 3개 업체 + 매칭 점수(%) + 자연어 추천 이유
출력 — 매칭 점수 구성
스타일 매칭 40% + 예산 적합도 30% + 위치 편의성 20% + 후기 점수 10%

04 — AI Non-Functional Requirements

AI 기능 비기능 요구사항

기존 비기능 요구사항(NFR-001~022)에 AI 기능 관련 항목을 추가합니다.

우선순위 기준: P0: MVP 필수  |  P1: 2주차  |  P2: 3주차  |  P3: v2 이후

NFR-023 AI 응답 시간
요구 내용
Claude API 호출 후 스트리밍 첫 토큰까지 2초 이내. 전체 응답 완료 10초 이내.
예외 처리
초과 시 타임아웃 처리 및 "잠시 후 다시 시도해 주세요" 안내.
NFR-024 AI 분석 정확도
요구 내용
예산 견적 AI: 시세 DB 기준 이상 항목 감지 정밀도(Precision) 85% 이상 목표.
리스크
오탐(정상 항목을 이상으로 표시) 발생 시 커플 신뢰도 저하 리스크로 모니터링 필수.
NFR-025 AI 비용 관리
요구 내용
Claude API 호출 비용 월 30만원 이하로 관리. 동일 입력에 대한 응답은 1시간 캐시 적용.
제한
과도한 API 호출 방지를 위해 사용자당 일 50회 제한.
NFR-026 AI 결과 투명성
요구 내용
"이 결과는 AI가 분석한 참고 정보입니다. 최종 판단은 직접 확인하세요." 면책 문구를 모든 AI 출력 하단에 표시.
목적
AI 추천을 절대적 사실로 오인하지 않도록 안내.
NFR-027 시세 DB 데이터 품질
요구 내용
시세 DB 최신화 주기: 월 1회. 데이터 출처 명시(수집 날짜, 샘플 수). 이상치(outlier) 제거 후 분위 계산.
활성화 조건
카테고리·지역별 최소 샘플 수 30건 이상 확보 후 활성화.
NFR-028 AI 기능 폴백 처리
요구 내용
Claude API 장애 또는 타임아웃 시 AI 분석 없이 기본 비용 명세서만 표시. 추천 AI 장애 시 일반 업체 목록(평점순)으로 폴백.
원칙
서비스 전체 장애로 이어지지 않도록 격리.
05 — External Interfaces

AI 관련 외부 인터페이스
