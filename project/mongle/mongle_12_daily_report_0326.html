03 — Work Done

실제 수행한 작업

a. 중요 정보 카드

플래너가 로그인 후 첫 번째로 보는 대시보드 상단 영역을 구성했습니다. 플래너의 이름 · 직함과 오늘 처리해야 할 업무 수를 요약하는 플래너 정보 카드를 최상단에 배치했으며, 그 아래에는 현재 담당 커플의 수 및 각 커플의 결혼 예정일 · 준비율을 간략히 확인할 수 있는 담당 커플 현황 카드를 삽입했습니다. 또한 이번 달과 다음 달에 예정된 결혼식 건수와 날짜를 한눈에 볼 수 있는 월별 결혼 정보 카드도 구성했습니다.

b. Todo List

플래너가 처리해야 할 모든 할 일을 대시보드에서 바로 확인하고 관리할 수 있는 Todo List 섹션을 구현했습니다. 해당 날짜에 마감되는 할 일은 색상과 배지로 강조 처리하여 즉시 식별할 수 있도록 했으며, 각 항목 옆 체크박스를 탭하면 완료 처리가 되고 시각적으로 구분됩니다.

구현 포인트: 마감일은 YYYYMMDD 숫자 입력 후 자동으로 'YYYY년 M월 D일까지' 형식으로 변환. 오늘 마감 항목은 별도 섹션 상단에 강조 배치.

c. Couple List

담당 커플 목록을 결혼 D-DAY 임박 순으로 자동 정렬하여 표시하고, 각 커플 카드에 진행 준비율을 퍼센트와 프로그레스 바로 시각화했습니다. 대시보드에서는 임박한 커플을 최대 4개까지 표시하며, '전체 보기' 버튼을 누르면 커플 전체 목록 페이지(couple_list.jsx)로 이동합니다.

기능 상세
D-DAY 임박순 정렬 wedding_date 기준 오름차순 자동 정렬, 대시보드 최대 4개 표시
진행률 시각화 progress 값 기반 프로그레스 바 + 퍼센트 텍스트 동시 표시
커플 추가 모달 신랑 · 신부 이름 분리 입력, 년/월/일 분리 입력, 진행 단계 선택(최종점검/준비중/초기상담/계약완료)
상세 모달 커플 정보 · 준비율 · 일정 타임라인 표시, 정보 수정 및 삭제 기능 포함
Alert 삭제 확인 삭제 전 Alert 다이얼로그로 사용자 의도 재확인

d. 협력 업체 카드

플래너가 계약 · 협력하고 있는 업체들을 대시보드에서 카테고리별로 빠르게 확인할 수 있도록 협력 업체 카드를 구성했습니다. 대시보드에서는 스튜디오 · 드레스 · 메이크업 등 최대 3개 카테고리까지 요약 표시하며, 업체가 없는 카테고리는 자동으로 숨깁니다.

지원 업체 유형: 스튜디오 / 드레스 / 메이크업 / 웨딩홀 / 허니문 / 청첩장 / 영상·스냅 / 기타

e. 비용 현황 카드

플래너가 관리하는 전체 수금 및 지출 현황을 대시보드에서 한눈에 파악할 수 있도록 비용 현황 카드를 구성했습니다. 총 수금 금액, 미수금 금액, 미지급 업체 건수 및 금액을 KPI 형태로 상단에 표시했습니다.

구성 요소 설명
KPI 카드 총 수금액 / 미수금액 / 미지급 업체 금액을 대시보드 상단에 표시
수금 현황 탭 커플별 카드, 탭 시 계약금 · 중도금 · 잔금 납부 일정 아코디언 펼침
업체 지출 탭 업체별 그룹화, 탭 시 연결된 커플별 지출 금액 아코디언
통합 추가 모달 수금 항목 탭 / 업체 지출 탭 선택 후 금액 · 기한 입력
금액 단위 처리 UI 입력은 만원 단위, 내부적으로 원 단위로 변환하여 저장

f. 알림 화면 UI 구성 및 전역 상태 관리

알림 관련 기능 전체를 독립 화면(notifications.jsx)으로 분리하고, Context API를 통해 앱 전역에서 동일한 알림 데이터를 사용할 수 있도록 구조를 개선했습니다.

개발 경과

1차 구현 인라인 모달 방식

대시보드 헤더 알림 버튼 탭 시 상단에서 슬라이드 다운되는 모달 시트 표시. React Native 기본 animationType은 하단→상단만 지원하므로 Animated.spring()을 직접 사용하여 상단→하단 슬라이드 구현. overflow: 'hidden' 적용으로 라운드 모서리 밖 텍스트 삐져나옴 문제 해결.

2차 구현 상세 모달 추가

알림 항목 탭 시 하단 슬라이드 업 상세 모달 추가. 항목 탭 시 읽음 자동 처리, 상세 화면 내 삭제 버튼, 오버레이/X 버튼으로 닫기 기능 구현.

3차 구현 전용 화면 분리 (현재)

모든 알림 코드를 notifications.jsx로 이관. 버튼 탭 시 router.push('/(planner)/notifications')로 전용 화면 이동. 이로써 dashboard.jsx에서 Modal · Animated · 알림 관련 state/handler/style 전부 제거.

Context API 전역 상태 구조

앱의 홈 · 일정 · 대시보드 등 여러 화면에 알림 버튼이 존재하므로, 모든 화면이 동일한 알림 데이터를 참조해야 합니다. _layout.jsx에서 NotificationProvider로 전체 화면을 감싸면 Provider 안의 어느 화면에서든 useNotifications() 훅으로 알림 데이터를 바로 꺼내 쓸 수 있습니다.

화면 Context 사용 방식
dashboard.jsx unreadCount 읽어 헤더 알림 아이콘에 빨간 점 뱃지 표시
home.jsx unreadCount 읽어 홈 상단 알림 버튼 뱃지 표시
schedule.jsx unreadCount 읽어 일정 화면 알림 버튼 뱃지 표시
notifications.jsx 알림 목록 전체 읽기, 읽음 처리, 삭제 기능 실행

g. 플래너 상세 화면 — 상담 신청 모달

플래너 상세 화면(/(couple)/planners/[id].jsx) 하단의 '상담 신청하기' CTA 버튼을 탭하면 하단에서 슬라이드 업되는 Sheet 형태의 Modal이 열립니다. 사용자는 희망 날짜(년 · 월 · 일 분리 입력), 희망 시간(오전/오후 토글 + 시간 숫자 입력), 예산(만원 단위)을 입력한 뒤 전송하면 해당 플래너에게 알림이 발송됩니다.

항목 상세 구현
날짜 입력 년(4자리) / 월(2자리) / 일(2자리) TextInput을 가로 배치. 전송 시 padStart(2,'0')로 포맷 통일 후 YYYY-MM-DD 조합
시간 입력 오전/오후 토글 버튼 + 시간 숫자 입력. '오전 2시' 형식의 문자열로 조합하여 전송
예산 입력 만원 단위 숫자 입력. numeric 키보드 사용
유효성 검사 년/월/일/시간/예산 중 하나라도 비어 있으면 Alert 표시 후 전송 중단
전송 처리 submitting 상태 중 버튼 비활성화(opacity 0.6), 성공 시 Alert + 모달 닫기 + 입력값 초기화

06 — Retrospective

느낀점 및 회고

플래너 대시보드 화면은 원래 웹에 더 어울리는 정보 밀도 높은 UI입니다. 모바일 앱이라는 제약 안에서 가독성을 유지하면서도 핵심 정보를 빠르게 파악할 수 있도록 카드 레이아웃을 설계하는 것이 가장 큰 도전이었습니다. 우선 앱에서는 '정보를 확인하는 용도'에 집중해 간결하게 구성하는 것을 목표로 했고, 상세 기능은 별도 페이지로 분리하는 방식으로 해결했습니다.
각 정보 카드에 CRUD를 반복 적용하는 작업은 어제부터 이어온 패턴 덕분에 전날보다 훨씬 빠르게 진행할 수 있었습니다. 컴포넌트 구조와 모달 패턴이 팀 내에서 어느 정도 통일되기 시작한 것도 개발 속도 향상에 기여했습니다.
채팅창 뒤로가기 오동작 문제는 처음에는 사소한 버그처럼 보였지만, Expo Router의 중첩 레이아웃 구조와 히스토리 스택 관리 방식에 대한 깊은 이해가 필요한 문제임을 알게 되었습니다. 시간이 허락한다면 Expo Router 공식 문서의 Nested Navigation 섹션을 정독하고 문제를 근본적으로 해결하고 싶습니다.
