Troubleshooting

이슈 및 해결 내역

1. 모듈 경로 오류 (Unable to resolve)

원인: Expo Router 중첩 구조에서 상대경로(../../../) 오계산으로 루트 도달 실패
해결: @/ 절대경로 별칭(Alias)으로 코드 베이스 통일

2. chat_members ↔ profiles 조인 오류

Foreign Key 미설정으로 인한 조인 실패 현상을 분리 조회 후 JS에서 병합하는 방식으로 즉시 대응했습니다. 

const { data: memberData } = await supabase.from('chat_members').select('user_id');
const userIds = memberData.map((m) => m.user_id);
const { data: profilesData } = await supabase.from('profiles').in('id', userIds);

3. RLS 정책 무한 재귀 오류

현상: "infinite recursion detected in policy"
해결: SELECT 정책 내 서브쿼리 제거. auth.uid() = user_id 단순 비교로 수정
Database

DB 스키마 변경

기존 1:1 구조에서 1:N 다중 사용자 채팅이 가능하도록 구조를 고도화했습니다. 

CREATE TABLE chat_members (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  chat_id uuid REFERENCES chats(id),
  user_id uuid REFERENCES auth.users(id),
  role text -- owner | member
);

⚠️ 기존 채팅방 데이터는 SQL Editor를 통해 소유자(owner) 수동 등록 마이그레이션이 필수적입니다.
