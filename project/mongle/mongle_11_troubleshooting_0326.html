01 — Problem Definition

문제 정의

timeline.jsx 화면에서 '문의하기' 버튼을 눌러 채팅방(chat/[id].jsx)으로 이동한 후, 뒤로가기 버튼을 누르면 timeline 화면으로 돌아오지 않고 홈 화면으로 이동합니다.

🔴 증상   timeline.jsx → (문의하기 버튼) → chat/[id].jsx → (뒤로가기) → ❌ 홈 화면   ✅ 기대: timeline.jsx

1-1. 영향 범위

영역 영향 내용
사용자 경험 timeline에서 채팅방 진입 후 뒤로가기 시 원하는 화면이 아닌 홈으로 이동하여 UX 흐름 단절
Android 물리 뒤로가기 기기의 물리 뒤로가기 버튼도 동일 증상 발생
헤더 뒤로가기 버튼 router.back() 호출 시 동일 증상. router.dismiss()로 변경 후 일부 개선되었으나 완전 해결 안 됨
로그인 화면 뒤로가기 (별개 문제) 로그인 화면에서 Android 뒤로가기 시 GO_BACK 에러 발생
02 — Root Cause Analysis

원인 분석

문제의 근본 원인은 3개 레이아웃 파일이 형성하는 네비게이션 구조에 있습니다. 각 레이아웃의 역할과 상호작용을 분석합니다.

2-1. 초기 레이아웃 구조 (문제 발생 구조) 

app/_layout.jsx        → Slot (히스토리 스택 없음, 전역 Stack 부재)
(couple)/_layout.jsx   → Tabs (탭 네비게이션)
chat/_layout.jsx       → Stack (독립 스택 생성)

2-2. 스택 히스토리 생성 메커니즘

app/_layout.jsx가 Slot이므로 뒤로가기 히스토리를 전역으로 관리하는 최상위 Stack이 존재하지 않습니다. 그 결과 chat/_layout.jsx의 Stack이 유일한 스택이 되고, Expo Router는 이 스택의 첫 번째 항목으로 chat/index.jsx(채팅 목록 화면)를 자동 삽입합니다. 

chat 스택 구조: [index.jsx (채팅 목록)] → [id].jsx (채팅방)
뒤로가기 동작:
  1st back: [id].jsx → index.jsx (채팅 목록)
  2nd back: index.jsx → 홈 화면
  → timeline에서 진입했다는 정보가 chat 스택에 보존되지 않음

2-3. 핵심 의심 지점

Tabs 안에 Stack이 중첩될 때 Expo Router가 히스토리를 어떻게 관리하는지, 그리고 timeline에서 chat으로 push 시 해당 라우트가 어느 스택에 쌓이는지가 핵심 문제입니다.

  • timeline.jsx에서 router.push(chat/[id]) 호출 시 Tabs 내부 Stack이 아닌 chat의 독립 Stack에 쌓임
  • chat Stack은 자체 히스토리만 관리하므로 진입 이전 화면(timeline) 정보를 보유하지 않음
  • router.back()은 현재 스택 내 이전 항목으로만 이동하므로, chat 스택 내에서 index.jsx → 홈 순서로 이동
03 — Attempted Solutions

시도한 해결 방법

시도 1 router.dismiss() 적용 부분 성과
접근 방법
chat/[id].jsx 뒤로가기 버튼의 router.back()을 router.dismiss()로 변경
의도
router.dismiss()는 현재 modal/stack 전체를 닫고 상위 navigator로 복귀하므로 chat Stack 전체를 닫고 timeline으로 돌아올 것으로 예상
코드 변경 
// chat/[id].jsx — 변경 전
<TouchableOpacity onPress={() => router.back()} style={styles.backBtn}>
  <Ionicons name='chevron-back' size={24} color='#3a2e2a' />
</TouchableOpacity>

// chat/[id].jsx — 변경 후
<TouchableOpacity onPress={() => router.dismiss()} style={styles.backBtn}>
  <Ionicons name='chevron-back' size={24} color='#3a2e2a' />
</TouchableOpacity>
결과
헤더 뒤로가기 버튼은 chat Stack 전체가 닫히는 동작까지는 성공. 그러나 Android 물리 뒤로가기는 여전히 chat Stack 내부 순서(index.jsx → 홈)로 이동. 근본 해결 아님.
시도 2 chat/_layout.jsx 삭제 포기
접근 방법
chat 전용 Stack 레이아웃 파일(chat/_layout.jsx)을 삭제하여 (couple) Tabs 스택 안에서 처리하도록 시도
의도
독립 Stack 제거 시 (couple) Tabs의 히스토리 안에서 chat 화면들이 관리되어 뒤로가기가 정상 동작할 것으로 예상
결과
chat/index.jsx, chat/[id].jsx가 Tabs 내에서 렌더링되어 뒤로가기 개념 자체가 사라짐. 채팅 탭 진입 경로가 여러 개(Tabs 탭 직접 이동, timeline 문의하기, AI 채팅 등)이므로 chat/_layout.jsx를 완전히 삭제하는 것은 불가. 이 방향 포기.
시도 3 app/_layout.jsx Slot → Stack 변경 부분 성과
접근 방법
app/_layout.jsx를 Slot 구조에서 Stack으로 변경하여 전역 히스토리 스택 생성
의도
최상위 Stack이 생기면 timeline → chat 이동 시 해당 스택에 쌓여 뒤로가기가 정상 동작할 것으로 기대
코드 변경 
// app/_layout.jsx — 변경 후
import { Stack } from 'expo-router';
export default function RootLayout() {
  return (
    <SafeAreaProvider>
      <NotificationProvider>
        <StatusBar style='dark' />
        <Stack screenOptions={{ headerShown: false }} initialRouteName='(couple)'>
          <Stack.Screen name='(auth)' />
          <Stack.Screen name='(couple)' />
          <Stack.Screen name='(planner)' />
          <Stack.Screen name='planners' />
          <Stack.Screen name='vendors' />
        </Stack>
      </NotificationProvider>
    </SafeAreaProvider>
  );
}
결과
부분 성과: (auth) 첫 화면 문제가 발생했으나 initialRouteName='(couple)' 추가로 해결. 미해결: 채팅창 뒤로가기는 여전히 홈으로 이동. 최상위 Stack 추가만으로는 해결 불충분.
시도 4 Stack에 (couple)/chat/[id] 직접 등록 실패
접근 방법
app/_layout.jsx의 Stack.Screen에 중첩 경로명을 직접 등록하여 최상위 스택에서 chat 화면을 명시적으로 관리 시도
의도
chat/[id].jsx를 app 레벨 Stack에 직접 등록하면 전역 스택에서 히스토리가 관리되어 뒤로가기가 올바른 화면으로 돌아올 것을 기대
시도 코드 
// app/_layout.jsx — 시도 코드
<Stack.Screen name='(couple)/chat/[id]' />
결과
경고 발생: Warning: No route named '(couple)/chat/[id]' exists in nested children. Expo Router의 파일 기반 라우팅 구조에서 중첩 경로를 상위 Stack.Screen에 직접 문자열로 등록하는 것은 실제 라우트 트리와 경로명이 불일치하여 동작하지 않음.

05 — Next Steps

다음 세션 확인 방향

5-1. 파일 및 코드 재확인 항목

확인 파일 / 항목 확인 내용
app/_layout.jsx 현재 실제 파일 내용 재확인 — Stack 구조가 올바르게 적용되어 있는지 검증
(couple)/_layout.jsx Tabs 구성 내 chat 탭 진입 코드 확인 — router.replace vs router.push 여부
chat/_layout.jsx 독립 Stack 구조 재확인 — initialRouteName 설정 여부
timeline.jsx 문의하기 버튼의 router 코드 확인 — push인지 replace인지 구분 (replace 사용 시 히스토리 소실)
Expo Router 버전 package.json에서 expo-router 버전 확인 — 버전별 중첩 Stack 동작 차이가 큼

5-2. 시도 예정 접근법

아래 접근들을 우선순위 순으로 시도할 예정입니다.

  • timeline.jsx에서 router.push 대신 router.navigate 사용 시도 — navigate는 이미 스택에 있으면 pop, 없으면 push 처리
  • chat/_layout.jsx에서 initialRouteName='[id]' 지정하여 채팅 목록이 스택 바닥에 자동 삽입되는 것 방지
  • (couple)/_layout.jsx의 chat 탭 href 설정 확인 — href: null 처리 시 탭 자체에서 직접 라우팅 불가하게 되어 스택 구조 변화
  • Expo Router 공식 문서의 Nested Navigation / Modal 패턴 재검토
  • Expo Router v3의 withAnchor 옵션 또는 useLocalSearchParams 활용 가능성 검토

💡 핵심 가설: timeline.jsx에서 router.push('/(couple)/chat/id?xxx=yyy')로 이동 시, 해당 push가 app 레벨 Stack이 아닌 chat/_layout.jsx의 Stack에 쌓이는 것이 문제의 핵심. chat Stack이 timeline의 히스토리를 모르기 때문에 뒤로가기 시 홈으로 이동함. 해결 방향: chat Stack이 아닌 app 레벨 Stack에서 chat/[id]를 관리하거나, router.dismiss()와 router.navigate()를 조합하여 강제로 timeline으로 복귀하는 방법 모색.
