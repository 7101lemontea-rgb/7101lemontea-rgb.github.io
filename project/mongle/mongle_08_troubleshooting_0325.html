Issue 01 — Git

브랜치 push 시 폴더가 사라지는 문제

이슈 ID
TS-01
증상
design/#2-timeline-ui 브랜치에 push할 때마다 app/(couple)/chat 폴더가 사라짐
해결 여부
✅ 해결 완료

원인

chat 폴더는 feat/#10-chat 브랜치에서 추가되었으나, design/#2-timeline-ui 브랜치는 그 이전 커밋에서 분기되어 chat 폴더의 존재를 알지 못합니다. 

# 브랜치 구조 (분기 시점 기준)
main      A -- B -- C(chat 추가) -- D
               \
design/#2       E -- F  ← 이 시점에는 chat 없음

진단 명령어

# chat 폴더가 언제 추가됐는지 확인
git log --oneline -- "app/(couple)/chat"

# 브랜치 분기 시점 확인
git merge-base main design/#2-timeline-ui

해결 방법

main을 현재 브랜치에 merge하여 chat 폴더를 포함시킵니다.

git checkout design/#2-timeline-ui
git merge main

merge 충돌 발생 시 VS Code에서 timeline.jsx 충돌을 해결합니다.
• Accept Current Change → 내 브랜치(design/#2) 내용 유지 (권장)
• Accept Incoming Change → main 내용으로 덮어씀 

# 충돌 해결 후 커밋 및 push
git add "app/(couple)/timeline.jsx"
git commit -m "merge: main 병합 및 timeline UI 수정"
git push origin design/#2-timeline-ui
Issue 02 — Git / PowerShell

PowerShell에서 괄호() 경로 인식 오류

이슈 ID
TS-02
해결 여부
✅ 해결 완료

에러 메시지

couple : 'couple' 용어가 cmdlet, 함수, 스크립트 파일 또는
실행할 수 있는 프로그램 이름으로 인식되지 않습니다.

원인

PowerShell은 괄호 ()를 명령어 또는 표현식으로 해석하여 경로의 일부로 인식하지 못합니다.

해결 방법

경로를 큰따옴표로 감싸서 입력합니다. 괄호가 포함된 경로는 항상 따옴표로 감싸는 것을 권장합니다. 

# ❌ 잘못된 방법
git add app/(couple)/timeline.jsx

# ✅ 올바른 방법
git add "app/(couple)/timeline.jsx"
Issue 03 — Git

not a git repository 에러

이슈 ID
TS-03
해결 여부
✅ 해결 완료

에러 메시지

fatal: not a git repository (or any of the parent directories): .git

원인

git 명령어를 저장소 루트가 아닌 상위 폴더에서 실행했습니다. 실제 저장소는 하위 폴더(Mongle-app)에 위치합니다.

해결 방법

올바른 저장소 경로로 이동 후 명령어를 실행합니다.

cd C:\workspace\mongle\mongle\Mongle-app
git status
실제 저장소 경로
C:\workspace\mongle\mongle\Mongle-app
Issue 04 — timeline.jsx

타임라인 모달 알려진 이슈 및 해결 내역

이슈 ID
TS-04
영향 파일
timeline.jsx
해결 여부
✅ 전체 해결 완료
이슈 원인 해결
모달 내 항목 목록 미표시 ScrollView에 flex:1 적용 시 maxHeight 부모 안에서 높이 계산 실패 style을 flexGrow:1 또는 maxHeight 고정값으로 변경
날짜 미입력으로 표시 모달 접힌 행이 item.date를 참조 (formatDate 미사용) collapsedDate를 formatDate(item)으로 변경
formatDate ReferenceError 함수가 컴포넌트 내부에 선언됨 컴포넌트 함수 바깥(최상단)으로 이동
날짜 입력창 줄바꿈 modalStyles에 dateRow / dateInput / dateSep 미정의 modalStyles에 날짜 관련 스타일 추가

재발 방지 원칙: 헬퍼 함수(formatDate 등)는 컴포넌트 최상단 밖에 선언하세요. 모달 내 스크롤이 필요한 경우 flex:1 대신 flexGrow:1을 우선 사용하고, 날짜 관련 스타일은 반드시 modalStyles에 별도 정의하세요.
