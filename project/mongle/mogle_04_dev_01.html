1장 — Git Branch & Commit Convention

Git 브랜치 & 커밋 컨벤션

이 프로젝트에서 Git 브랜치를 생성하거나 커밋을 수행할 때 반드시 지켜야 하는 팀 규칙입니다.

1.1 브랜치명 구조

📌 브랜치명 형식
타입/#이슈번호-작업내용
예시: feat/#1-login-ui

1.2 타입(Type) 정의

타입 설명
feat/ 새로운 핵심 기능 개발
fix/ 버그 수정
design/ UI/UX 퍼블리싱 및 스타일링 작업
refactor/ 기능 변화 없이 코드 구조·로직 개선
chore/ 패키지 설치, 환경 설정 파일 수정
docs/ README.md 등 문서 수정

1.3 작성 세부 규칙

  • 반드시 소문자만 사용할 것
  • 단어 사이는 하이픈(-)으로 연결 (띄어쓰기·언더바 금지)
  • 영어로 명확하고 간결하게 작성할 것

⚠️ 주의사항  브랜치 생성이나 커밋 전 항상 양식에 맞는지 Self-Check 하세요.

1.4 타입별 명령어 모음

feat/ 새로운 기능 개발 
git checkout -b feat/#1-feature-name
git commit -m "feat: 기능 설명"
fix/ 버그 수정 
git checkout -b fix/#1-bug-name
git commit -m "fix: 버그 수정 내용"
design/ UI/UX 퍼블리싱 
git checkout -b design/#1-ui-name
git commit -m "design: UI 스타일링 내용"
refactor/ 코드 리팩토링 
git checkout -b refactor/#1-refactor-name
git commit -m "refactor: 리팩토링 내용"
chore/ 환경 설정·패키지 
git checkout -b chore/#1-chore-name
git commit -m "chore: 설정 변경 및 패키지 설치 내용"
docs/ 문서 작업 
git checkout -b docs/#1-doc-name
git commit -m "docs: 문서 작성 및 수정 내용"

1.5 Pull Request 템플릿

PR 생성 시 아래 항목을 작성하고 체크리스트를 확인하세요.

💡 어떤 작업을?
작업 내용을 체크리스트로 작성
📋 관련 이슈
Resolves: #이슈번호
📸 스크린샷
UI 변경이 있는 경우 AS-IS / TO-BE 첨부
✅ PR 체크리스트
아래 항목 확인 후 머지 요청
🔍 리뷰어에게
특별히 확인 요청할 내용 기재

PR 머지 전 체크리스트

  • 내가 작성한 코드를 스스로 꼼꼼히 리뷰(Self-Review)했나요?
  • 로컬 환경에서 에러 없이 정상적으로 앱이 실행되는 것을 확인했나요?
  • 불필요한 console.log나 디버깅용 주석은 모두 지웠나요?
  • .env에 추가할 새로운 환경 변수가 있다면 팀원들에게 공유했나요?
  • PR 제목 양식(브랜치·커밋 컨벤션)을 잘 지켰나요? (예: feat/#1-login-ui)
2장 — Expo Router Routing Structure

Expo Router 라우팅 구조

Expo Router는 파일 시스템 기반 라우팅을 사용합니다. app/ 폴더 구조가 곧 화면 구조입니다.

2.1 앱 실행 흐름

🚀 앱 진입 순서
1. app/_layout.jsx  →  RootLayout 실행 (SafeAreaProvider + Slot)
2. app/(couple)/_layout.jsx  →  Tabs 네비게이터 실행
3. app/(couple)/index.jsx  →  홈 화면 표시

2.2 파일 구조 및 역할

파일 경로 컴포넌트명 역할
app/_layout.jsx RootLayout SafeAreaProvider + Slot. 앱 진입점
app/(auth)/_layout.jsx AuthLayout Stack 네비게이터. 로그인·회원가입 그룹
app/(auth)/login.jsx LoginScreen 이메일/비밀번호 + 카카오 로그인 UI
app/(auth)/register.jsx RegisterScreen 회원가입 화면 (임시)
app/(couple)/_layout.jsx CoupleLayout Tabs 네비게이터. 홈·일정·채팅·마이 탭
app/(couple)/index.jsx HomeScreen 메인 홈 화면 (배너·카테고리·업체 카드)
app/(couple)/timeline.jsx TimelineScreen 일정 화면 (임시)
app/(couple)/chat.jsx ChatScreen 채팅 화면 (임시)
app/(couple)/mypage.jsx MypageScreen null 반환. tabBarButton이 로그인으로 이동
app/vendors/index.jsx VendorListScreen 업체 목록 + 카테고리 필터
app/vendors/[id].jsx VendorDetailScreen 업체 상세 (소개·가격·후기 탭)

2.3 그룹 폴더 개념

괄호 ()로 감싼 폴더명은 URL 경로에 포함되지 않고 그룹핑 용도로만 사용됩니다.

(auth) 그룹 Stack Navigator
  • 로그인·회원가입 화면
  • 탭 네비게이션 없음
  • Stack 네비게이터 사용
(couple) 그룹 Tabs Navigator
  • 커플 전용 탭 화면들
  • 하단 탭바 표시
  • Tabs 네비게이터 사용

2.4 탭 네비게이터 구성

(couple)/_layout.jsx의 탭 구성입니다.

탭 연결 파일 아이콘 비고
홈 index.jsx home-outline 기본 화면
일정 timeline.jsx calendar-outline —
채팅 chat.jsx chatbubble-outline —
마이 mypage.jsx person-outline 클릭 시 로그인 화면으로 이동
(숨김) budget.jsx · docs.jsx · vendors.jsx — href: null 처리

2.5 마이 탭 — 로그인 연동 방식

mypage.jsx 파일은 존재하지만 null을 반환합니다. 실제 이동은 tabBarButton에서 처리합니다. 

// (couple)/_layout.jsx
<Tabs.Screen
  name="mypage"
  options={{
    tabBarButton: (props) => (
      <TouchableOpacity
        {...props}
        onPress={() => router.push("/(auth)/login")}
      />
    ),
  }}
/>

2.6 vendors 라우트 구조

업체 관련 화면은 탭 밖의 독립 라우트로 구성되어 있어 탭바 없이 풀스크린으로 열립니다. 

vendors 폴더 구조app/vendors/
  ├── index.jsx  →  업체 목록 + 카테고리 필터 (더보기 클릭)
  └── [id].jsx   →  업체 상세 (소개 · 가격 · 후기 탭)
3장 — Layout UI Complete

레이아웃 UI 완성

오늘 완성한 화면 목록과 주요 구현 내용입니다.

3.1 완성된 화면 목록

화면 파일 주요 구현
메인 홈 (couple)/index.jsx 배너 슬라이더, 카테고리 탭, 업체 카드 그리드, 서브탭
로그인 (auth)/login.jsx 이메일·비밀번호 입력, 카카오 로그인 버튼, 회원가입 링크

3.2 메인 홈 화면 (couple/index.jsx)

헤더

Mongle 로고(가운데 정렬) + 알림 버튼(우측)

검색바

둥근 모서리, 우측 필터 아이콘

배너 슬라이더

가로 스크롤, 하단 인디케이터 dots

카테고리 탭

스튜디오·웨딩홀·드레스·메이크업 + 더보기 버튼

업체 카드 그리드

2열 카드, 이름·거리·별점 표시. 카드 클릭 → /vendors/[id] (업체 상세)

서브 탭

본식·패키지·웨딩플래너. 더보기 클릭 → /vendors (업체 목록)

3.3 하단 탭바 스타일

tabBarStyle: {
  backgroundColor: "#fff",
  borderTopColor: "#f0e8e4",
  borderTopWidth: 1,
  borderBottomColor: "#f0e8e4",
  borderBottomWidth: 1,
  height: Platform.OS === "android" ? 70 : 60,
  paddingBottom: Platform.OS === "android" ? 12 : 8,
  paddingTop: 8,
},

💡 Platform 분기 이유  Android 기기의 제스처 네비게이션 바(24~48px)와 탭바가 겹치는 문제를 해결하기 위해 Platform.OS로 Android/iOS 높이를 다르게 설정했습니다.

3.4 공통 디자인 시스템

항목 값 용도
Primary #6b4c4c 로고, 주요 텍스트
Accent #c9a98e 활성 탭, 버튼, 밑줄, 아이콘
Background #f5f0ee 검색바, 카드 배경, 비활성 탭
Text #3a2e2a 본문 텍스트
Subtext #8a7870 보조 텍스트, 비활성 아이콘
Border #f0e8e4 구분선, 카드 테두리
Border Radius 16~24px 카드, 버튼, 검색바

3.5 SafeAreaView 처리

react-native 기본 SafeAreaView는 deprecated되어 react-native-safe-area-context로 교체했습니다.

변경 전 
import { SafeAreaView } from "react-native"
변경 후 
import { SafeAreaView } from "react-native-safe-area-context"

app/_layout.jsx에 SafeAreaProvider를 추가하여 전체 앱에 적용했습니다. 

// app/_layout.jsx
import { SafeAreaProvider } from "react-native-safe-area-context";

export default function RootLayout() {
  return (
      <SafeAreaProvider>
        <Slot />
      </SafeAreaProvider>
  );
}
