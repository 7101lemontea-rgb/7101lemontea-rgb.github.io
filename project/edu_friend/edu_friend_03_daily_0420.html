01 — As-Is Analysis

현황 분석

1.1 대상 도메인 및 서비스 범위

다문화 가정 초등학생의 수학 교과 학습 보조에 집중합니다. 특히 언어 장벽으로 인해 수학적 원리가 아닌 '문장제 지문' 자체를 이해하지 못해 발생하는 학습 결손 문제를 해결하고자 합니다.

1.2 현재 사용자 행동 흐름 (As-Is Flow)

01
지문 봉착
수학 문제를 접하지만 지문의 뜻을 모름
02
도움 요청
부모님께 질문하나, 부모님 역시 한국어 교과 개념 미숙지
03
학습 중단
질문 답변 실패로 인해 이해를 포기하고 그냥 넘어감

1.3 현재 문제점 정리

항목 현황 (2025 기준) 문제점
다문화 학생 수 약 20만 명 맞춤형 학습 지원 부족
국적 비중 베트남(31.3%) 최대 비중 베트남어 특화 도구 부재
해결 소요 시간 개념 1개당 20~30분 소요 학습 효율 극저하
  • 정성적 문제: 번역기는 단어 뜻만 제공할 뿐 교과 개념 설명이 불가능하며, 유튜브 강의는 학생 수준에 맞지 않는 경우가 많습니다.
  • 기술적 한계: 기존 앱은 한국어 단일 언어 기반이며, 문제 풀이 정답 제공 위주의 구조로 설계되어 있습니다.
02 — To-Be Requirements

요구사항 정의

2.1 프로젝트 목표 (10일 MVP 기준)

수학 1개 단원 범위 내에서, 학생이 모르는 개념을 한국어 또는 베트남어로 즉시 질문하고 AI로부터 쉬운 설명과 예시를 받을 수 있는 최소 기능 학습 도우미를 구현합니다.

2.2 기능적 요구사항 (Functional)

FR-001 질문 인터페이스
문제 해설 화면에서 모르는 개념을 텍스트로 질문 가능
FR-002 이중언어 AI
AI가 한국어/베트남어로 개념 설명 및 예시 제공
FR-003 오답 케어
오답 문제에 대해 관련 개념 재설명 기능 제공

2.3 비기능적 요구사항 및 제약사항

성능/안정성

응답 시간 5초 이내 목표, API 오류 시 재시도 처리

기술 제약

RAG + GPT API 활용, 10일 이내 MVP 완성

범위 제약

수학 1과목 특정 1개 단원으로 한정

03 — MoSCoW Priority

요구사항 우선순위

✅ Must Have 반드시 포함되어야 할 기능
  • AI 개념 질문 및 답변 기능
  • 이중언어(한국어·베트남어) 설명 엔진
  • 문제 풀이 화면 UI
🔵 Should Have 중요하지만 연기 가능한 기능
  • 오답 재설명 및 예시 문제 제공
  • 핵심 용어 한-베 대응 표시
❌ Won't Have 이번 범위에서 제외
  • 전 과목 확장 및 자동 문제 생성
  • 고도화된 추천 시스템
Conclusion

정리 요약

핵심 문제: 다문화 가정 학생은 언어 장벽으로 인해 혼자 공부할 때 개념을 이해하지 못하는 상황이 반복됩니다.

해결 방향: 학생이 막히는 순간 친구처럼 물어볼 수 있는 이중언어 AI 학습 도우미를 통해, 스스로 문제를 해결할 수 있도록 지원합니다.
