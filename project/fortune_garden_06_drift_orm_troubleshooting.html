TS-01 · 오류 (Error)

argument_type_not_assignable tables.dart

2.1 오류 정보

발생 파일
lib/data/database/tables.dart
발생 위치
43번째 줄 (Accounts 테이블 indexes 블록)
오류 코드
argument_type_not_assignable
심각도
오류 (severity: 8) — 컴파일 불가
발생 시점
VS Code Dart 분석기 실시간 감지 / build_runner 실행 시

2.2 오류 메시지

The argument type 'List<String>' can't be assigned to the parameter type 'String'.

2.3 원인 분석

Drift v2에서는 테이블 인덱스를 선언하는 방식으로 List<Index> get indexes 오버라이드를 지원하지 않는다. 초기 구현 시 Index() 생성자에 List<String> 타입의 컬럼 목록을 전달하였으나, 이 생성자의 두 번째 파라미터는 String 단일 값을 요구하기 때문에 타입 불일치 오류가 발생하였다.

Transactions 테이블에서도 동일한 패턴으로 5개 인덱스가 동일하게 오류를 유발하였으며, CategoryRules, SyncHistory 테이블도 같은 문제가 있었다. 

// Drift v2에서 지원하지 않는 방식
class Accounts extends Table {
  ...
  @override
  List<Index> get indexes => [
    Index('idx_acc_profile', ['profile_id']), // List<String> → 타입 오류
  ];
}

2.4 해결 방법

Drift v2의 올바른 인덱스 선언 방식인 @TableIndex 어노테이션으로 전면 교체하였다. 어노테이션은 클래스 위에 선언하며 columns에는 List<String> 대신 Dart Symbol 집합({#컬럼명})을 사용한다. 

// Drift v2 올바른 방식 --- @TableIndex 어노테이션
@TableIndex(name: 'idx_acc_profile', columns: {#profileId})
class Accounts extends Table {
  ...
  // List<Index> get indexes 블록 전체 삭제
}

핵심 변경: 클래스 내부의 List<Index> get indexes 블록을 삭제하고, 클래스 선언 바로 위에 @TableIndex 어노테이션을 추가한다. 복수 인덱스는 어노테이션을 여러 줄로 나열한다.

2.5 변경 파일 및 범위

대상 클래스 변경 유형 변경 내용
Accounts 교체 @TableIndex(name: 'idx_acc_profile', columns: {#profileId}) 어노테이션으로 교체. indexes 블록 삭제
Transactions 교체 4개 @TableIndex 어노테이션으로 교체. idx_txn_hash는 txnHash 컬럼의 .unique()로 대체되어 별도 선언 불필요
CategoryRules 교체 @TableIndex(name: 'idx_rule_keyword', columns: {#keyword}) 어노테이션으로 교체
SyncHistory 교체 @TableIndex(name: 'idx_sync_profile_start', columns: {#profileId, #startedAt}) 어노테이션으로 교체

2.6 idx_txn_hash 인덱스 처리

기존 코드에는 txn_hash 컬럼에 Index()로 UNIQUE 인덱스를 별도 선언하였으나, 해당 컬럼이 이미 .unique()로 선언되어 있어 SQLite가 자동으로 UNIQUE 인덱스를 생성한다. 따라서 @TableIndex 전환 시 idx_txn_hash는 중복 선언이므로 제거하였다. 

// txnHash 컬럼의 .unique()가 idx_txn_hash 역할을 대신함
TextColumn get txnHash => text().unique()(); // ← 이미 UNIQUE 인덱스 자동 생성

// 따라서 아래 어노테이션은 불필요 --- 추가하지 않음
// @TableIndex(name: 'idx_txn_hash', columns: {#txnHash}, unique: true)  ← 제거

2.7 테이블 선언 순서 주의사항

이번 수정 과정에서 테이블 선언 순서가 ERD 번호 순서(4.1 → 4.2 → 4.3)와 다르다는 점이 확인되었다. 이는 오류가 아니라 FK 의존 관계에 의한 의도적인 순서이다.

선언 순서 클래스 이유
1번째 Profiles 모든 테이블의 루트 엔티티. 참조 대상이므로 가장 먼저 선언
2번째 Institutions Accounts의 FK 참조 대상. Accounts보다 먼저 선언 필요
3번째 Accounts Profiles, Institutions를 참조. 두 클래스 선언 후에 위치
4번째 Categories Transactions의 FK 참조 대상. Transactions보다 먼저 선언
5번째 Transactions Profiles, Accounts, Categories를 모두 참조. 가장 나중
6~8번째 나머지 CategoryRules, SyncHistory, AppSettings 순서는 자유

Drift는 테이블 클래스를 위에서 아래로 읽으며 참조 대상이 이미 선언되어 있는지 확인한다. FK 참조 대상 클래스가 아직 선언되지 않은 상태에서 참조하면 컴파일 오류가 발생하므로 이 순서를 변경해서는 안 된다.

3.2 오류 메시지

Unused import: 'package:drift/drift.dart'.
Try removing the import directive.

Unused import: 'tables.dart'.
Try removing the import directive.

3.3 원인 분석

seed_institutions.dart가 사용하는 심볼은 InstitutionsCompanion과 db.institutions 두 가지이다. 이 두 심볼의 출처를 추적하면 다음과 같다.

사용 심볼 실제 출처 설명
InstitutionsCompanion app_database.g.dart build_runner가 Institutions 클래스를 기반으로 자동 생성한 클래스. .g.dart 파일에 위치
db.institutions app_database.g.dart AppDatabase의 테이블 접근자. 역시 .g.dart에 자동 생성

app_database.dart 파일 상단에 part 'app_database.g.dart' 선언이 있어, app_database.dart를 import하면 .g.dart의 모든 심볼이 함께 노출된다. 따라서 drift와 tables.dart를 별도로 import할 필요가 없으며, 이 두 import는 실제로 파일 내 어디서도 직접 사용되지 않아 경고가 발생하였다. 

// app_database.dart 내부 구조
import 'package:drift/drift.dart';
import 'tables.dart';
part 'app_database.g.dart'; // ← .g.dart의 모든 심볼이 여기에 포함됨

// seed_institutions.dart에서 app_database.dart만 import하면
// drift와 tables.dart의 심볼을 모두 간접적으로 사용할 수 있음

3.4 해결 방법

불필요한 import 2개를 제거하고 app_database.dart import 하나만 유지하였다.

전
수정 전 import 3개 
import 'package:drift/drift.dart'; // ❌ 미사용
import 'app_database.dart';
import 'tables.dart';              // ❌ 미사용
후
수정 후 import 1개 
import 'app_database.dart'; // ✅ 이 하나로 drift + tables + .g.dart 모두 포함

3.5 import 구조 이해

Drift 프로젝트에서 파일 간 import 의존 관계는 아래와 같다. 이 구조를 이해하면 불필요한 중복 import를 방지할 수 있다. 

seed_institutions.dart
└─ import 'app_database.dart'    // 이것 하나면 충분
   ├─ import 'package:drift/drift.dart'  // drift 심볼 포함
   ├─ import 'tables.dart'              // 테이블 클래스 포함
   └─ part   'app_database.g.dart'      // Companion 클래스 등 포함
Section 04

재발 방지 원칙

4.1 Drift 인덱스 선언 규칙

Drift v2 이상에서 인덱스를 선언할 때는 반드시 @TableIndex 어노테이션을 사용한다. 아래 규칙을 준수한다.

규칙 설명
어노테이션 위치 반드시 class 선언 바로 위에 작성. 복수 인덱스는 어노테이션을 여러 줄로 나열
columns 타입 Set<Symbol> 타입 사용. {#컬럼명} 형태로 작성. List<String> 사용 금지
.unique() 중복 금지 컬럼에 .unique()가 이미 선언된 경우 동일 컬럼의 @TableIndex(unique: true) 추가 불필요
indexes 오버라이드 금지 List<Index> get indexes 오버라이드 방식은 Drift v2 미지원. 사용 금지

4.2 Drift 파일 import 규칙

Drift 프로젝트에서 파일 간 import를 작성할 때는 아래 원칙을 따른다.

규칙 설명
app_database.dart 단일 import DB를 사용하는 파일은 app_database.dart 하나만 import. drift, tables.dart 직접 import 불필요
part 구조 인지 app_database.g.dart는 part로 선언되어 app_database.dart에 포함됨. 직접 import 금지
중복 import 주기적 점검 VS Code의 Dart 분석기 경고(노란 밑줄) 즉시 처리. unused import는 빌드 전 모두 제거

4.3 올바른 코드 패턴 요약

이번 트러블슈팅을 통해 확립된 Drift 코드 작성 패턴이다.

// 올바른 Drift v2 인덱스 + import 패턴

// [tables.dart] 인덱스 선언
@TableIndex(name: 'idx_acc_profile', columns: {#profileId})
@TableIndex(name: 'idx_txn_profile_date', columns: {#profileId, #txnDate}, unique: true)
class Accounts extends Table { ... }

// [seed_institutions.dart] import --- 1개만 사용
import 'app_database.dart'; // drift + tables + .g.dart 모두 포함
