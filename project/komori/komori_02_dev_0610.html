2. 대화 시스템 방식

자유 대화와 핵심 규칙 처리를 어떻게 나눌지 결정했습니다. 프롬프트 엔지니어링만으로는 캐릭터 행동을 완전히 제어하기 어렵다는 판단 하에, 코드 레이어와의 하이브리드 방식을 채택했습니다.

LLM 담당
자유 대화 전반 — 날씨, 일상, 감정 반응 등 캐릭터 말투로 응답
코드 레이어 담당
핵심 규칙 — 집 진입 요청 거절, 이상 출력 감지 및 재시도

3. 모델 현황

모델 한국어 라이선스 상태
EXAONE 3.5 7.8B ★★★★★ 비상업 (LG) 품질 양호, 배포 불가
Qwen 2.5 7B ★★★ Apache 2.0 중국어 혼용·포맷 이탈 발생
Qwen 2.5 14B ★★★★ Apache 2.0 안정적이나 사양 과도

미결정 — 배포용 모델 선택: EXAONE 상업 라이선스 문의 / Qwen 14B / 클라우드 API 세 가지 방향 중 아직 결정되지 않음.

Part 2 — 이상 출력 대응

이상 출력 시행착오 상세

A1. 중국어 혼용 (Qwen 2.5 7B)

증상
한국어 문장 도중 한자 또는 중국어 어구가 갑자기 삽입됨. 특히 감정이 들어간 문장이나 문장 말미에서 자주 발생.
예시
그래도 친구로서... 보고싶었어. (友达として気に...)
원인
Qwen 2.5의 중국어·일본어 학습 데이터 비중이 높아, 한국어 생성 중 타 언어 토큰이 혼입됨.
시도 결과
"한국어로만 대답해" 프롬프트 추가 줄어들었으나 완전 차단 안 됨
프롬프트 최상단에 언어 규칙 배치 다소 개선
시스템 + 사용자 메시지 양쪽 언어 규칙 명시 여전히 간헐적 발생

코드 레이어에서 유니코드 범위를 감지해 재시도합니다. 일본 가나는 별도 범위이므로 미포함 (필요 시 추가 가능). 

function hasChinese(text) {
  return /[一-鿿㐀-䶿]/.test(text);
}
// CJK 통합 한자(一-鿿) + 확장 영역(㐀-䶿) 포함

A2. 대화 스크립트 포맷 출력

증상
모델이 대화 생성 요청을 "스크립트를 작성하는 작업"으로 이해해, 채팅 응답 대신 극본 형식 전체를 출력. 타임스탬프, 장소 태그, 캐릭터 이름 접두어가 모두 노출됨.
예시
SCHOOL: 01:30 PM / 코모리: 뭐야. / 히키: 왜 안 나와?
시도 결과
"대화 스크립트 금지" 명시 부분 개선
예시를 코모리: ~~ 형태로 작성 오히려 해당 패턴 강화
예시 형식을 "입력" → "응답"으로 변경 개선됨
출력 규칙을 프롬프트 최상단으로 이동 추가 개선 
function hasFormatLeak(text) {
  return /히키\s*:|코모리\s*:|SCHOOL\s*:|HOME\s*:|\d{1,2}:\d{2}\s*(AM|PM)/i.test(text);
}

A3. 영어 단어 혼용

증상
한국어 문장 중간에 의미 없는 영문자열 삽입. 실제 영어 단어가 아닌 난수성 알파벳도 발생 (모델 hallucination으로 추정).
예시
나도 모르겠어. lowes 어디서 나온 건지.  /  ekaomooro nunda

1글자(이니셜, 단위 등)는 허용하고 2글자 이상 연속만 차단합니다.

function hasEnglish(text) {
  return /[a-zA-Z]{2,}/.test(text);
}

A4. 문자 반복 출력

증상
모델이 토큰 생성 루프에 빠져 같은 문자 또는 어구를 수십 회 반복 출력. Qwen 7B에서 특히 자주 발생. 응답 길이 제한이 없으면 수백 토큰이 낭비됨.
예시
@@@@@@@@@@@@@@@@@@@@@@  /  그래그래그래그래그래그래그래그래

(.)\1{3,} — 임의 문자가 자신과 동일하게 3번 더 연속되면(총 4회 이상) 감지합니다. 

function hasRepetition(text) {
  return /(.)\1{3,}/.test(text);
}

A5. 이상 기호 출력

증상
응답에 대괄호, 중괄호, @, # 등 프로그래밍·마크다운 기호가 포함됨. 모델이 내부 주석이나 메타 정보를 노출하는 경우.
예시
그건 말하기 싫어. [[[집 안 상황]]] {blocked}  /  나 지금 바빠 @mention #tag 
function hasWeirdSymbols(text) {
  return /[@#$%^&*=|<>{}\[\]\\]/.test(text);
}

A6. 재시도 전략 변경 이력

1차 구현 폐기
스트리밍 중 이상 감지 → 즉시 중단 후 (재응답 중…) 메시지 표시.
문제: 사용자가 이상 응답 내용을 일부 목격한 뒤 로딩 메시지가 보임. UX 부자연스러움.
2차 구현 현재 채택
  • 스트리밍을 백그라운드에서 완전히 수신
  • 사용자에게는 타이핑 인디케이터(……)만 표시
  • 검수 통과 시 한 번에 출력
  • 실패 시 재시도 최대 2회 (재시도 중에도 인디케이터 유지)
  • 2회 후에도 실패 시 …… fallback 출력 
const RETRY_PROMPT = '한국어로만 답해줘. 영어, 중국어 한 글자도 쓰지 마. 짧게.';
const FALLBACK_RESPONSE = '……';

for (let attempt = 0; attempt < 2 && needsRetry(fullText); attempt++) {
  const retryMessages = [...baseMessages,
    { role: 'assistant', content: fullText },
    { role: 'user', content: RETRY_PROMPT }
  ];
  fullText = await requestOllama(retryMessages);
}
if (needsRetry(fullText)) fullText = FALLBACK_RESPONSE;

재시도 시 컨텍스트 처리 — 이전 이상 응답을 assistant 턴으로 포함시킨 뒤 user 턴으로 교정 요청. 모델이 자신의 직전 출력을 보고 수정할 수 있도록 설계.

A7. 긴 입력 처리

플레이어가 길고 복잡한 문장을 입력하면 LLM 혼란 응답 가능성이 높아집니다. 또한 게임 내 캐릭터(코모루)가 장황한 설명을 이해 못 하는 것이 오히려 자연스럽다는 판단에서, 50자 초과 입력은 LLM을 호출하지 않습니다. 

const MAX_INPUT_LENGTH = 50;
const CONFUSED_RESPONSES = [
  '……무슨 말인지 모르겠어.',
  '……그게 다 무슨 소리야.',
  '너무 많이 말하지 마.',
  '……',
  '잠깐, 천천히 말해줘.',
];

딜레이를 두는 이유: 즉각 응답하면 LLM을 사용하지 않은 것이 드러나 몰입감이 깨짐. 600ms 딜레이 후 랜덤 응답을 반환합니다.

응답 검수 통합 함수

function needsRetry(text) {
  return hasChinese(text) || hasFormatLeak(text) || hasEnglish(text)
      || hasRepetition(text) || hasWeirdSymbols(text);
}
감지 함수 감지 조건 주요 증상
hasChinese [一-鿿㐀-䶿] 한자·중국어 혼용
hasFormatLeak 히키:, 코모리:, SCHOOL:, 타임스탬프 스크립트 포맷
hasEnglish 영문자 2글자 이상 연속 영단어·알파벳 혼용
hasRepetition 같은 문자 4회 이상 연속 @@@@, 그래그래그래그래
hasWeirdSymbols @#$%^&*=|<>{}[]\ 기호·태그 노출

시스템 프롬프트 구조

프롬프트는 우선순위 순서대로 8개 블록으로 구성합니다. 출력 규칙을 최상단에 배치하는 것이 형식 이탈 방지의 핵심입니다.

① 출력 규칙 최상단 배치
형식 이탈 방지 — 이름 태그 금지, 지문 금지, 스크립트 포맷 금지
② 언어 규칙
한국어 전용, 영어·중국어 한 글자도 금지
③ 현재 상황
문 5cm 열린 채 대화, 집 안에 코모리 있음
④ 배경·심리
코모루의 동기와 두려움
⑤ 말투 규칙
주제별 온도 차 (안전/위험 주제 구분)
⑥ 절대 금지 목록
집 초대, 이유 설명, 감정 표현 등
⑦ 금지 패턴 목록
"나중에 이야기할게" 등 반복 문구
⑧ 대화 예시
"입력" → "응답" 형태 (캐릭터 이름 접두어 없이)

프롬프트 반복 수정 이력

문제 해결책
존댓말 사용 반말 명시, 금지 어미 목록 추가
집 안 초대 ("앉아줘", "뭐 마실래") 금지 명시
이유 설명 ("복잡해서", "위험해서") "그냥." 또는 침묵으로만 대체
반복 패턴 ("나중에 이야기할게") 금지 문구 목록 추가
질문 과다 "질문 먼저 하지 않는다" 명시
감정 직접 표현 ("불편해지네") 금지 명시
~다, ~냐 어미 올바른 어미 목록 명시
규칙을 대사로 출력 ("들어가긴 없어") "규칙은 내적 상태" 명시
날씨 주제 반복 주제 전환 예시 삭제, 침묵으로만 처리
(침묵) 지문 출력 예시 형식 변경 + 금지 명시
공간 맥락 부재 "문 5cm 열린 채" 명시
거짓 전제 수용 ("저번엔 됐잖아") 부정 예시 추가
