GameManager.cs

Assets/Scripts/GameManager.cs — 씬 간 게임 상태 유지 싱글톤

using UnityEngine;
using System.Collections.Generic;

public class GameManager : MonoBehaviour
{
    public static GameManager Instance;

    public int  chapter = 1;
    public int  suspicionGauge = 0;
    public bool hasDiary = false;
    public List<string> clues = new List<string>();

    void Awake()
    {
        if (Instance == null)
        {
            Instance = this;
            DontDestroyOnLoad(gameObject);
        }
        else
        {
            Destroy(gameObject);
        }
    }

    public void AddClue(string clueId)
    {
        if (!clues.Contains(clueId)) clues.Add(clueId);
    }
}
DontDestroyOnLoad
씬 전환 시에도 오브젝트가 파괴되지 않고 유지됨
싱글톤 패턴
두 번째 인스턴스가 생성될 경우 즉시 파괴 — 씬 이동 후 중복 생성 방지
상태 동기화
서버 응답 후 DialogueUI.cs에서 GameManager.Instance에 접근해 챕터·의심 게이지 갱신

InteractableObject.cs

Assets/Scripts/InteractableObject.cs — 벨, 쓰레기통, 창문 등 고정 텍스트 상호작용 오브젝트

기본 동작
히키가 일정 거리 이내에 있을 때 F키를 누르면 NarratorPanel을 활성화하고 지정 텍스트를 표시합니다.
벨 전용 모드 isBell = true
누른 횟수에 따라 다른 텍스트를 순서대로 표시합니다. bellTexts[] 배열로 Inspector에서 설정.
챕터별 텍스트
GameManager.Instance.chapter 값을 읽어 Ch.1이면 defaultText, Ch.2 이후면 ch2Text를 표시합니다.
단서 등록
상호작용 시 GameManager.AddClue(clueId)를 호출합니다. clueId가 비어 있으면 호출하지 않습니다.
프롬프트 표시
히키가 범위 내에 들어오면 promptObject ([F] 살펴보기)를 활성화, 벗어나면 비활성화합니다.

PlayerMovement.cs 수정

기존 스크립트에 이동 범위 제한 필드 추가

public bool  limitMovement = false;
public float minX, maxX, minY, maxY;

void Update()
{
    // ... 기존 이동 로직 ...
    transform.Translate(move);

    if (limitMovement)
    {
        Vector3 pos = transform.position;
        pos.x = Mathf.Clamp(pos.x, minX, maxX);
        pos.y = Mathf.Clamp(pos.y, minY, maxY);
        transform.position = pos;
    }
}

limitMovement를 Inspector에서 씬마다 개별 설정합니다. KomoriHouseScene처럼 고정 배경 씬에서는 활성화하고, 카메라가 따라다니는 씬에서는 비활성화합니다.

Part 3 — 트러블슈팅

트러블슈팅

카메라 줌 문제

증상
씬 재생 시 히키가 화면을 꽉 채워 배경이 보이지 않음
원인
Orthographic Size 기본값이 너무 작아 화면 범위가 좁음
해결
Main Camera → Inspector → Orthographic Size 값 증가. 배경 전체가 보일 때까지 조정.

Bell 위치 문제

증상
Bell 오브젝트에 가까이 가도 상호작용 반응 없음
원인
Bell Transform Position X = 15.21 — 화면 밖으로 배치되어 있었음
해결
Scene 뷰에서 현관문 앞으로 직접 드래그 후 Interact Distance를 5로 설정

BellPrompt 렌더링 문제

증상
[F] 살펴보기 텍스트가 투명한 검정색으로 보이거나, 흰색 배경이 생김
원인
3D TextMeshPro + URP 환경에서 MALGUN SDF Material의 Face Color(검정)가 Vertex Color와 곱해져 검정으로 출력됨. Face Color를 흰색으로 바꾸면 글자 외 영역까지 흰색으로 렌더링됨

시도한 방법들

방법 결과
3D TextMeshPro → World Space Canvas로 교체 실패
Material Face Color 흰색으로 변경 흰색 배경 문제 발생
Extract Material SubAsset / Ctrl+D 복제 Material Preset에 표시 안 됨
MALGUN_White SDF 새 폰트 에셋 생성 동일 문제 반복

현재 상태 — 기존 설정 복구 후 텍스트 화면 표시 중. URP 환경 TextMeshPro 렌더링 방식은 추후 확정 필요.
