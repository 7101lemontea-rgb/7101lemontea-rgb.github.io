01

프로젝트 개요

기본 정보

프로젝트명Flutter 기반 개인용 자동 동기화 가계부 Fortune Garden (포춘 가든)
대상 플랫폼Windows (Flutter Desktop) / Android (Flutter App)
핵심 기술Flutter, CODEF API, SQLite, Shelf (로컬 서버), Dart Isolate
개발 유형2인 가구용 공유 가계부 애플리케이션

프로젝트 목표

CODEF API를 활용하여 여러 금융기관(시중 은행, 토스뱅크, 저축은행 등)의 금융 데이터를 자동으로 수집하고, 사용자의 로컬 PC 환경에서 외부 서버 전송 없이 안전하게 자산을 관리할 수 있는 데스크탑 가계부 애플리케이션 개발을 목표로 한다.

  • →여러 금융기관 계좌를 단일 인터페이스에서 통합 조회
  • →사용자 2인 각자의 계좌를 개별 또는 합산하여 조회
  • →간편인증(카카오톡, PASS 등) 최초 1회 연동 후 자동 거래 내역 수집
  • →모든 데이터는 로컬 SQLite에 저장 — 개인 금융 정보 외부 유출 방지
  • →카테고리 자동 분류 및 지출 리포트로 재정 관리 지원

배경 및 필요성

현재 시장의 대부분 가계부 앱은 클라우드 기반으로 동작하여 금융 데이터가 외부 서버에 저장된다. 개인 금융 정보의 민감성을 고려할 때, 데이터가 로컬에만 보관되는 프라이버시 중심의 가계부가 필요했다. 또한 2인 가구의 특성을 반영해 두 명의 사용자가 공동 앱에서 각자의 금융 정보를 함께 관리할 수 있는 기능이 요구됐다.

용어 정의

Connected IDCODEF가 사용자의 금융기관 인증을 대리하여 발급하는 고유 식별자. 최초 1회 간편인증 후 발급되며, 이후 해당 ID로 반복 조회가 가능하다.
CODEF API한국의 금융기관 스크래핑 및 인증 중계 서비스. 은행, 카드사 등 다수 금융기관의 데이터를 표준화된 API로 제공한다.
Shelf ServerDart 기반 경량 HTTP 서버 패키지. 본 프로젝트에서 CODEF Secret Key 은닉 및 API 프록시 역할로 Flutter 앱 내에서 로컬 구동된다.
Dart IsolateDart의 경량 병렬 처리 단위. UI 스레드와 분리된 별도 Isolate에서 백그라운드 동기화를 수행하여 UI 블로킹을 방지한다.
Background Sync앱 실행 시 자동으로 전날 이후의 미수집 거래 내역을 가져오는 자동 동기화 프로세스.
프로필 (Profile)사용자 2인 각각의 사용자 단위. 각 프로필은 독립적인 Connected ID, 계좌 목록, 설정을 가진다.
통합 뷰두 프로필의 계좌와 거래 내역을 하나로 합산하여 보여주는 조회 모드.

05

시스템 아키텍처 (System Architecture)

5.1 계층형 아키텍처

본 시스템은 4개의 주요 계층으로 구성된다. 각 계층은 단방향 의존성을 유지하며, Presentation → Logic → Data → External 순으로 호출 흐름이 발생한다.

PresentationFlutter UI

대시보드, 거래 내역 리스트, 리포트, 인증 온보딩, 설정 화면. 상태 관리는 Riverpod 또는 BLoC 패턴 사용.

FlutterRiverpodBLoC
LogicShelf Server

CODEF API 토큰 갱신, API 프록시(Secret Key 은닉), 동기화 스케줄링, 카테고리 자동 분류 로직.

DartShelfIsolate
DataSQLite

거래 내역, 계정·기관 정보, 카테고리 규칙, 사용자 설정 저장. Repository 패턴으로 추상화.

driftsqfliteAES-256
ExternalCODEF API

금융기관 인증 중계, 거래 내역 및 잔액 스크래핑. REST API 방식으로 호출.

RESTOAuth 2.0JSON

5.2 데이터 흐름

동기화 흐름: 앱 시작 → Flutter UI가 Shelf 서버에 동기화 요청 → Shelf가 Connected ID로 CODEF API 호출 → 응답 데이터 파싱 후 중복 체크 → SQLite 저장 → 화면 갱신

인증 흐름: 간편인증 선택 → Flutter UI가 Shelf 서버에 인증 요청 → CODEF 인증 API 호출 → 사용자 카카오톡/PASS 인증 완료 → Connected ID 발급 → 암호화하여 SQLite 저장

5.3 SQLite 데이터베이스 스키마

테이블명 주요 컬럼 설명
profiles id, name, color, connected_id_enc, created_at 프로필
accounts profile_id, id, institution_code, account_number_enc, alias, balance, last_synced_at 연결된 금융기관 계좌 정보
transactions profile_id, id, account_id, txn_date, amount, type, merchant, balance, category_id, memo, is_manual 거래 내역
categories id, name, icon, color, is_custom, parent_id 카테고리 마스터
category_rules id, keyword, category_id, priority 거래처명 기반 자동 분류 규칙
sync_history id, started_at, completed_at, status, txn_count, error_message 동기화 이력
app_settings key, value, updated_at 앱 설정 키-값 저장소

07

외부 인터페이스 요구사항

7.1 CODEF API 연동

API 방식REST API (HTTPS), JSON 요청/응답
인증 방식OAuth 2.0 Client Credentials + Connected ID
주요 엔드포인트/v1/account/list · /v1/account/transaction-list · /v1/account/balance
오류 처리HTTP 4xx/5xx 코드별 재시도 로직. 429(Rate Limit) 시 지수 백오프(Exponential Backoff)
Secret Key 관리앱 번들 미포함, Shelf 서버 환경 변수 또는 암호화 설정 파일에 저장

7.2 OS 통합

Windows 시스템 트레이 아이콘 표시 및 우클릭 메뉴(앱 열기/동기화/종료) 제공. 동기화 완료 및 새 거래 감지 시 OS 네이티브 알림 표시. 향후 확장을 위해 budget-app:// custom URI scheme 등록 고려.

제약사항 및 가정

기술적 제약사항

  • →Flutter 3.x 이상, Dart 3.x 이상 환경 필요
  • →CODEF API는 유료 서비스, API 호출 건수 및 기관 수에 따라 비용 발생
  • →금융기관 점검 시간(새벽 00:00~01:00)에는 동기화 실패 가능
  • →2인 프로필 동시 동기화 시 SQLite DB 잠금 처리 방식 적용 필요
