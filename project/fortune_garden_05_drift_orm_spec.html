Section 01

문서 개요

1.1 목적

본 문서는 Fortune Garden 애플리케이션의 데이터 영속성 계층(L4)을 구현하기 위해 Drift ORM 기반 SQLite 스키마를 코드 수준으로 명세한다. ERD v1.0에서 정의된 8개 테이블을 Dart 클래스로 변환한 결과물과 구현 과정에서 적용된 기술적 결정 사항을 기록한다.

1.2 범위

본 문서가 다루는 구현 범위는 다음과 같다.

  • 8개 Drift 테이블 클래스 정의 (tables.dart)
  • AppDatabase 메인 클래스 및 핵심 쿼리 4개 (app_database.dart)
  • Riverpod Provider 연동 (database_provider.dart)
  • CODEF 금융기관 Seed Data 20개 기관 (seed_institutions.dart)
  • ERD 7절 기본 카테고리 11개 및 앱 설정 8개 Seed data

제외 범위: Repository 레이어, UseCase, Presentation 계층, 암호화 로직(flutter_secure_storage)

Section 03

테이블 구현 명세 tables.dart

ERD v1.0의 8개 테이블을 Drift Table 클래스로 구현한 결과를 기술한다. 컬럼명은 lowerCamelCase Dart 규칙을 따르며, Drift ORM이 snake_case SQL 컬럼명으로 자동 변환한다.

3.1 테이블 목록

No 테이블명 Drift 클래스 역할 요약
1 profiles Profiles 사용자 프로필 최대 2개. 모든 데이터의 루트 엔티티
2 institutions Institutions CODEF 금융기관 코드 마스터. code가 PK(TEXT)
3 accounts Accounts 금융기관 계좌. 계좌번호 AES-256-GCM 암호화 저장
4 categories Categories 카테고리 마스터 11개 기본값. 자기 참조 계층 구조
5 transactions Transactions 거래 내역 핵심 테이블. txn_hash UNIQUE로 중복 방지
6 category_rules CategoryRules 거래처명 키워드 → 카테고리 자동 분류 규칙
7 sync_history SyncHistory 동기화 이력 로그. 상태 4단계 관리
8 app_settings AppSettings 앱 전역 설정 Key-Value 저장소. key가 PK(TEXT)

3.2 테이블별 컬럼 명세

3.2.1 profiles — 사용자 프로필

2인 가구의 각 사용자 프로필을 저장하는 루트 엔티티. id는 1 또는 2로 운용.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 프로필 고유 ID (1 또는 2)
name TextColumn — NOT NULL 프로필 표시 이름
colorHex TextColumn — NOT NULL 프로필 구분 색상 HEX (예: #2E6DA4)
connectedIdEnc BlobColumn — nullable AES-256-GCM 암호화된 CODEF Connected ID
defaultViewMode TextColumn — DEFAULT 'personal' 'personal' | 'combined'
createdAt IntColumn — NOT NULL 생성 시각 Unix timestamp (ms)
3.2.2 institutions — 금융기관 코드 마스터

CODEF 지원 금융기관 코드를 DB로 관리. code 컬럼이 TEXT PK. is_active로 비활성화 처리.

컬럼명 Dart 타입 PK/FK 제약 설명
code TextColumn PK NOT NULL CODEF 금융기관 코드 (예: 0010 = KB국민은행)
name TextColumn — NOT NULL 기관명
type TextColumn — NOT NULL 'bank' | 'card' | 'stock'
isActive IntColumn — DEFAULT 1 0이면 신규 연동 불가. ON DELETE RESTRICT 대체
3.2.3 accounts — 금융기관 계좌

프로필별 금융 계좌 정보 저장. 계좌번호는 AES-256-GCM으로 암호화된 BLOB으로 저장.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 계좌 고유 ID
profileId IntColumn FK CASCADE profiles.id 참조. 프로필 삭제 시 계좌 삭제
institutionCode TextColumn FK RESTRICT institutions.code 참조. 기관 삭제 불가
accountNumberEnc BlobColumn — NOT NULL AES-256-GCM 암호화된 계좌번호 BLOB
alias TextColumn — nullable 사용자 지정 계좌 별명
balance IntColumn — DEFAULT 0 최종 동기화 잔액 (원 단위 정수)
lastSyncedAt IntColumn — nullable 마지막 동기화 Unix timestamp (ms)
3.2.4 categories — 카테고리 마스터

지출 카테고리 11개 기본값 포함. parent_id 자기 참조로 계층 구조 지원. Drift 자기참조 제약 특성상 쿼리 레벨에서 JOIN 처리.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 카테고리 고유 ID
name TextColumn — UNIQUE 카테고리 이름 (예: 식비, 교통비)
icon TextColumn — nullable Flutter Icons 코드포인트 문자열
colorHex TextColumn — nullable 카테고리 색상 HEX
isCustom IntColumn — DEFAULT 0 0: 기본 11개, 1: 사용자 정의
parentId IntColumn FK(self) nullable 자기 참조. 상위 카테고리 삭제 시 SET NULL
3.2.5 transactions — 거래 내역

거래 데이터 핵심 테이블. txn_hash UNIQUE 제약으로 동기화 중복 자동 방지. amount 음수=지출, 양수=수입.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 거래 고유 ID
profileId IntColumn FK CASCADE profiles.id. 성능 역정규화 컬럼
accountId IntColumn FK CASCADE accounts.id. 계좌 삭제 시 거래도 삭제
txnHash TextColumn — UNIQUE SHA-256(account_id+date+amount+merchant)
txnDate IntColumn — NOT NULL 거래 일시 Unix timestamp (ms)
amount IntColumn — NOT NULL 금액(원). 지출 음수(−), 수입 양수(+)
merchant TextColumn — NOT NULL 거래처명 (CODEF 원문)
balanceAfter IntColumn — nullable 거래 후 계좌 잔액
categoryId IntColumn FK SET NULL categories.id. NULL = 미분류
memo TextColumn — nullable 사용자 메모
isManual IntColumn — DEFAULT 0 0: 자동수집, 1: 수동입력
createdAt IntColumn — NOT NULL 레코드 생성 Unix timestamp (ms)
3.2.6 category_rules — 자동 분류 규칙

거래처명 키워드 LIKE 검색으로 카테고리를 자동 분류. profile_id=NULL은 두 프로필 공용 규칙.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 규칙 고유 ID
keyword TextColumn — NOT NULL 거래처명 매칭 키워드 (LIKE 검색)
categoryId IntColumn FK CASCADE categories.id. 카테고리 삭제 시 규칙도 삭제
profileId IntColumn FK SET NULL profiles.id. NULL = 공용 규칙
priority IntColumn — DEFAULT 0 높을수록 먼저 적용
3.2.7 sync_history — 동기화 이력

동기화 시작/완료 시각, 상태, 수집 건수, 오류 메시지를 기록. completedAt=NULL은 진행 중 상태.

컬럼명 Dart 타입 PK/FK 제약 설명
id IntColumn PK AUTOINCREMENT 이력 고유 ID
profileId IntColumn FK CASCADE profiles.id 참조
startedAt IntColumn — NOT NULL 동기화 시작 Unix timestamp (ms)
completedAt IntColumn — nullable NULL = 진행 중. 완료 시 timestamp 기입
status TextColumn — NOT NULL 'running' | 'success' | 'partial' | 'failed'
txnCount IntColumn — DEFAULT 0 수집된 거래 건수
errorMsg TextColumn — nullable 오류 메시지 (status='failed'일 때)
3.2.8 app_settings — 앱 전역 설정

Key-Value 저장소. key가 TEXT PK. value는 JSON 직렬화 허용. 8개 기본값 Seed data 포함.

컬럼명 Dart 타입 PK/FK 제약 설명
key TextColumn PK NOT NULL 설정 키 (예: theme, auto_sync_enabled)
value TextColumn — nullable 설정 값. JSON 직렬화 허용
updatedAt IntColumn — NOT NULL 최종 수정 Unix timestamp (ms)
Section 04

인덱스 구현 전략

ERD 5절의 인덱스 전략을 Drift @Index 어노테이션으로 구현하였다. 총 8개 인덱스가 정의되며, 대부분 transactions 테이블에 집중된다.

테이블 인덱스명 유형 목적
transactions idx_txn_profile_date UNIQUE 개인·통합 뷰 날짜순 조회. profile_id 필터 + txn_date 정렬 동시 처리
transactions idx_txn_account_date 일반 계좌별 거래 내역 조회 (계좌 필터 화면)
transactions idx_txn_category 일반 카테고리별 필터 조회 및 리포트 집계 (GROUP BY)
transactions idx_txn_hash UNIQUE 동기화 중복 체크. INSERT OR IGNORE 활용
transactions idx_txn_date 일반 통합 뷰 전체 날짜순 정렬
accounts idx_acc_profile 일반 프로필별 계좌 목록 조회
category_rules idx_rule_keyword 일반 거래처명 LIKE 검색 카테고리 자동 분류
sync_history idx_sync_profile_start 일반 마지막 동기화 시각 조회 (앱 시작 시 기준일 계산)
Section 05

핵심 쿼리 구현 app_database.dart

ERD 6절에 정의된 4개 핵심 쿼리를 AppDatabase 메서드로 구현하였다. customSelect()를 사용하여 SQL 의사 코드를 Dart로 직접 표현한다.

5.1 개인 뷰 — 월별 수입/지출 합계

메서드
getMonthlySummary({profileId, monthStart, monthEnd})
반환 타입
({int income, int expense})
설명
해당 프로필의 지정 월 전체 거래를 SUM 집계하여 수입/지출을 분리 반환한다. 
SELECT
  SUM(CASE WHEN amount > 0 THEN amount ELSE 0 END) AS income,
  SUM(CASE WHEN amount < 0 THEN amount ELSE 0 END) AS expense
FROM transactions
WHERE profile_id = :profileId
  AND txn_date >= :monthStart
  AND txn_date < :monthEnd

5.2 통합 뷰 — 두 프로필 합산 대시보드

메서드
getCombinedDashboard({monthStart, monthEnd})
반환 타입
List<QueryRow>
설명
두 프로필의 수입/지출을 profile_id로 GROUP BY하여 각 프로필명, 색상과 함께 반환한다. 
SELECT p.name, p.color_hex,
  SUM(CASE WHEN t.amount > 0 THEN t.amount ELSE 0 END) AS income,
  SUM(CASE WHEN t.amount < 0 THEN t.amount ELSE 0 END) AS expense
FROM transactions t
JOIN profiles p ON t.profile_id = p.id
WHERE t.txn_date >= :monthStart
  AND t.txn_date < :monthEnd
GROUP BY t.profile_id

5.3 카테고리 자동 분류

메서드
matchCategory({merchant, profileId})
반환 타입
Future<int?>
설명
거래처명을 category_rules 테이블에서 LIKE 검색. 개인 규칙 우선, 공용 규칙 폴백. 
SELECT category_id
FROM category_rules
WHERE :merchant LIKE '%' || keyword || '%'
  AND (profile_id = :profileId OR profile_id IS NULL)
ORDER BY (profile_id IS NOT NULL) DESC, priority DESC
LIMIT 1

5.4 중복 거래 방지 INSERT

메서드
insertTransactionIfNotExists(TransactionsCompanion entry)
설명
txn_hash UNIQUE 제약을 활용하여 insertOnConflictUpdate()로 중복 거래를 자동 무시. 동기화 시 매 거래에 적용. 
INSERT OR IGNORE INTO transactions
  (profile_id, account_id, txn_hash, txn_date, amount, merchant, ...)
  VALUES
  (:profileId, :accountId, :hash, :date, :amount, :merchant, ...);

-- txn_hash UNIQUE 제약에 의해 중복 자동 무시

패키지명 버전 구분 용도
drift ^2.18.0 runtime Drift ORM 핵심 라이브러리

Section 08

기술 결정 사항 및 주의사항

자기 참조 FK
(categories)
Drift는 자기 참조 FK 어노테이션을 미지원. parentId를 일반 IntColumn으로 선언하고, 쿼리 레벨에서 수동 JOIN으로 처리
암호화 컬럼 (BLOB)
connectedIdEnc, accountNumberEnc는 BlobColumn으로 선언. 실제 AES-256-GCM 암호화/복호화는 flutter_secure_storage + encrypt 패키지에서 별도 처리
app_database.g.dart
build_runner 자동 생성 파일. 직접 수정 금지. .gitignore에 *.g.dart 추가하여 형상 관리 제외
institutions.code PK
CODEF 코드는 TEXT 타입 PK. Drift에서 Set<Column> get primaryKey 오버라이드로 구현
app_settings.key PK
앱 설정도 TEXT 타입 PK. 동일하게 primaryKey 오버라이드 적용
Unix timestamp (ms)
모든 날짜/시각 컬럼은 IntColumn으로 Unix timestamp(밀리초) 저장. DateTime 변환은 Repository 레이어에서 처리
amount 부호 규칙
지출은 음수(−), 수입은 양수(+). UI 렌더링 시 절댓값 표시는 Presentation 레이어 책임
insertOnConflictUpdate
중복 거래 방지에 사용. txn_hash 충돌 시 기존 레코드 유지(무시) 동작
