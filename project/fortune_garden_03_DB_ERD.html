03

테이블 간 관계 명세

아래 표는 모든 외래 키 관계의 Cardinality, 삭제/수정 정책, 비즈니스 의미를 정의합니다.

부모 테이블 관계 자식 테이블 (FK) 삭제 정책 비즈니스 의미
profiles (id) 1 : N accounts (profile_id) CASCADE 프로필 삭제 시 해당 프로필의 모든 계좌 자동 삭제.
profiles (id) 1 : N transactions (profile_id) CASCADE 성능을 위한 역정규화. 프로필 삭제 시 거래도 삭제.
profiles (id) 1 : N import_history (profile_id) NEW CASCADE 프로필 삭제 시 가져오기 이력도 삭제.
profiles (id) 1 : N (opt) category_rules (profile_id) SET NULL 프로필 삭제 시 해당 규칙은 공용 규칙(NULL)으로 전환.
accounts (id) 1 : N transactions (account_id) CASCADE 계좌 삭제 시 해당 계좌의 모든 거래 자동 삭제.
accounts (id) 1 : N import_history (account_id) NEW CASCADE 계좌 삭제 시 해당 계좌의 가져오기 이력도 삭제.
categories (id) 1 : N (opt) transactions (category_id) SET NULL 카테고리 삭제 시 해당 거래는 '미분류'로 전환.
categories (id) 1 : N (opt) category_rules (category_id) CASCADE 카테고리 삭제 시 해당 분류 규칙도 삭제.
categories (id) 1 : N (opt, self) categories (parent_id) SET NULL 상위 카테고리 삭제 시 하위는 최상위 카테고리로 전환.
institutions (code) 1 : N accounts (institution_code) RESTRICT 계좌가 연결된 기관은 삭제 불가. is_active=0으로 비활성화.
institutions (code) 1 : N csv_parser_profiles (institution_code) NEW RESTRICT 파서 프로필이 연결된 기관은 삭제 불가. is_active=0으로 비활성화.

테이블 상세 스키마

각 테이블의 컬럼 명세를 상세히 정의합니다. PK(기본 키)는 적색, FK(외래 키)는 청색으로 표시합니다.

profiles 사용자 프로필
컬럼명 타입 Key 제약 설명

name TEXT — NOT NULL 프로필 표시 이름
color_hex TEXT — NOT NULL 프로필 구분 색상 HEX (예: #2E6DA4)
default_view_mode TEXT — NOT NULL / DEFAULT 'personal' 기본 뷰 모드: 'personal' | 'combined'
created_at INTEGER — NOT NULL 생성 시각 Unix timestamp (ms)

05

인덱스 전략

조회 성능에 직접 영향을 주는 쿼리 패턴을 분석하여 아래 인덱스를 정의합니다. drift ORM은 @Index 어노테이션으로 자동 생성합니다.

테이블 인덱스명 컬럼 (순서 중요) 유형 사용 패턴 및 근거
transactions idx_txn_profile_date profile_id, txn_date DESC UNIQUE 개인 뷰/통합 뷰에서 날짜순 조회 시 최다 사용. 복합 인덱스로 profile_id 필터 + 날짜 정렬 동시 처리.
transactions idx_txn_account_date account_id, txn_date DESC 일반 계좌별 거래 내역 조회 (SCR-003 계좌 필터).
transactions idx_txn_category category_id 일반 카테고리별 필터 조회 및 리포트 집계 (GROUP BY category_id).
transactions idx_txn_hash txn_hash UNIQUE CSV 가져오기 시 중복 거래 체크. INSERT OR IGNORE로 활용.
transactions idx_txn_date txn_date DESC 일반 통합 뷰에서 날짜순 전체 정렬 시 사용.
accounts idx_acc_profile profile_id 일반 프로필별 계좌 목록 조회 (SCR-007).
category_rules idx_rule_keyword keyword 일반 거래처명 LIKE 검색으로 카테고리 자동 분류 시 사용.
import_history idx_import_account_date account_id, imported_at DESC 일반 계좌별 가져오기 이력 조회 (SCR-007 ImportHistoryList). NEW
csv_parser_profiles idx_parser_institution institution_code 일반 institution_code 기준 파서 프로필 자동 매칭 시 사용 (SCR-008). NEW

06

주요 쿼리 패턴

drift ORM으로 작성할 핵심 쿼리의 SQL 의사 코드를 정의합니다. 실제 구현은 drift Query Builder로 변환합니다.

6.1 개인 뷰 — 이번 달 수입/지출 합계

SQL Pseudocode

SELECT
  SUM(CASE WHEN amount > 0 THEN amount ELSE 0 END) AS income,
  SUM(CASE WHEN amount < 0 THEN amount ELSE 0 END) AS expense
FROM transactions
WHERE profile_id = :profileId               -- 개인 뷰
  AND txn_date >= :monthStart
  AND txn_date <  :monthEnd;

6.2 통합 뷰 — 두 프로필 합산 대시보드

SQL Pseudocode

SELECT
  p.name, p.color_hex,
  SUM(CASE WHEN t.amount > 0 THEN t.amount ELSE 0 END) AS income,
  SUM(CASE WHEN t.amount < 0 THEN t.amount ELSE 0 END) AS expense
FROM transactions t
JOIN profiles p ON t.profile_id = p.id      -- 통합 뷰: profile_id 조건 없음
WHERE t.txn_date >= :monthStart AND t.txn_date < :monthEnd
GROUP BY t.profile_id;

6.3 카테고리 자동 분류

SQL Pseudocode

SELECT category_id FROM category_rules
WHERE :merchant LIKE '%' || keyword || '%'
  AND (profile_id = :profileId OR profile_id IS NULL)
  -- 개인 규칙 우선, 공용 규칙 폴백
ORDER BY (profile_id IS NOT NULL) DESC, priority DESC
LIMIT 1;

6.4 중복 거래 방지 (CSV 가져오기 시)

SQL Pseudocode

INSERT OR IGNORE INTO transactions
  (profile_id, account_id, txn_hash, txn_date, amount, merchant, ...)
VALUES (:profileId, :accountId, :hash, :date, :amount, :merchant, ...);
-- txn_hash UNIQUE 제약에 의해 중복 자동 무시

6.5 파서 프로필 자동 매칭 NEW

SQL Pseudocode

SELECT * FROM csv_parser_profiles
WHERE institution_code = :institutionCode
  AND is_active = 1
LIMIT 1;
-- SCR-008 CSV 가져오기 화면에서 계좌 선택 시 자동 호출

6.6 계좌별 가져오기 이력 조회 NEW

SQL Pseudocode

SELECT * FROM import_history
WHERE account_id = :accountId
ORDER BY imported_at DESC
LIMIT 20;
-- SCR-007 계좌 관리 화면 ImportHistoryList에서 사용
