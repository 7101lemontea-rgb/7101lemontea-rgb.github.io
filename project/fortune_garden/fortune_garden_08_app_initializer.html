01

작업 배경 및 목적

Drift ORM 구현(app_database.dart) 완료 후, 앱 시작 시 금융기관 Seed Data(seedInstitutions)와 CSV 파서 프로필 Seed Data(seedCsvParserProfiles)를 삽입하는 초기화 코드가 필요해졌습니다. 이를 main.dart에 직접 작성하면 진입점 파일이 복잡해지므로, 역할별로 파일을 분리하는 구조를 설계하였습니다.

결정 사항 내용
main.dart 역할 ProviderScope 래핑 + MyApp 호출만 담당. 초기화 로직 없음
app.dart 역할 MyApp 위젯 정의 + initializeApp() 호출 + 로딩 처리
app_initializer.dart seedInstitutions() + seedCsvParserProfiles() 실행
선택 이유 main.dart를 단순하게 유지. 초기화 로직 교체·테스트 용이

02

파일 구조 및 역할

3개 파일은 lib/ 루트에 위치하며, data/database/ 하위 파일들과 의존 관계를 가집니다.

lib/
├── main.dart # 수정 — ProviderScope + MyApp 호출만
├── app.dart # 신규 — MyApp 위젯 + initializeApp() 호출
├── app_initializer.dart # 신규 — Seed Data 삽입 초기화 함수
└── data/
    └── database/
        ├── app_database.dart
        ├── database_provider.dart ← app_initializer가 참조
        └── seed_institutions.dart ← app_initializer가 참조

파일별 역할

수정 main.dart 앱 진입점
WidgetsFlutterBinding.ensureInitialized() 호출 후 ProviderScope로 MyApp을 감싸 실행. 초기화·UI 로직 없음. MyApp과 MyHomePage 클래스를 app.dart로 이동.
신규 app.dart 루트 위젯 + 초기화 오케스트레이터
ConsumerStatefulWidget으로 구현. initState()에서 _init()을 호출해 initializeApp()을 실행. Seed 완료 전 로딩 화면, 완료 후 MaterialApp 렌더링. 기존 main.dart의 MyApp·MyHomePage 클래스를 이동.
신규 app_initializer .dart 앱 초기화 함수 모음
initializeApp(WidgetRef ref) 함수 하나를 export. appDatabaseProvider로 DB 인스턴스를 획득한 후 seedInstitutions(), seedCsvParserProfiles()를 순서대로 실행. 향후 초기화 항목 추가 시 이 파일에만 추가.

03

실행 흐름

main()
└── WidgetsFlutterBinding.ensureInitialized()
└── runApp(ProviderScope(child: MyApp()))
└── MyApp.initState()
    └── _init()
        └── initializeApp(ref)
            ├── appDatabaseProvider → AppDatabase 인스턴스 획득
            ├── seedInstitutions(db) // 기관 20개 upsert
            └── seedCsvParserProfiles(db) // 파서 프로필 7개 upsert
        └── _initialized = true
    └── MyHomePage 렌더링
단계 주체 설명
1 main() ensureInitialized() — Flutter 엔진 바인딩 초기화. DB 파일 I/O 등 비동기 작업 전에 반드시 필요
2 main() ProviderScope — Riverpod의 필수 최상위 래퍼. 모든 Provider가 이 범위 안에서 동작
3 MyApp initState() → _init() — ConsumerStatefulWidget 생명주기. 위젯 트리 생성 직후 1회 실행
4 app_initializer initializeApp(ref) — WidgetRef로 appDatabaseProvider에서 DB 인스턴스 획득 후 Seed 실행
5 seed_institutions insertAllOnConflictUpdate() — 중복 실행 안전. 앱 재시작 시에도 기존 데이터 유지하며 upsert
6 MyApp _initialized = true → setState() — 로딩 화면에서 MyHomePage로 전환

04

파일별 전체 코드

4.1 main.dart

변경 전 대비 추가된 부분: flutter_riverpod import, app.dart import, WidgetsFlutterBinding.ensureInitialized(), ProviderScope 래핑이 추가되었습니다. MyApp·MyHomePage 클래스는 app.dart로 이동하여 삭제했습니다. 

// lib/main.dart

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

import 'app.dart';

void main() {
  WidgetsFlutterBinding.ensureInitialized();
  runApp(
    const ProviderScope(
      child: MyApp(),
    ),
  );
}

4.2 app.dart

기존 main.dart의 MyApp + MyHomePage를 이동했습니다. MyApp을 ConsumerStatefulWidget으로 변경하여 initializeApp() 호출 및 로딩 처리를 추가하였습니다. 이후 화면이 추가될 때 MaterialApp의 home: 또는 routerConfig: 설정을 이 파일에서 관리합니다. 

// lib/app.dart

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

import 'app_initializer.dart';

class MyApp extends ConsumerStatefulWidget {
  const MyApp({super.key});

  @override
  ConsumerState<MyApp> createState() => _MyAppState();
}

class _MyAppState extends ConsumerState<MyApp> {
  bool _initialized = false;

  @override
  void initState() {
    super.initState();
    _init();
  }

  Future<void> _init() async {
    await initializeApp(ref);
    if (mounted) setState(() => _initialized = true);
  }

  @override
  Widget build(BuildContext context) {
    if (!_initialized) {
      return const MaterialApp(
        debugShowCheckedModeBanner: false,
        home: Scaffold(
          body: Center(child: CircularProgressIndicator()),
        ),
      );
    }

    return MaterialApp(
      debugShowCheckedModeBanner: false,
      title: 'Fortune Garden',
      theme: ThemeData(
        useMaterial3: true,
        colorSchemeSeed: Colors.green,
      ),
      home: const MyHomePage(),
    );
  }
}

class MyHomePage extends StatefulWidget {
  const MyHomePage({super.key});

  @override
  State<MyHomePage> createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('Fortune Garden'),
        centerTitle: true,
      ),
      body: const Center(
        child: Text(
          '빈 페이지',
          style: TextStyle(
            fontSize: 18,
            color: Color.fromARGB(255, 97, 126, 102),
          ),
        ),
      ),
    );
  }
}

4.3 app_initializer.dart

앱 시작 시 1회 실행되는 초기화 함수입니다. WidgetRef를 파라미터로 받아 appDatabaseProvider에서 DB 인스턴스를 획득한 후 Seed 함수를 순서대로 호출합니다. 

// lib/app_initializer.dart

import 'package:flutter_riverpod/flutter_riverpod.dart';

import 'data/database/database_provider.dart';
import 'data/database/seed_institutions.dart';

/// 앱 시작 시 1회 실행되는 초기화 함수.
/// WidgetRef를 통해 Provider에 접근한다.
Future<void> initializeApp(WidgetRef ref) async {
  final db = ref.read(appDatabaseProvider);

  // 금융기관 코드 20개 upsert (중복 실행 안전)
  await seedInstitutions(db);

  // CSV 파서 프로필 기본값 upsert (중복 실행 안전)
  await seedCsvParserProfiles(db);
}

07

주의사항 및 향후 확장 가이드

7.1 주의사항

  • WidgetsFlutterBinding.ensureInitialized()는 runApp() 이전에 반드시 호출해야 합니다. DB 파일 경로 획득 (path_provider)이 Flutter 엔진 초기화 완료 후에만 동작하기 때문입니다.
  • initializeApp()은 WidgetRef를 파라미터로 받으므로 ConsumerWidget 계열(ConsumerStatefulWidget)에서만 호출 가능합니다. 일반 StatelessWidget에서는 호출할 수 없습니다.
  • seedInstitutions()와 seedCsvParserProfiles()는 모두 insertAllOnConflictUpdate() 기반이므로 앱 재시작 시 중복 삽입 없이 안전하게 재실행됩니다.
  • _initialized 플래그가 false인 동안 로딩 화면이 표시됩니다. Seed 데이터 양이 적어 체감 시간은 거의 없습니다 (수십 ms 수준).

7.2 초기화 항목 추가 방법

앱 시작 시 추가로 초기화해야 할 작업이 생기면 app_initializer.dart에만 추가하면 됩니다. main.dart와 app.dart는 수정하지 않아도 됩니다. 

// app_initializer.dart에 추가 예시
Future<void> initializeApp(WidgetRef ref) async {
  final db = ref.read(appDatabaseProvider);

  await seedInstitutions(db);
  await seedCsvParserProfiles(db);

  // 향후 추가 예시:
  // await migrateLocalSettings(db);   // 설정 마이그레이션
  // await warmUpCache(ref);           // 캐시 워밍업
}

7.3 라우터 도입 시 수정 위치

go_router 등 라우터를 도입하거나 홈 화면이 변경될 때 app.dart의 MaterialApp 설정만 수정하면 됩니다. 

// app.dart — go_router 도입 예시
return MaterialApp.router(
  debugShowCheckedModeBanner: false,
  title: 'Fortune Garden',
  theme: ThemeData(
    useMaterial3: true,
    colorSchemeSeed: Colors.green,
  ),
  routerConfig: appRouter, // go_router 인스턴스
);
