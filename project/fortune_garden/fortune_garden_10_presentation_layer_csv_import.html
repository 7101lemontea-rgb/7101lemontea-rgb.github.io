01

작업 개요

오늘 작업은 두 파트로 진행되었습니다. 첫째로 SAD v1.1 §4.1 기준 Presentation Layer 전체를 구현하였고, 둘째로 CSV 가져오기 핵심 플로우의 미완성 부분(EUC-KR 인코딩, 임시 계좌 Seed, import_screen 개선, csv 패키지 오류 수정)을 완성하였습니다.

항목 내용
Presentation Layer app_providers, app_router, app_shell + SCR-001~SCR-011 11개 화면
app.dart 교체 MaterialApp → MaterialApp.router (go_router 연결)
CSV 플로우 charset_converter EUC-KR, 임시 계좌 Seed, import_screen 개선
오류 수정 CsvToListConverter 없는 클래스 오류 → 순수 Dart RFC4180 파서로 교체
빌드 오류 Windows 보안 정책 차단 → 해결 방법 안내

Presentation Layer

SAD v1.1 §4.1.1의 5개 Provider + ImportState 모델을 구현하였습니다.

Provider 이름 상태 타입 역할

Provider		 FutureProvider
<Profile?>		 현재 활성 프로필. 라우터 분기 기준

3.2 app_router.dart — go_router 라우팅

SAD v1.1 §5.1 앱 시작 시퀀스 기준으로 작성했습니다.
프로필 0개 → /setup, 1개 이상 → /dashboard 자동 분기. 

redirect: (context, state) async {
  final activeProfile = await ref.read(activeProfileProvider.future);
  final isSetup = state.matchedLocation == '/setup';

  if (activeProfile == null && !isSetup) return '/setup';
  if (activeProfile != null && isSetup) return '/dashboard';
  return null;
}
화면 ID 라우트 구성
SCR-001 /setup ShellRoute 밖 단독 표시. 프로필 설정 완료 후 /dashboard로 이동
SCR-002 /dashboard SummaryCard + ViewModeToggle + RecentTxnList
SCR-003 /transactions TransactionListView. ShellRoute 하위
SCR-004 /transactions/:id TxnDetailCard + CategorySelector + MemoField
SCR-005 /transactions/new ManualTxnForm
SCR-006 /report MonthlySummary + 프로필별 분석. 월 선택 가능
SCR-007 /accounts AccountListTile + 잔액 표시
SCR-008 /import FilePicker + 파서 상태 아이콘 + ImportResultCard
SCR-009 /categories CategoryGrid + 색상 아이콘 표시
SCR-010 /settings ThemeToggle + 주요 화면 링크
SCR-011 /profiles ProfileCard×2 + 이름/색상 수정

3.3 app_shell.dart — 플랫폼별 네비게이션

화면 너비 720px 기준으로 Desktop(NavigationRail)과 Mobile(BottomNavigationBar)을 자동 전환합니다.

≥ 720px Windows Desktop

NavigationRail 사이드바. labelType: all. VerticalDivider로 구분

< 720px Android Mobile

NavigationBar (Material 3). BottomNavigationBar 대체

공통 네비게이션 항목

대시보드, 거래내역, 리포트, 계좌, 설정 — 5개

3.4 app.dart 교체

기존 MaterialApp → MaterialApp.router로 교체하여 go_router를 연결하였습니다. 

// 변경 전
return MaterialApp(home: const MyHomePage(), ...);

// 변경 후
final router = ref.watch(appRouterProvider);
return MaterialApp.router(routerConfig: router, ...);

04

CSV 가져오기 핵심 플로우

4.1 seed_dev_data.dart — 개발용 임시 계좌

CSV 가져오기 테스트를 위해 kDebugMode 전용 임시 데이터를 생성했습니다. 이미 데이터가 있으면 자동으로 건너뜁니다 (idempotent).

프로필 계좌 별명 기관 코드 비고
프로필 1 (나) KB 주거래 KB 잔액 1,500,000원
프로필 1 (나) 신한카드 SHINHAN_CARD 잔액 0원
프로필 2 (파트너) NH 급여 NH 잔액 2,300,000원
프로필 2 (파트너) KB카드 KB_CARD 잔액 0원

계좌번호 암호화: 개발용이므로 utf8.encode('DEV:계좌번호') placeholder 사용. 실제 운영 시 flutter_secure_storage + encrypt 패키지로 AES-256-GCM 암호화 필요.

4.2 app_initializer.dart 업데이트

kDebugMode 플래그로 seedDevData() 호출을 분기하였습니다. 

import 'package:flutter/foundation.dart';

Future<void> initializeApp(WidgetRef ref) async {
  final db = ref.read(appDatabaseProvider);
  await seedInstitutions(db);
  await seedCsvParserProfiles(db);

  if (kDebugMode) {
    await seedDevData(db); // 개발 모드 전용 임시 계좌
  }
}

4.3 csv_parser_use_case.dart — EUC-KR 완성 + 파서 교체

두 차례 오류를 거쳐 최종적으로 csv 패키지 의존성을 제거하고 순수 Dart RFC 4180 파서로 교체하였습니다.

EUC-KR 디코딩

charset_converter.decode(encoding, bytes)로 완성. UTF-8은 dart:convert 직접 처리

delimiter 적용

콤마: csv 패키지(RFC 4180) → 순수 Dart 교체 후 내장 파서에서 처리

금액 부호 처리

△·▲(출금 표기), 괄호 (12,345), 음수 -12,345 모두 대응

csv 패키지 제거

CsvToListConverter 클래스명 오류로 의존성 제거. _parseCsv() 내장 구현

  // 빈 행 건너뜀
}

4.4 import_screen.dart — SCR-008 개선

모든 프로필의 계좌를 한 화면에서 선택할 수 있도록 개선하고, 파서 프로필 등록 여부를 아이콘으로 표시하였습니다.

계좌 목록 통합

단일 프로필 → 모든 프로필의 계좌 표시. _importAccountsProvider 신규

파서 상태 아이콘

파서 프로필 있음: 초록 체크. 없음: 주황 경고 아이콘

파서 없음 경고

파서 미등록 계좌 선택 시 경고 배너 표시

계좌 없음 안내

등록 계좌 없을 때 안내 화면 표시 (개발 모드 재시작 안내 포함)

결과 카드

신규/중복/오류 건수 표시. 완료 후 "다른 파일 가져오기" 버튼

05

발생한 오류 및 해결 내역

⚠ CsvToListConverter isn't defined for the type 'CsvParserUseCase'

원인

CsvToListConverter()를 non-const 생성자로 호출하면서 fieldDelimiter에 String을 직접 전달. csv 패키지 버전에 따라 파라미터 타입 불일치.

해결

csv 패키지 의존성 완전 제거. 순수 Dart RFC 4180 기반 _parseCsv() 메서드 내장 구현으로 교체. flutter pub remove csv 실행.


⚠ CsvToListConverter isn't a class (2차)

원인

1차 수정 후에도 동일 오류 지속. csv 패키지가 해당 버전에서 CsvToListConverter를 export하지 않는 것으로 확인.

해결

csv import 완전 제거. _parseCsv() 내장 파서 확정. charset_converter, intl만 유지.


⚠ Unable to start executable — 애플리케이션 제어 정책에서 이 파일을 차단

원인

Flutter 코드 문제 아님. Windows Defender 스마트 스크린 또는 WDAC/AppLocker가 서명되지 않은 로컬 빌드 exe 실행 차단.

해결

Windows 보안 → 제외 항목에 C:\workspace\fortune_garden\build 추가.

패키지 변경 이력

패키지 구분 버전 비고

flutter pub remove csv
