02 · Problem

문제 인식 및 배경

2인 가구의 가계부는 개인 지출과 공동 지출을 동시에 추적해야 합니다. 엑셀은 이 구조에 맞게 설계되지 않아 관리가 점점 번거로워졌습니다.

PROB.01
낮은 접근성
엑셀 파일을 열고 수식을 유지하며 입력하는 과정이 번거롭고, 모바일에서는 사실상 사용 불가능했습니다.
PROB.02
2인 구조 미지원
엑셀 시트는 개인 지출과 통합 지출을 한 화면에서 구분하기 어려워 매번 필터링이 필요했습니다.
PROB.03
시각화 부재
수동으로 차트를 업데이트해야 했고, 셀 데이터가 바뀌면 차트도 깨지는 경우가 잦았습니다.
PROB.04
디자인 불만족
기능에 집중하다 보니 UI가 조잡해지고, 데이터를 보는 경험 자체가 즐겁지 않았습니다.

03 · Solution

해결책 소개

기존 앱을 찾는 대신 직접 가계부 앱을 개발하는 것을 선택했습니다. 우리에게 딱 맞는 2인 뷰 구조와 자동화 기능을 온전히 구현하기 위해서입니다.

핵심 전략

  • Flutter 단일 코드베이스로 Windows와 Android 동시 지원 — 어디서든 같은 UX
  • 2인 뷰 분리를 앱 최상위 구조로 설계 — 화면 전환 없이 탭으로 전환
  • Drift ORM + SQLite로 로컬 DB 운용 — 인터넷 없이도 완전 동작
  • 은행 앱 CSV 가져오기로 수동 입력 최소화 — 기관별 파싱 프로파일 적용
  • Riverpod으로 상태 관리 — 차트와 리스트가 동일 데이터 소스에서 자동 갱신

09 · Troubleshooting

트러블슈팅

🔒 CODEF API 인증 요건 미충족

실제 계좌 연동을 위해 CODEF API를 검토했으나, 개인 프로젝트에서는 사업자 등록 및 정식 허가가 필요해 적용이 불가능했습니다.

📂 CSV 파싱 — 기관별 포맷 상이

은행마다 CSV 컬럼 구조와 날짜·금액 포맷이 달라, 단일 파서로 처리할 수 없었습니다. 기관별 파싱 프로파일을 별도 구현하여 해결했습니다.

교훈: 외부 API 도입 전 인증/허가 요건을 먼저 검토하는 것이 중요합니다. 제약이 발생했을 때 대안 설계로 빠르게 전환하는 능력이 프로젝트 완수의 핵심이었습니다.
