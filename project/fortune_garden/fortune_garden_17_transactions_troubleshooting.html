Visual Consistency

TS-001 · 수입 색상이 파란색으로 표시됨

증상: 거래 내역 및 대시보드에서 수입 금액이 민트 대신 파란색(#4ec6de 계열)으로 표시되는 현상이 발생하였습니다.

원인: dashboard_screen.dart에서 AppTheme.incomeLight/incomeDark 상수를 사용하지 않고 색상을 직접 하드코딩한 것이 원인이었습니다. 

// 잘못된 코드: 파란 민트색 직접 지정
final incomeColor = isDark
    ? const Color.fromARGB(255, 78, 193, 222)   // 파란 민트
    : const Color.fromARGB(255, 78, 210, 222);  // 파란 민트

해결: 해당 코드를 AppTheme 상수로 교체하여 테마에 대응하도록 수정하였습니다. 

// 수정 코드
final incomeColor = isDark ? AppTheme.incomeDark : AppTheme.incomeLight;

영향 파일: lib/presentation/screens/dashboard/dashboard_screen.dart 내 _SummaryCard, _TxnTile, FAB 배경색 등에 적용되었습니다.

Layout

TS-002 · Desktop 레이아웃 좌측 컬럼 오버플로우

증상: 데스크탑 화면에서 거래 목록이 우측으로 넘치는 현상이 발견되었습니다.

원인: _DesktopLayout의 좌측 패딩이 32px로 과도하게 설정되어 있어, NavigationRail(72px)과 합산 시 가용 공간이 부족해졌기 때문입니다.

해결: 데스크탑 좌우 패딩을 32px에서 20px로 축소하여 레이아웃 안정성을 확보하였습니다. 

// 수정 전: padding: EdgeInsets.fromLTRB(32, 12, 24, 8)
// 수정 후: padding: EdgeInsets.fromLTRB(20, 12, 16, 8)
Rendering

TS-003 · RenderFlex overflow — _HeaderBackground Column

증상: 하단과 같은 RenderFlex 오버플로우 에러가 발생하였습니다. 

RenderFlex#ea268 OVERFLOWING
creator: Column ← Padding ← ColoredBox ← Container ← _HeaderBackground
constraints: BoxConstraints(w=784.6, h=32.0)

원인: SliverAppBar의 FlexibleSpaceBar background 내부에서 Column이 제약 높이(32px)를 초과하였습니다. 상단 패딩(56px)과 텍스트 콘텐츠가 남은 공간(44px)에 맞지 않아 발생한 문제입니다.

해결: FlexibleSpaceBar와 _HeaderBackground 구조를 완전히 제거하고, 헤더를 SliverAppBar(고정형)와 SliverToBoxAdapter(제목 영역)로 분리하였습니다.

Desktop Optimization

TS-004~005 · 헤더 구조 개선 및 잔재 제거

TS-004 증상 및 해결: 데스크탑에서 녹색 헤더 영역이 좌우로 나뉘어 보이던 문제를 해결하기 위해 AppBar를 최상위 레벨로 올리고 좌우 컬럼을 그 아래 Row로 배치하는 통합 레이아웃으로 변경하였습니다. 

// 수정 후 구조
Column(
  Container(color: headerBg, height: 56)  // 통합 AppBar
  Row(
    _TxnListColumn(AppBar 없음),          // 좌: 제목 + 목록
    _SummaryPanel(AppBar 없음),           // 우: 요약
  )
)

TS-005: 수정 과정에서 _SummaryPanel 상단에 남아있던 불필요한 짙은 녹색 Container와 변수 선언을 모두 제거하였습니다.

List Components

TS-006~008 · 거래 항목 구분선(Divider) 미표시 해결

구분선이 보이지 않던 문제는 색상 토큰 가시성, 렌더링 조건, 위젯 계층 구조의 결함이 원인이었습니다.

  • TS-006: 다크 모드 배경과 유사한 outlineVariant를 시인성이 높은 outline으로 교체하였습니다.
  • TS-007: showDivider 조건을 !isLast && !nextIsHeader로 변경하여 마지막 항목과 날짜 헤더 직전 항목을 제외한 모든 곳에 표시되도록 하였으며, 구분선을 Dismissible 내부로 이동시켰습니다.
  • TS-008: 괄호 구조가 뒤섞여 발생하던 위치 오류를 해결하기 위해 _TxnCard의 build 메서드를 처음부터 완전히 재작성하였습니다.
Data Type

TS-009 · fold double 반환 타입 오류

증상: The returned type 'double' isn't returnable from a 'int' function. 에러가 발생하였습니다.

원인: fold의 초기값 0이 명확한 int로 추론되지 않아 타입 불일치가 발생한 것입니다.

해결: 타입 파라미터를 <int>로 명시하여 해결하였습니다. 

// 수정 후
.fold<int>(0, (s, t) => s + t.amount)
General Rules

재발 방지를 위한 공통 원칙

  • 색상은 반드시 AppTheme 상수를 사용하며 Color.fromARGB() 하드코딩을 금지합니다.
  • 구분선 등 아이템 부속 위젯은 반드시 Dismissible의 child 내부에 배치하여 레이어 간섭을 방지합니다.
  • 구조가 복잡한 위젯은 부분 수정보다 전체 재작성을 통해 논리적 결함을 원천 차단합니다.
  • 데스크탑 분할 레이아웃에서는 AppBar를 최상위 레벨에서 단일하게 구성합니다.
