02 — SCR-007

SCR-007 계좌 관리 (accounts_screen.dart)

기존 목록 조회만 있던 화면을 SAD v1.1 §4.1.2 명세(AccountListTile, BalanceSummary, ImportHistoryList) 기준으로 전면 재구현하였습니다.

화면 구성 요소

Component 01 _BalanceSummary

전체 계좌 잔액 합산 카드. 아이콘 + 총액 + 계좌 수 표시

Component 02 _ProfileSection

프로필별 섹션 헤더(색상 원 + 이름) + 해당 프로필 계좌 목록

Component 03 _AccountTile

계좌 타일. 별명·기관코드·잔액·최근 가져오기 일시 표시. 탭 시 옵션 시트

Component 04 _AccountFormSheet

계좌 등록/수정 BottomSheet. 프로필·기관·별명 입력

Component 05 _ImportHistorySheet

DraggableScrollableSheet. 파일명·일시·신규/중복/오류 건수 표시

Component 06 _EmptyGuide

계좌 없을 때 안내 화면 + 계좌 등록 버튼

Component 07 FAB

우측 하단 "계좌 등록" FloatingActionButton.extended

계좌 등록/수정 폼 (_AccountFormSheet)

신규 등록과 수정을 하나의 BottomSheet로 처리합니다. existing 파라미터가 null이면 신규, 값이 있으면 수정 모드입니다.

입력 항목 세부 내용
프로필 선택 DropdownButtonFormField. 수정 모드에서는 비활성(disabled)
금융기관 선택 institutions 테이블에서 로드. 은행/카드/증권 유형별 그룹 헤더 표시
계좌 별명 TextField (선택 입력). 미입력 시 기관명으로 대체 표시
계좌번호 암호화 개발용 placeholder(utf8.encode). 실제 운영 시 AES-256-GCM 필요

계좌 삭제

AlertDialog로 확인 절차를 거칩니다. 삭제 시 CASCADE로 transactions·import_history도 함께 삭제됨을 경고 문구로 안내합니다.

// 삭제 확인 다이얼로그 경고 문구
'${account.alias ?? account.institutionCode}' 계좌를 삭제하면
관련 거래 내역과 가져오기 이력도 함께 삭제됩니다.

가져오기 이력 (_ImportHistorySheet)

DraggableScrollableSheet로 구현. 초기 높이 60%, 최소 30%, 최대 90%. IImportHistoryRepository.getByAccount()로 데이터 로드.

표시 항목 내용
파일명 fileName 컬럼. 말줄임 처리
가져오기 일시 importedAt Unix ms → yyyy.MM.dd HH:mm 포맷
신규 건수 newRows — 초록 Chip
중복 건수 duplicateRows — 회색 Chip (0이면 미표시)
오류 건수 errorRows — 빨간 Chip (0이면 미표시)
03 — CSV Import Fix

CSV 가져오기 플로우 수정

수정 배경

KB국민은행 EUC-KR CSV 가져오기 시 아래 오류가 발생하였습니다.

Bad state: The BackgroundIsolateBinaryMessenger.instance value is invalid until BackgroundIsolateBinaryMessenger.ensureInitialized is executed.

원인: charset_converter는 Flutter 네이티브 플러그인이라 플러그인 채널(BinaryMessenger)을 사용합니다. Dart Isolate는 기본적으로 Flutter 엔진과 분리되어 있어 BinaryMessenger 초기화 전에는 네이티브 플러그인 호출이 불가합니다.

수정 내용

3.2.1 csv_import_use_case.dart — 인코딩 변환을 Isolate 밖으로 이동

변경 전
Isolate.run() 내에서 CsvParserUseCase().parse(file, profile) 호출 → parse() 내부에서 CharsetConverter.decode() 호출 → 오류 발생
변경 후
① 메인 스레드: file.readAsBytes() + CharsetConverter.decode() 완료
② Isolate: parseFromString(content, profile) 호출 — 네이티브 플러그인 미사용 
// 변경 전
final parsed = await Isolate.run(() async {
  return CsvParserUseCase().parse(file: file, parserProfile: profile);
  // ↑ 내부에서 CharsetConverter.decode() 호출 → Isolate 오류
});

// 변경 후
// ① 메인 스레드에서 인코딩 변환
final bytes = await file.readAsBytes();
final content = await _decodeBytes(bytes, parserProfile.encoding);

// ② Isolate에는 문자열만 전달
final parsed = await Isolate.run(() {
  return CsvParserUseCase().parseFromString(
    content: content, parserProfile: parserProfile,
  );
});

3.2.2 csv_parser_use_case.dart — parseFromString() 메서드 추가

Isolate 안전 메서드와 파일 직접 파싱 메서드를 분리하였습니다.

parse() 메인 스레드 전용

파일 읽기 + 디코딩 + parseFromString() 위임. async. charset_converter 사용.

parse
FromString() Isolate 안전

이미 디코딩된 문자열로 파싱. 동기 반환(List, not Future). 네이티브 플러그인 미사용.

parseFromString()은 Future가 아닌 동기 List<ParsedTransaction> 반환. Isolate.run()의 콜백은 async가 필요 없으며 동기 코드가 더 안전합니다.

04 — Test Script

EUC-KR 검증용 테스트 CSV 스크립트 (make_test_csv.py)

실제 은행 CSV 파일 없이 EUC-KR 파싱 동작을 검증할 수 있는 Python 스크립트입니다. 프로젝트 루트에서 실행합니다. 

# 프로젝트 루트에서 실행
python make_test_csv.py

# 생성 결과: test_csv/ 폴더
# test_kb.csv      --- KB국민은행 (EUC-KR, yyyy.MM.dd HH:mm:ss)
# test_nh.csv      --- NH농협은행 (EUC-KR, yyyyMMdd)
# test_shinhan.csv --- 신한은행 (UTF-8, yyyy-MM-dd)
# test_hana.csv    --- 하나은행 (EUC-KR, yyyy-MM-dd HH:mm:ss)
# test_woori.csv   --- 우리은행 (EUC-KR, yyyyMMdd)
파일 기관 인코딩 날짜 형식 비고
test_kb.csv KB EUC-KR yyyy.MM.dd HH:mm:ss 7건
test_nh.csv NH EUC-KR yyyyMMdd 7건
test_shinhan.csv SHINHAN UTF-8 yyyy-MM-dd 7건
test_hana.csv HANA EUC-KR yyyy-MM-dd HH:mm:ss 7건
test_woori.csv WOORI EUC-KR yyyyMMdd 7건 — 우리은행 추가

검증 체크리스트

  • 한글 거래처명(스타벅스, 이마트 등)이 깨지지 않고 표시되는가
  • 신규 가져오기 시 newRows = 7, duplicateRows = 0 표시되는가
  • 같은 파일 재가져오기 시 newRows = 0, duplicateRows = 7 표시되는가
  • 날짜가 올바르게 파싱되는가 (거래 상세 화면에서 확인)
  • 금액 부호가 맞는가 (지출 음수, 입금 양수)

하나·우리은행의 amountCol(출금액/출금금액)은 CSV에 양수로 저장된 지출 금액입니다. 실제 은행 파일에서 부호가 반대로 나오면 파서 프로필에 별도 처리 필요합니다.
