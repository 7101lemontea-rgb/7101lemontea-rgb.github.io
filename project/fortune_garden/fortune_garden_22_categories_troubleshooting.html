Database Migration

TBS-001 · 기존 DB 기기 신규 카테고리 미반영

증상: 앱 업데이트 후 2인 가구용 신규 카테고리 4개(외식, 카페/간식, 주거, 잡화)를 추가했으나, 이미 앱이 설치된 기기에서는 해당 카테고리가 나타나지 않았습니다.

원인: Drift의 onCreate 전략은 DB 파일이 최초로 생성될 때만 실행됩니다. 스키마 변경이 없는 단순 데이터 추가(Seeding) 로직이 onCreate에만 묶여 있어 기존 설치 기기에서는 호출되지 않았던 것입니다. 

// 해결: beforeOpen에서 매번 호출하여 데이터 무결성 확보
beforeOpen: (details) async {
  await customStatement('PRAGMA journal_mode=WAL');
  await customStatement('PRAGMA foreign_keys=ON');
  await _seedCategories(); // 앱 시작 시마다 실행
},
SQLite Constraint

TBS-002 · UNIQUE constraint failed: categories.name

증상: beforeOpen에 시딩 로직을 추가한 후 앱 실행 시 SqliteException(2067)과 함께 UNIQUE constraint failed 에러가 발생하며 앱이 중단되었습니다.

원인: insertAllOnConflictUpdate는 PK(id)를 기준으로 충돌을 감지합니다. 시딩 시 ID를 명시하지 않았기 때문에 Drift는 이를 신규 삽입으로 간주하여 INSERT를 시도했고, 이 과정에서 UNIQUE 제약이 걸린 name 컬럼(예: 식비)이 기존 데이터와 충돌했습니다. 

// 수정 전: PK 기준 UPDATE 시도로 인한 이름 중복 충돌
await batch((b) => b.insertAllOnConflictUpdate(categories, cats));

// 수정 후: 모든 UNIQUE 제약 충돌 시 해당 항목 무시(Skip)
await batch((b) => b.insertAll(
  categories,
  cats,
  mode: InsertMode.insertOrIgnore,
));

InsertMode.insertOrIgnore를 적용함으로써 이미 존재하는 카테고리는 안전하게 건너뛰고, 새로 추가된 4개의 카테고리만 성공적으로 DB에 삽입되도록 해결했습니다.
