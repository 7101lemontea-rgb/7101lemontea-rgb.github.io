자료형 (Data Type)

자료형 개요

자바스크립트의 자료형(Data Type)은 원시형(Primitive Type)과 참조형(Reference Type)으로 나뉩니다.

원시형
Primitive Type

값 자체를 저장하며, 불변(immutable)합니다. String, Number, BigInt, Boolean, Null, Undefined, Symbol

참조형
Reference Type

값이 저장된 주소(참조값)를 저장하며, mutable합니다. Object, Array

01

원시형 (Primitive Type)

원시형은 값 자체를 저장하며, 불변(immutable)합니다.

'불변'은 변수에 저장된 값을 바꿀 수 없다는 뜻이 아니라, 메모리에 저장된 '그 값 자체'를 수정할 수 없다는 뜻입니다. 값을 변경하면 기존 값을 수정하지 않고 새로운 객체를 생성하며, 기존의 값은 나중에 가비지 컬렉터(GC)에 의해 사라집니다.

※ 원시형 자료형의 종류

자료형 설명 예시
String 문자열. 쌍따옴표("), 홑따옴표('), 백틱(`)으로 묶어서 표현 "hello", 'a', `apple`
Number 정수, 실수 구분없이 하나의 타입. NaN, Infinity 포함 42, 3.14, NaN, Infinity
BigInt 아주 큰 정수(2^53-1 초과) 9007199254740993n
Boolean 논리값 true, false
Null '값이 없음'을 명시적으로 표현 null
Undefined var, let 키워드로 선언된 변수에 암묵적으로 할당되는 값 (값이 할당되지 않음을 표현) undefined
Symbol 고유하고 변경 불가능한 식별자(ES6 도입). 라이브러리나 프레임워크를 만들 때 주로 활용 Symbol('id')
js
let a = undefined; // ❌ 변수에 값이 할당되지 않음을 뜻합니다. 프로그래머가 명시적으로 할당하지 마세요.
let b = null;      // ✅ 명시적으로 값이 없음을 표현하고 싶으면 null을 할당하세요.

a. 문자열 (String)

js
const greeting = "안녕하세요";
const message  = '작은따옴표도 OK';
const template = `백틱으로 ${greeting}`; // 템플릿 리터럴

// 문자열 연결
const firstName = "길동";
const lastName  = "홍";
const fullName  = lastName + firstName; // "홍길동"

// 템플릿 리터럴
const intro = `제 이름은 ${fullName}입니다.`;
console.log(intro); // "제 이름은 홍길동입니다."

b. 숫자 (Number)

js
const age      = 25;    // 정수
const price    = 19.99; // 소수
const negative = -10;   // 음수

// 기본 연산
console.log(10 + 3); // 13  (덧셈)
console.log(10 - 3); // 7   (뺄셈)
console.log(10 * 3); // 30  (곱셈)
console.log(10 / 3); // 3.333... (나눗셈)
console.log(10 % 3); // 1   (나머지)

// 자바스크립트는 몫(//) 연산자는 없음

c. 불리언 (Boolean)

js
const isStudent  = true;
const hasLicense = false;

// 비교 연산의 결과도 불리언
console.log(10 > 5);       // true
console.log(10 < 5);       // false
console.log(10 === 10);    // true  (값과 타입 모두 같은지)
console.log(10 == "10");  // true  (값만 비교 - 사용 자제)
console.log(10 === "10"); // false (타입이 다름)

d. undefined와 null

js
let noValue;
console.log(noValue);    // undefined (값을 할당하지 않음)

const empty = null; // null (의도적으로 "비어있음"을 표현)

02

참조형 (Reference Type)

참조형은 값이 저장된 주소(참조값)를 저장하며, mutable합니다.

변수 자체를 바꾸지 않고도 내부 데이터를 자유롭게 수정할 수 있는 상태를 Mutable(가변성)이라고 합니다.

※ 참조형 자료형의 종류

자료형 설명 예시
Object 키-값 쌍의 집합 { name: "Tom", age: 20 }
Array 순서가 있는 값들의 집합 [1, 2, 3]

a. 배열 (Array)

js
const fruits = ["사과", "바나나", "오렌지"];

// 접근 (인덱스는 0부터!)
console.log(fruits[0]); // "사과"
console.log(fruits[1]); // "바나나"
console.log(fruits[2]); // "오렌지"

// 길이
console.log(fruits.length); // 3

// 추가
fruits.push("포도");
console.log(fruits); // ["사과", "바나나", "오렌지", "포도"]

b. 객체 (Object)

js
const person = {
  name:      "홍길동",
  age:       25,
  isStudent: true
};

// 접근 방법 1: 점 표기법
console.log(person.name);      // "홍길동"

// 접근 방법 2: 대괄호 표기법
console.log(person["age"]);   // 25

// 수정 (키가 있으면 수정)
person.age = 26;

// 추가 (키가 없으면 추가)
person.email = "hong@email.com";

※ 참고 — const로 선언된 변수에는 값을 재할당 할 수 없지만, const로 선언된 변수가 참조형 데이터인 경우 데이터 내부의 값은 변경할 수 있습니다.

❌ 재할당 — error
const user = { name: "김철수", age: 20 };
user = { name: "홍길동", age: 30 }; // error
✅ 내부 수정 — ok
const user = { name: "김철수", age: 20 };
user.name = "홍길동"; // ok
console.log(user); // { name: "홍길동", age: 20 }

03

타입 확인 (typeof)

js
console.log(typeof "안녕");    // "string"
console.log(typeof 42);       // "number"
console.log(typeof true);     // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof null);      // "object" ← JS의 오래된 버그!
console.log(typeof [1, 2, 3]); // "object"
console.log(typeof {a: 1});    // "object"

연습 문제 1

typeof 출력 결과 예측하기

다음 코드의 출력 결과를 예상하세요.

js
console.log(typeof "100");         // ?
console.log(typeof 100);           // ?
console.log(typeof null);          // ?
console.log(typeof undefined);     // ?
console.log(typeof [1, 2, 3]);     // ?
console.log(typeof { name: "홍길동" }); // ?
표현식 결과 이유
typeof "100" string 큰따옴표나 작은따옴표로 감싸진 값은 문자열이다.
typeof 100 number 정수, 실수 모두 자바스크립트에서는 숫자형으로 취급한다.
typeof null object 자바스크립트 설계상의 유명한 오류(Bug)이다. null은 원시형이지만 역사가 깊은 버그로 인해 object를 반환한다.
typeof undefined undefined 값이 할당되지 않은 변수의 기본 상태인 undefined 타입이다.
typeof [1,2,3] object 자바스크립트에서 배열은 객체(Object)의 한 종류이다. 별도의 array 타입을 반환하지 않는다.
typeof { name: ... } object 중괄호로 만들어진 전형적인 객체 타입이다.

연습 문제 2

에러가 발생하는 코드 고르기

다음 코드 중 에러가 발생하는 것을 고르고, 그 이유를 설명하세요.

js
// (1)
const numbers = [1, 2, 3];
numbers.push(4);

// (2)
const numbers = [1, 2, 3];
numbers = [4, 5, 6];

// (3)
const user = { name: "철수" };
user.name = "영희";

// (4)
const user = { name: "철수" };
user.age = 20;

// (5)
const score = 100;
score = 200;
번호 에러 여부 에러 종류 이유
(1) ✅ 정상 — numbers가 가리키는 '배열의 주소'는 바뀌지 않았다. 주소는 그대로 두고 그 안에 내용(4)만 추가한 것이므로 const 규칙을 어기지 않았다.
(2) ❌ 에러 TypeError const로 선언된 변수 numbers에 새로운 배열(새로운 주소값)을 통째로 대입하려고 했기 때문에 재할당 에러가 발생한다.
(3) ✅ 정상 — user 변수가 담고 있는 '객체의 메모리 주소'는 고정이다. 하지만 그 객체 내부의 속성을 수정하는 것은 허용된다.
(4) ✅ 정상 — 존재하지 않던 새로운 속성을 추가하는 것 역시 객체 내부를 수정하는 행위이므로 정상 작동한다.
(5) ❌ 에러 TypeError score는 원시형(Number) 값을 가진 상수이다. const로 선언된 상수는 한 번 할당된 값을 다른 값으로 바꿀 수 없다.

연습 문제 3

출력 결과 예측하기

다음 코드의 출력 결과를 예측하세요.

js
const student = {
  name:   "김코딩",
  age:    20,
  scores: [90, 85, 95]
};

console.log(student.name);          // (1) ?
console.log(student["age"]);       // (2) ?
console.log(student.scores[1]);     // (3) ?
console.log(student.scores.length); // (4) ?

student.scores.push(100);
console.log(student.scores);        // (5) ?
출력 (1) 김코딩

마침표(.)를 사용해 name 속성값에 접근한다.

출력 (2) 20

대괄호([])와 문자열 키를 사용해 age 속성값에 접근한다.

출력 (3) 85

scores 배열의 1번 인덱스(두 번째 요소)인 85를 출력한다.

출력 (4) 3

현재 scores 배열에 들어있는 요소의 개수(90, 85, 95)인 3을 출력한다.

출력 (5) [90,85,95,100]

push(100)에 의해 배열 끝에 100이 추가된 전체 배열을 출력한다.

[90, 85, 95, 100]
