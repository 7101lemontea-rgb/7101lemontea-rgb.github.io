React 11 — React Router와 SPA

Single Page Application의 개념을 이해하고 React Router로 페이지 전환을 구현합니다.

목차

  • 1.라우팅과 SPA
  • 2.설치 및 기본 설정
  • 3.핵심 컴포넌트 구조
  • 4.Link와 NavLink

01

라우팅(Routing)과 SPA

웹사이트에서 URL 주소에 따라 서로 다른 화면을 보여주는 것을 라우팅이라고 합니다.

SPA(Single Page Application)란?

전통적인 방식은 페이지 이동 시 서버에서 새 HTML을 받아 화면 전체를 새로고침(깜빡임)하지만, SPA는 처음 한 번만 HTML을 받고 이후에는 JavaScript가 필요한 부분만 교체하여 부드럽게 전환됩니다.

React Router의 역할:

React는 자체 라우팅 기능이 없기에 외부 라이브러리인 React Router를 사용하여 URL 변경을 감지하고 매칭되는 컴포넌트를 렌더링합니다.

02

설치 및 기본 설정

프로젝트 터미널에서 아래 명령어를 통해 라이브러리를 설치합니다.

npm install react-router-dom

앱 최상위에서 BrowserRouter로 감싸 라우팅 기능을 활성화합니다. 

// main.jsx
import { BrowserRouter } from "react-router-dom";


createRoot(document.getElementById("root")).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);

03

핵심 컴포넌트 구조

Routes는 경로들의 울타리 역할을 하며, Route는 주소(path)와 컴포넌트(element)를 연결합니다. 

// App.jsx
function App() {
  return (
    <div>
      <h1>My App</h1>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        {/* 404 페이지 처리 */}
        <Route path="/*" element={<NotFound />} />
      </Routes>
    </div>
  );
}

04

Link와 NavLink

<a> 태그 대신 Link나 NavLink를 사용해야 새로고침 없이 페이지가 전환됩니다.

컴포넌트 특징
Link 새로고침 없이 URL만 변경
NavLink 현재 경로와 일치할 때 자동으로 .active 클래스 추가 
function Navigation() {
  return (
    <nav>
      <NavLink to="/">Home</NavLink>
      <NavLink to="/about">About</NavLink>
      {/* 외부 링크는 기존 a 태그 사용 */}
      <a href="http://naver.com" target="_blank">Naver</a>
    </nav>
  );
}
