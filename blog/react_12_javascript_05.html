연산자 (Operator)

연산자 개요

연산자(Operator)는 데이터를 가지고 특정한 작업(연산)을 수행하도록 지시하는 기호나 키워드입니다. JavaScript에서 사용하는 연산자는 다음과 같습니다.

산술 연산자

+  -  *  /  %  **

증감 연산자

++  --

연결 연산자

+

할당 연산자

=  +=  -=  *=  /=  %=

비교 연산자

>  <  >=  <=  ==  !=  ===  !==

논리 연산자

&&  ||  !

01

증감 연산자

증감 연산자는 ++, -- 와 같이 변수의 값을 1씩 증가시키거나 감소시키는 단항 연산자입니다. 연산자의 위치에 따라 결과값이 할당되는 시점이 달라지므로 이 차이를 명확히 아는 것이 중요합니다.

a. ++n, --n : 선 연산 후 할당

js
let a = 5;
let b = ++a; // a를 먼저 6으로 만들고, 그 값을 b에 대입

console.log(a); // 6
console.log(b); // 6

b. n++, n-- : 선 할당 후 연산

js
let x = 5;
let y = x++; // 현재 x값(5)을 y에 먼저 대입하고, 그 다음 x를 6으로 만듦

console.log(x); // 6
console.log(y); // 5
++n  — 선 연산 후 할당

먼저 값을 증가시킨 뒤 결과를 반환합니다.

반환값 = 증가된 값

n++  — 선 할당 후 연산

현재 값을 먼저 반환한 뒤 증가시킵니다.

반환값 = 증가 전 값

02

비교 연산자

a. == , != : 동등 연산자

동등 연산자는 타입을 변환 후 비교합니다. (느슨한 비교)

js
5 == "5"            // true  (문자열 "5"가 숫자 5로 변환)
1 == true          // true  (true가 1로 변환)
0 == false         // true  (false가 0으로 변환)
null == undefined  // true  (특수 규칙)
"" == 0           // true  (빈 문자열이 0으로 변환)
" " == 0          // true  (공백 문자열도 0으로 변환)

b. === , !== : 일치 연산자

일치 연산자는 타입까지 비교합니다. (엄격한 비교)

js
5 === "5"             // false (타입이 다름: 숫자 vs 문자열)
1 === true           // false (타입이 다름: 숫자 vs 불리언)
0 === false          // false (타입이 다름: 숫자 vs 불리언)
null === undefined   // false (타입이 다름)
"" === 0            // false (타입이 다름: 문자열 vs 숫자)
5 === 5            // true  (값과 타입이 모두 일치)
"hello" === "hello" // true  (값과 타입이 모두 일치)

실무에서는 예기치 않은 타입 변환으로 인한 버그를 방지하기 위해 == 대신 ===을 사용하는 것을 권장합니다.

03

논리 연산자

연산자 이름 설명 예시
&& AND 두 피연산자가 모두 true일 때만 true true && false → false
|| OR 두 피연산자 중 하나라도 true면 true true || false → true
! NOT 피연산자의 논리값을 반전시킴 !true → false

연습 문제 1

출력 결과 예측하기

다음 코드의 출력 결과를 예측하세요.

js
let a = 10;
let b = a++;  // ?
let c = ++a;  // ?

console.log(a); // (1) ?
console.log(b); // (2) ?
console.log(c); // (3) ?
출력 (1) 12

a++에서 한 번(10→11), ++a에서 한 번(11→12), 총 두 번 증가합니다.

출력 (2) 10

현재 값(10)을 먼저 b에 할당한 후, a를 11로 올립니다.

출력 (3) 12

a(11)를 먼저 12로 올린 후, 그 결과값을 c에 할당합니다.

연습 문제 2

비교 연산 결과 예측하기

다음 비교 연산의 결과를 예측하세요.

js
console.log(0 == false);         // (1) ?
console.log(0 === false);        // (2) ?
console.log(null == undefined);  // (3) ?
console.log(null === undefined); // (4) ?
console.log("10" == 10);         // (5) ?
console.log("10" === 10);        // (6) ?
표현식 결과 이유
0 == false true 둘 다 Falsy 값이므로 타입을 맞춰 비교하면 같다.
0 === false false 값은 비슷해 보여도 타입(Number vs Boolean)이 다르다.
null == undefined true 자바스크립트 규칙상 null과 undefined는 서로 같다고 약속되어 있다.
null === undefined false 타입(null 객체 vs undefined형)이 서로 다르다.
"10" == 10 true 문자열 "10"을 숫자 10으로 자동 변환하여 비교한다.
"10" === 10 false 타입(String vs Number)이 다르므로 거짓이다.

연습 문제 3

논리 연산 결과 예측하기

다음 논리 연산의 결과를 예측하세요.

js
console.log(true && false);       // (1) ?
console.log(true || false);       // (2) ?
console.log(!true);              // (3) ?
console.log(10 > 5 && 3 < 1);   // (4) ?
console.log(10 > 5 || 3 < 1);   // (5) ?
console.log(!(10 === 10));       // (6) ?
출력 (1) false

&&는 둘 다 true여야만 true이다.

true && false → false
출력 (2) true

||는 둘 중 하나만 참이어도 참이다.

true || false → true
출력 (3) false

NOT(!)은 반대로 뒤집는다.

!true → false
출력 (4) false

10 > 5는 true, 3 < 1은 false이므로 true && false가 된다.

true && false → false
출력 (5) true

10 > 5는 true이므로, OR 연산 규칙에 의해 참이 된다.

true || false → true
출력 (6) false

10 === 10은 true이고 !로 반전되어 false가 된다.

!(true) → false
