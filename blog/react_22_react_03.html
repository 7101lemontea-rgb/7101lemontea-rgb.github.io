React 03 — JSX 문법의 정의와 주요 규칙

React 프로젝트에서 화면을 설계할 때 가장 기본이 되는 JSX(JavaScript XML)의 개념과 필수 규칙에 대해 정리합니다.

01

JSX의 정의

JSX는 JavaScript 코드 내부에서 HTML과 유사한 마크업을 작성할 수 있도록 지원하는 확장 문법입니다.

  • 변환 과정: 브라우저는 JSX를 직접 해석하지 못하므로, Babel과 같은 도구를 통해 일반 JavaScript 코드로 변환하는 과정이 필요합니다.
  • 장점: UI 로직(JavaScript)과 UI 구조(HTML)를 한 곳에서 관리할 수 있어 가독성과 개발 효율성이 높습니다.

02

JSX 기본 규칙

a. 하나의 부모 요소로 감싸기

JSX에서는 반드시 하나의 부모 요소가 다수의 자식 요소를 감싸는 형태여야 합니다. 불필요한 태그 생성을 피하려면 Fragment(<>...</>)를 사용합니다. 

function App() {
  return (
    <>
      <h1>제목</h1>
      <p>내용</p>
    </>
  );
}

b. JavaScript 표현식 사용 ({ })

중괄호 { }를 통해 JavaScript 변수나 표현식을 마크업 내에 삽입합니다.

  • 표현식만 가능: 변수, 수식, 삼항 연산자 등 값으로 평가되는 식만 허용됩니다. (if, for문 사용 불가)
  • 렌더링 특성: 숫자, 문자열은 표시되나 true, false, null 등은 화면에 나타나지 않습니다.
  • 객체 처리: 객체 자체를 넣으면 오류가 발생하므로 점 표기법으로 개별 값을 추출해야 합니다.

c. 태그 닫기 의무화

모든 태그는 반드시 닫아야 하며, 자식이 없는 태그는 <input />과 같은 셀프 클로징 형식을 권장합니다.

d. camelCase 속성명 사용

HTML 속성은 JavaScript 예약어와의 충돌 방지를 위해 카멜케이스를 사용합니다.

  • class → className
  • onclick → onClick
  • for → htmlFor

e. 인라인 스타일의 객체화

스타일은 문자열이 아닌 객체 형태로 전달하며, 속성명에 카멜케이스를 적용합니다. 

<div style={{ color: "red", fontSize: "20px" }}>텍스트</div>

f. 주석 작성

JSX 내부 주석은 {/* ... */} 형식을 사용합니다.

03

디버깅 퀴즈

아래의 오류가 포함된 코드를 올바르게 수정하는 방법입니다.

function App() {
  const title = "리액트 수업";
  return (
    <h1>{title}</h1>
    <p class="desc">반갑습니다.</p>
    <br>
    <div style="color: red">경고</div>
  );
}
  • 부모 태그 누락: 전체를 <>...</>로 감싸야 합니다.
  • 속성명 오류: class를 className으로 수정합니다.
  • 태그 미종결: <br>을 <br />로 닫습니다.
  • 스타일 형식 오류: style={{ color: "red" }}로 수정합니다.
