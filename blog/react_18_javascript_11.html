JavaScript 11 — 스프레드 연산자

배열과 객체를 자유자재로 다루기 위한 스프레드 연산자(...)의 개념과 리액트에서의 활용법을 정리합니다.

01

스프레드 연산자 (Spread Operator)

스프레드 연산자(...)는 말 그대로 배열이나 객체의 요소를 펼쳐서 개별 요소로 분리해주는 연산자입니다.

주요 용도

  • 배열이나 객체를 개별 인자로 풀어 함수에 전달
  • 배열이나 객체를 복사하거나 결합

02

배열 스프레드

2.1. 배열 복사

const original = [1, 2, 3];
const copy = [...original];

※ 참조와 복사의 차이

const copy = arr;와 같이 단순 할당하면 두 변수는 같은 주소를 가리킵니다. 복사본을 수정하면 원본도 변경되는 문제가 발생합니다. 

// 단순 할당 (원본 영향 O)
const arr = [1, 2, 3];
const copy = arr;
copy[0] = 100; // console.log(arr) -> [100, 2, 3]

// 스프레드 연산자 (원본 유지)
const arr2 = [1, 2, 3];
const copy2 = [...arr2];
copy2[0] = 100; // console.log(arr2) -> [1, 2, 3]

⚠️ 주의할 점: 얕은 복사(Shallow Copy)

가장 바깥쪽만 복사하며, 중첩된 배열/객체는 여전히 참조를 공유합니다.

[참고] 깊은 복사 방법: JSON.parse(JSON.stringify(obj)) 또는 Lodash의 _.cloneDeep() 사용

2.2. 배열 결합 및 추가

// 배열 결합
const arr1 = [1, 2, 3], arr2 = [4, 5, 6];
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// 요소 추가
const newArr = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

03

객체 스프레드

3.1. 객체 복사 및 합치기

const original = { name: "홍길동", age: 25 };
const copy = { ...original };

// 객체 합치기
const details = { city: "서울" };
const merged = { ...original, ...details };

3.2. 속성 덮어쓰기

동일한 키가 존재할 경우, 뒤에 오는 값이 앞의 값을 덮어씁니다. 

const user = { name: "홍길동", age: 25 };

// 스프레드가 뒤에 오는 경우 (원본에 의해 덮어씌워짐)
const result1 = { age: 30, ...user }; // { age: 25, name: '홍길동' }

// 스프레드가 앞에 오는 경우 (새로운 값으로 업데이트)
const result2 = { ...user, age: 30 }; // { name: '홍길동', age: 30 }

3.3. React 상태 업데이트 적용

리액트의 상태(State)를 업데이트할 때 기존 객체를 보존하면서 특정 필드만 수정하는 용도로 자주 쓰입니다. 

const [user, setUser] = useState({ name: '홍길동', age: 25 });

// 나이만 업데이트
const handleBirthday = () => {
  setUser({ ...user, age: user.age + 1 });
};

연습 문제 종합

Q1. 출력 결과는?

const fruits = ['사과', '바나나'];
const more = [...fruits, '포도'];
fruits.push('망고');

A: fruits: ['사과', '바나나', '망고'], more: ['사과', '바나나', '포도']

Q2. 리액트 이메일 변경 함수 완성

const handleEmailChange = (newEmail) => {
  setUser({ ...user, email: newEmail });
};
