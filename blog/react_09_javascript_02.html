Javascript 변수

변수란?

변수는 데이터를 저장하는 이름 붙은 공간입니다. JavaScript에서는 let과 const 키워드로 변수를 선언합니다.

let
변하는 값

재할당이 필요한 경우 사용합니다. 블록 스코프를 따릅니다.

const
변하지 않는 값 (상수)

선언과 동시에 값을 할당해야 하며, 재할당이 불가합니다. 블록 스코프를 따릅니다.

01

변수의 종류

a. let

let은 변하는 값을 할당할 때 사용합니다.

변수를 선언하면 변수가 생성되며, 변수를 선언만 하고 값을 할당하지 않으면 undefined로 값이 초기화됩니다.

변수 선언만 하는 경우

js
let l1;
console.log(l1); // undefined

변수의 선언과 할당을 분리하여 작성할 수 있습니다.

js
let l1;
console.log(l1); // undefined

l1 = 1;
console.log(l1); // 1

값을 재할당 할 수 있습니다.

js
let l1;
console.log(l1); // undefined

l1 = 1;
console.log(l1); // 1

l1 = 2;
console.log(l1); // 2

값을 재선언 할 수 없습니다.

js
let l1;
console.log(l1); // undefined

l1 = 1;
console.log(l1); // 1

l1 = 2;
console.log(l1); // 2

let l1; // error
항목 let
선언만 ✅ 가능 (undefined로 초기화)
재할당 ✅ 가능
재선언 ❌ 불가능 (SyntaxError)
스코프 블록 스코프

b. const

const는 상수로, 변하지 않는 값을 할당할 때 사용합니다.

변수 선언과 동시에 값을 할당해야 합니다.

js
const c1; // error
const c1 = 1; // ok

변수를 재선언 할 수 없습니다.

js
const c1 = 1;
const c1 = 2; // error

값을 재할당 할 수 없습니다.

js
const c1 = 1;
c1 = 2; // error
항목 const
선언만 ❌ 불가능 (반드시 초기화)
재할당 ❌ 불가능 (TypeError)
재선언 ❌ 불가능 (SyntaxError)
스코프 블록 스코프

※ 주의사항

  • 변수를 키워드 없이 선언하면 암묵적으로 전역 변수가 되어 예기치 않은 오류를 유발할 수 있으므로 반드시 키워드를 사용해야 합니다.
  • var는 구버전 문법으로 사용하지 않아야 합니다.
js
// 기본적으로 const 사용
const API_URL = "https://api.example.com";
const MAX_ITEMS = 100;

// 값이 변해야 할 때만 let 사용
let count = 0;
count = count + 1;

// var는 구버전 문법
var oldWay = "쓰지 마세요"; //

4.2. 변수의 스코프

let과 const는 블록 스코프를 따릅니다.

let — 블록 스코프
function f() {
  {
    let num = 1;
    console.log(num); // 1
  }
  console.log(num); // error
}
f();
const — 블록 스코프
function f() {
  {
    const num = 1;
    console.log(num); // 1
  }
  console.log(num); // error
}
f();

블록 { } 안에서 선언된 변수는 블록 바깥에서 접근할 수 없습니다.

02

변수명 규칙

a. 필수 규칙

필수 규칙 — 어기면 에러가 발생합니다.

  • 영문자, 언더스코어(_), 달러기호($), 숫자로 변수명을 작성해야 합니다.
  • 첫 글자는 숫자로 시작할 수 없습니다.
  • 예약어는 식별자로 사용할 수 없습니다.
js
// ✅ 알파벳, 언더스코어(_), 달러($), 숫자로 구성
let userName = "홍길동";
let user_name = "홍길동";
let _private = "비공개";
let $money = 1000;
let userName2 = "김철수";

// ❌ 불가능
let 2ndPlace = "은메달"; // 숫자로 시작 불가
let user-name = "홍길동"; // 하이픈 불가
let let = "값"; // 예약어 사용 불가
let my name = "홍길동"; // 공백 불가

b. 네이밍 컨벤션

두 개 이상의 영단어로 구성된 경우 camelCase 사용을 권장합니다.

js
// 카멜케이스(camelCase) - JavaScript 표준
let firstName = "길동";
let lastName = "홍";
let totalPrice = 15000;
let isLoggedIn = true;
let maxItemCount = 100;

// 상수는 대문자 + 언더스코어
const MAX_SIZE = 100;
const API_KEY = "abc123";
const BASE_URL = "https://api.example.com";

c. 의미 있는 이름 사용

✅ 적절한 이름
let userName = "홍길동";
let userAge = 25;
let isAdult = true;
let productPrice = 10000;
❌ 적절하지 않은 이름
let x = "홍길동";
let data = 25;
let temp = true;
let a1 = 10000;

연습 문제 1

변수 선언 에러 찾기

다음 코드 중 에러가 발생하는 것을 모두 고르고, 그 이유를 설명하세요.

js
// (1)
let score;
score = 100;

// (2)
const MAX;
MAX = 500;

// (3)
let count = 1;
let count = 2;

// (4)
const LIMIT = 10;
LIMIT = 20;

// (5)
let total = 0;
total = 50;
번호 내용 에러 여부 에러 종류 이유
(1) let score;
score = 100; 		✅ 정상 — let은 선언 후 나중에 값 할당 가능
(2) const MAX;
MAX = 500; 		❌ 에러 SyntaxError const는 선언과 동시에 초기화(값 할당)를 반드시 해야 합니다. const MAX;처럼 선언만 하고 나중에 값을 넣을 수 없습니다.
(3) let count = 1;
let count = 2; 		❌ 에러 SyntaxError let은 같은 스코프 내에서 중복 선언이 불가능합니다. 이미 count라는 이름이 존재하므로 다시 let을 붙여 선언할 수 없습니다.
(4) const LIMIT = 10;
LIMIT = 20; 		❌ 에러 TypeError const로 선언한 변수는 재할당이 불가능한 '상수'입니다. 한 번 10이 들어갔다면, 20으로 값을 바꿀 수 없습니다.
(5) let total = 0;
total = 50; 		✅ 정상 — let은 재할당 가능

연습 문제 2

변수명 올바른지 판단하기

다음 변수명 중 올바른 것과 잘못된 것을 구분하고, 잘못된 이유를 설명하세요.

js
let 1stPlace
let first_place
let first-place
let $price
let const
let _count
let user name
let totalAmount
변수명 판정 이유
first_place ✅ 가능 언더스코어(_)는 변수명에 사용할 수 있다. (Snake Case 방식)
$price ✅ 가능 달러 기호($)는 변수명 첫 글자나 중간 어디든 사용할 수 있다.
_count ✅ 가능 언더스코어(_)로 변수명을 시작하는 것도 가능하다.
totalAmount ✅ 가능 영문자로 시작하며 중간에 대문자를 섞는 방식(Camel Case)은 자바스크립트 관습이자 표준이다.
1stPlace ❌ 불가 숫자로 시작할 수 없다. 변수명은 반드시 문자, $, _로 시작해야 한다.
first-place ❌ 불가 하이픈(-)을 사용할 수 없다. 자바스크립트에서 -는 뺄셈 연산자로 인식된다.
const ❌ 불가 const, let, if, for 등 언어 자체에서 이미 기능이 정해진 단어는 예약어(Reserved Word)로, 변수명으로 쓸 수 없다.
user name ❌ 불가 공백(띄어쓰기)을 포함할 수 없다. 두 단어를 잇고 싶다면 userName이나 user_name을 써야 한다.

연습 문제 3

출력 결과 예측하기

다음 코드의 출력 결과를 예측하세요.

js
function test() {
  const a = 10;
  {
    const b = 20;
    console.log(a + b); // (1) ?
  }
  console.log(a);     // (2) ?
  console.log(b);     // (3) ?
}
test();
출력 (1) 30

중괄호 { } 블록 안에서 상위 스코프의 a(10)와 현재 블록의 b(20)를 더할 수 있습니다.

출력 (2) 10

변수 a는 test 함수 전체에서 유효하므로 정상적으로 출력됩니다.

출력 (3) ReferenceError

변수 b는 중괄호 { } 블록 내부에서 const로 선언되었기 때문에, 해당 블록을 벗어나는 순간 메모리에서 사라집니다.

ReferenceError: b is not defined
