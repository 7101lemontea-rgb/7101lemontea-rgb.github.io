자료형 변환 (Type Conversion)

자료형 변환 개요

자바스크립트의 자료형 변환(Type Conversion)은 크게 두 가지로 나뉩니다. 개발자가 의도를 가지고 직접 바꾸는 명시적 변환과, 자바스크립트 엔진이 필요에 따라 자동으로 바꾸는 암묵적 변환(강제 형 변환)입니다.

🔧
명시적 변환

개발자가 Number(), String(), Boolean() 등 내장 함수를 사용해 타입을 직접 지정하는 방식입니다.

⚡
암묵적 변환

자바스크립트 엔진이 연산 과정에서 자동으로 타입을 변환합니다. 의도치 않은 버그가 자주 발생하므로 주의가 필요합니다.

01

명시적 변환

개발자가 내장 함수(Number(), String(), Boolean())를 사용하여 타입을 직접 지정하는 방식입니다.

a. 문자열(String) 타입으로 변환

변환 함수/메서드 설명
String(value) 가장 표준적. null, undefined를 포함한 모든 값을 안전하게 변환
value.toString() 객체의 메서드를 사용. null, undefined에서는 에러 발생. 진법 변환 가능
js
String(123);          // "123"
String(true);         // "true"
String(null);         // "null"

(123).toString();    // "123"
[1, 2, 3].toString();// "1,2,3"
(255).toString(16);  // "ff"

b. 숫자(Number) 타입으로 변환

변환 함수/메서드 설명
Number(value) 단순 숫자로 변환. 숫자가 아닌 문자가 섞이면 NaN 반환
parseInt(value) 정수 파싱. 앞부분에서 정수만 추출
parseFloat(value) 실수 파싱. 앞부분에서 소수점 포함 추출
js
Number("123");        // 123
Number("123.45");     // 123.45
Number("123.45abc");  // NaN

parseInt("123.45");    // 123
parseInt("123abc.45"); // 123
parseInt("123.45abc"); // 123
parseInt("abc123");    // NaN

parseFloat("123.45");    // 123.45
parseFloat("123abc.45"); // 123
parseFloat("123.45abc"); // 123.45
parseFloat("abc123.45"); // NaN

c. 불리언(Boolean) 타입으로 변환

변환 함수/메서드 설명
Boolean(value) 불리언 타입으로 변환
js
// Falsy 값: false, 0, -0, 0n, "", null, undefined, NaN
Boolean(0);       // false
Boolean("");      // false
Boolean(null);   // false

// Truthy 값: 위를 제외한 나머지 (빈 배열 [], 빈 객체 {} 포함)
Boolean([]);      // true
Boolean({});      // true
Boolean("false"); // true
Boolean("true");  // true

Falsy 값이란 Boolean()으로 변환했을 때 false가 되는 값입니다. false, 0, -0, 0n, "", null, undefined, NaN이 해당됩니다. 그 외의 모든 값은 Truthy로 true가 됩니다.

02

암묵적 변환

자바스크립트 엔진에 의해 자동으로 타입이 변환되는 것으로, 이 부분에서 의도치 않은 버그가 자주 발생하므로 주의해야 합니다.

a. 예시 1 : + 연산자 → 문자열 타입으로 변환

+ 연산자를 문자열과 함께 사용할 때, 문자열 타입으로 변환됩니다.

js
console.log("10" + 5);         // "105"
console.log("10" + undefined);  // "10undefined"
console.log("10" + null);      // "10null"
console.log("10" + true);      // "10true"

b. 예시 2 : + 이외의 산술연산자 → 숫자 타입으로 변환

+ 이외의 산술연산자를 문자열과 함께 사용할 때, 숫자 타입으로 변환됩니다.

js
console.log("10" * 5);         // 50
console.log("10" * undefined);  // NaN
console.log("10" * null);      // 0
console.log("10" * true);      // 10

연습 문제 1

출력 결과 예측하기

다음 코드의 출력 결과를 예측하세요.

js
console.log(Number("42abc")); // ?
console.log(parseInt("42abc"));// ?
console.log("5" + 3);          // ?
console.log("5" - 3);          // ?
출력 (1) NaN

Number() 함수는 숫자가 아닌 문자가 섞여 있으면 엄격하게 NaN(Not a Number)을 반환한다.

출력 (2) 42

parseInt()는 앞에서부터 읽다가 숫자가 아닌 문자를 만나는 지점까지만 숫자로 바꾼다. 따라서 '42'까지만 인식하여 출력한다.

출력 (3) "53"

+ 연산자는 피연산자 중 하나가 문자열이면 문자열 연결을 우선시하므로, 숫자 3이 문자열 '3'으로 바뀌어 붙는다.

출력 (4) 2

- 연산자는 오직 '산술 계산'만 수행한다. 문자열 "5"를 숫자 5로 자동 변환(암시적 형 변환)하여 계산을 완료한다.

연습 문제 2

Boolean 변환 결과 작성하기

다음 값들의 Boolean 변환 결과를 작성하세요.

js
Boolean("0");   // ?
Boolean(0);    // ?
Boolean([]);   // ?
Boolean("");   // ?
Boolean({});   // ?
Boolean(null); // ?
표현식 결과 이유
Boolean("0") true 비어 있지 않은 문자열은 무조건 true이다. (내용이 "0"이어도 상관없음)
Boolean(0) false 숫자 0은 자바스크립트의 대표적인 Falsy 값이다.
Boolean([]) true 빈 배열은 객체이며, 자바스크립트에서 모든 객체는 true로 취급된다.
Boolean("") false 빈 문자열(길이가 0인 문자열)은 Falsy 값이다.
Boolean({}) true 빈 객체 역시 객체이므로 true이다.
Boolean(null) false null은 값이 없음을 의미하는 대표적인 Falsy 값이다.

연습 문제 3

실행 결과 예측하고 이유 설명하기

다음 코드의 실행 결과를 예측하고 그 이유를 설명하세요.

js
console.log("100" / "20");    // ?
console.log("10" + 5 + 5);    // ?
console.log(5 + 5 + "10");    // ?
출력 (1) 5

나누기(/) 연산자는 문자열 연결 기능이 없다. 오직 산술 계산만 수행하므로, 두 문자열을 숫자로 변환하여 계산한다.

"100" → 100 "20" → 20 100 / 20 = 5
출력 (2) "1055"

왼쪽부터 순서대로 계산한다.

"10" + 5 → "105" (문자열 연결) "105" + 5 → "1055" (문자열 연결)
출력 (3) "1010"

왼쪽부터 순서대로 계산한다.

5 + 5 → 10 (숫자 합계) 10 + "10" → "1010" (문자열 연결)
