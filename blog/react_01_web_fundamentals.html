웹의 동작 원리

클라이언트-서버 구조부터 웹 페이지 렌더링, 3계층 아키텍처까지 — 웹이 어떻게 동작하는지 정리합니다.

주제

HTTP · HTML · CSS · JS · 아키텍처

목차

  • 01 클라이언트 / 서버
  • 02 웹 페이지의 구성
  • 03 웹 애플리케이션 구성 (3-Tier)

01

클라이언트 / 서버

클라이언트-서버 모델은 네트워크 구성 방식 중 하나로, 서비스 리소스를 제공하는 공급자(Server)와 이를 요청하여 사용하는 사용자(Client)로 역할을 명확히 분리한 분산 컴퓨팅 구조입니다.

💻

클라이언트 (Client)

서버가 제공하는 서비스를 호출하고 결과를 UI로 제공하는 주체입니다. 요청(Request)을 먼저 시작하는 능동적 역할을 하며, 웹 환경에서는 웹 브라우저가 대표적입니다.

🖥️

서버 (Server)

클라이언트의 요청이 올 때까지 대기하며 응답하는 수동적 주체입니다. 웹 환경에서는 Web Server, WAS, DB Server 등이 해당합니다.

웹 브라우저는 클라이언트의 가장 대표적인 예입니다. 주소창에 URL을 입력하는 순간, 브라우저가 서버에 요청을 보내는 것이 바로 클라이언트-서버 통신의 시작입니다.

02

웹 페이지의 구성

웹 브라우저가 서버로부터 리소스를 받아 화면에 출력하는 과정을 Critical Rendering Path(주요 렌더링 경로)라고 합니다. 웹 페이지는 구조·디자인·동작을 분리하여 관리하는 계층적 구조를 가집니다.

HTML

구조 (Structure)

웹 문서의 의미적 구조를 정의합니다. 브라우저는 이를 파싱하여 DOM 트리를 생성합니다.

CSS

표현 (Presentation)

HTML 구조에 시각적 스타일을 입힙니다. 브라우저는 이를 파싱하여 CSSOM 트리를 생성합니다.

JS

동작 (Behavior)

웹 페이지에 동적인 기능과 로직을 부여합니다. DOM과 CSSOM을 동적으로 조작하여 화면을 실시간으로 변경합니다.

03

웹 애플리케이션 구성

웹 애플리케이션은 복잡한 비즈니스 로직을 효율적으로 처리하기 위해 계층화된 구조를 갖습니다. 이를 3-Tier Architecture(3계층 구조)라고 부릅니다.

클라이언트 Presentation

사용자와 직접 상호작용하는 최전방 계층. 로직 처리는 최소화하고 UI 렌더링에 집중합니다.

HTML CSS JavaScript React Vue Angular
미들웨어 Logic

클라이언트와 DB 사이에서 비즈니스 로직을 수행합니다. 로그인 인증, 결제 처리, 데이터 변환, 부하 분산 등을 담당합니다.

Spring Django Node.js FastAPI Go
서버 (DB) Data

데이터를 영구적으로 저장하고 관리합니다. 미들웨어로부터 전달받은 쿼리를 수행하고 무결성을 보장합니다.

MySQL PostgreSQL MongoDB Redis

3-Tier 구조는 각 계층이 독립적으로 확장·교체될 수 있어 유지보수성과 확장성이 높습니다. 현대 웹 서비스 대부분이 이 구조를 기반으로 합니다.
